### 潜在关注点（非阻塞）

**1. `getActivePlayers()` 在 timeline 重建中的使用**

```ts
// hand-history.service.ts:buildReplayTimeline()
const activePlayers = table.getActivePlayers();
```

- 每次调用 `table.getActivePlayers()` 会返回当前 table 状态的玩家快照
- 但 `table` 是从 `tableManager.getTable(roomId)` 获取的**当前**活跃 table
- **如果**同一 room 在历史手局之后有新玩家加入，`getActivePlayers()` 会返回错误的（增多的）玩家列表

**建议:** 历史 replay 应该使用 `hand.players` 字段（或从 `HandAction` 去重）来构建 activePlayers，而非从当前 table 读取

**2. Hole Cards 的来源**

- Replay 数据中 `players[].holeCards` 来自 `table.getActivePlayers()[i].holeCards`
- 这依赖于 table 对象在 hand 结束时正确保存了 hole cards
- 需要确认 `table.stateSnapshot` 或其他机制在 hand 结束后保留了 hole cards 状态

**这两点需要 Coding 代理确认是否为实际问题（可能 getActivePlayers 在当前实现中已返回历史玩家）。**
