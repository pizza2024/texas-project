## P0 — 立即修复

### P0-BACKEND-001: `resolveFoldWin()` 缺少 `bestCards` 字段

**文件:** `table-round.ts:293–321`

Fold 获胜时，`lastHandResult` 中 winner 条目缺少 `bestCards: []`。当客户端收到结果时，winner 的 `bestCards` 为 `undefined`，可能导致 UI 渲染错误。

**对比:** `performShowdown()` 正确为 folded 玩家设置了 `bestCards: []`。

**修复建议:** 在 winner 条目中添加 `bestCards: []`。

---

### P0-BACKEND-002: Straddle 后 `calledAllIn` 未重置

**文件:** `table-round.ts` — `attemptStraddle()` (lines 58–102)

Straddle 操作后，`this.table.calledAllIn` 从未被重置。`processAction` 中 `raise` 检查 `if (this.table.calledAllIn !== null)` 会阻止 raise，但 Straddle 流程不会清除该状态，可能导致后续操作被错误阻止。

**修复建议:** 在 `attemptStraddle()` 开头添加 `this.table.calledAllIn = null`。

---

### P0-BACKEND-003: `persistSettlementRecords()` catch 块中使用未初始化 `handId`

**文件:** `table-manager.service.ts:293–377`

```ts
let handId: string;  // 未初始化
try {
  const hand = await this.prisma.hand.create({...});
  handId = hand.id;  // 仅在成功时赋值
} catch (err) {
  void this.adminLogSettlementFailure(roomId, handResult, err, handId);  // BUG: handId 未定义
  return;
}
```

如果 `prisma.hand.create` 抛出异常，`handId` 未定义，`adminLogSettlementFailure` 会抛出 `ReferenceError`，吞掉原始错误。

**修复建议:** 初始化 `handId` 为 `undefined`，catch 中检查后再使用。

## P1 — 高优先级

### P1-BACKEND-001: 手牌复盘 `netProfit` 计算错误

**文件:** `hand-history.service.ts:135`

```ts
const netProfit = (settlement?.amount ?? 0) - pa.bet;
```

`pa.bet` 是玩家本局投入总量，`netProfit = 500` 表示赢了 500。但若 `settlement = 500`，玩家实际净亏 500（投入 1000）。正确公式应使用 `totalBet` 而非 `pa.bet`。

**修复建议:** 使用 `TableSnapshot` 中的 `totalBet` 计算真实 P&L。

---

### P1-BACKEND-002: `checkAndAutoFoldSittingOut` 3次超时后无视觉反馈

**文件:** `table-player-ops.ts:189–218`

3次超时后玩家标记为 `SITOUT` + `hasActed = true`，但玩家仍在 `activePlayerIndex` 中，且无 UI 提示玩家已被跳过。

**修复建议:** 考虑触发系统消息通知玩家。

### P1-BACKEND-003: WebSocket `player-action` 缺少 `userCurrentRoomId` 校验

**文件:** `game.handler.ts:270–288`

客户端发送 `roomId: null` 时，`validated.roomId &&` 检查被跳过，`roomId` 直接从 `userCurrentRoomId` 派生。若 `userCurrentRoomId` 伪造，校验失效。

**修复建议:** 在 `!action || !roomId` 检查前添加 `if (!userCurrentRoomId) return`。

---

### P1-BACKEND-004: `getUserAvailableBalance` 返回 null 时静默使用 0

**文件:** `game.handler.ts:143`

```ts
const balance = isAlreadySeated
  ? null
  : await gateway.tableManager.getUserAvailableBalance(userId);
```

若 `balance` 返回 `null`（类型签名是 `Promise<number>`），`?? 0` 会静默使用 0，导致玩家以 0 筹码入桌。

**修复建议:** 验证返回值类型，拒绝 null。

### P1-WEB-001: ReplayModal `fetch` 缺少认证 Token

**文件:** `components/replay/ReplayModal.tsx:74`

```ts
fetch("/api/table-engine/hands/${handId}/replay"); // 绕过 api 客户端
```

裸 `fetch()` 不经过 axios interceptor，无 `Authorization: Bearer <token>`。若该 endpoint 需要认证，会返回 401 或暴露数据。

**修复建议:** 使用 `api` 客户端或手动添加 `Authorization` header。

### P1-WEB-003: `SocketSessionProvider` useEffect 依赖问题

**文件:** `components/socket-session-provider.tsx:28`

`setForceLogoutHandler` 和 `setRejoinAvailableHandler` 每帧注册（无 guard），但 `useEffect` 依赖只有 `[router]`。若 router 变化可能导致重复注册。

**修复建议:** 用 `useCallback` 包装 handler 或将注册逻辑移至 `useEffect([])`。

---

### P1-WEB-004: `AutoPlayPanel` 闭包导致 Auto-play 不稳定

**文件:** `components/replay/AutoPlayPanel.tsx:45–57`

`useEffect` 依赖 `onPlayingChange` 和 `onIndexChange`，interval 回调内调用 `onPlayingChange(false)` 会触发依赖变化，导致 interval 频繁重建，可能出现双步进或跳步。

**修复建议:** 用 `useRef` 存储 callbacks 避免破坏稳定性。
