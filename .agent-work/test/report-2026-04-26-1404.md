## CodeReview 发现

### P1 — 新发现

#### 🔴 P1-WITHDRAW-003 — `refundChips` 无事务保护（REJECT 路径）

**文件:** `apps/backend/src/withdraw/withdraw.service.ts:360-367`

```ts
if (action === 'REJECT') {
  await this.refundChips(request.userId, request.amountChips, id, reason ?? 'Rejected by admin');
```

`refundChips`（第439-474行）执行 `wallet.upsert` + `user.update` + `transaction.create`，但**不在 `$transaction` 内**。REJECT 路径中，并发请求可导致钱包余额丢失更新（lost update）。APPROVE 路径（第212行）正确使用了事务。

**建议:** 将 `refundChips` 逻辑包裹在 `prisma.$transaction` 中。

#### 🔴 P1-WITHDRAW-004 — `refundChips` 无事务保护（handleWithdrawFailure）

**文件:** `apps/backend/src/withdraw/withdraw.service.ts:594-598`

```ts
await this.refundChips(request.userId, request.amountChips, requestId, reason);
```

同上，`handleWithdrawFailure` 调用 `refundChips` 也缺少事务保护。

#### 🔴 P1-CHAT-XSS — 聊天室消息内容未转义（XSS 漏洞）

**文件:** `apps/web/components/chat/ChatPanel.tsx:198`（同时存在于 `apps/web/app/room/[id]/components/ChatPanel.tsx:198`）

```tsx
<span>{msg.content}</span>
```

用户发送的聊天内容直接作为 React children 渲染，未经过任何 HTML 转义或净化。恶意用户可发送 `<script>`、`onerror`、`onload` 等 payload 注入其他用户浏览器。

**建议:** 使用 `dompurify` 或对 HTML 特殊字符进行转义后再渲染。

### P2 — 新发现

#### 🟡 P2-DEPOSIT-ATOMIC — `checkAddressDeposits` 每条 deposit 非原子操作

**文件:** `apps/backend/src/deposit/deposit.service.ts:357-428`

循环内执行 `setBalance` → `depositRecord.create` → `transaction.create` → `user.update`，若第3/4步抛出异常，用户筹码已增加但账本无记录，且游标已推进，存款变成"幽灵记录"。

**建议:** 将每个 deposit 事件的处理包裹在 `prisma.$transaction` 中。

#### 🟡 P2-DEPOSIT-TOCTOU — `getOrCreateDepositAddress` 索引竞争

**文件:** `apps/backend/src/deposit/deposit.service.ts:62-91`

两个并发调用可能计算出相同的 `nextIndex`，导致 P2002 冲突后降级为 findUnique，浪费一次 HD 钱包派生计算。

**建议:** 使用 `INSERT ... ON CONFLICT` 语义（Prisma `upsert`）或唯一约束简化。

#### 🟡 P2-CHAT-IDEMPOTENCY — 聊天消息无幂等性键

**文件:** `apps/backend/src/websocket/game.handler.ts:560`

服务器为每条聊天消息生成 `crypto.randomUUID()` 作为 ID。客户端若因超时重试，同一消息会被当作两条消息广播。

**建议:** 客户端生成唯一幂等键，服务器去重。

#### 🟡 P2-AUTH-OTP-PARSE — OTP JSON 解析无 try/catch

**文件:** `apps/backend/src/auth/auth.service.ts:200`

```ts
const { code: storedCode, expiresAt } = JSON.parse(stored);
```

Redis 数据损坏时直接抛出未捕获异常。应捕获并返回 `BadRequestException`。

#### 🟡 P2-WS-RATE-UNIT — 密码尝试窗口时间单位混淆

**文件:** `apps/backend/src/websocket/connection-state.service.ts:107`

```ts
Math.max(0, ttl) +
  (this.PASSWORD_BAN_MS - this.PASSWORD_ATTEMPT_WINDOW_MS) / 1000;
```

`ttl`（秒）与毫秒除以1000后的值相加，若 `ttl` 本身已是毫秒值则单位不匹配。需要确认 `redisService.ttl()` 返回值的单位。

#### 🟡 P2-CODE-PATTERN — mission/table-engine sequential DB calls

**文件:** `apps/backend/src/mission/mission.service.ts:374-380`

`onHandWon` 中4个 `progressMission` 顺序 await，可合并为 `Promise.all`。

**文件:** `apps/backend/src/table-engine/table-manager.service.ts:63-98`

启动清理 `cleanupOfflineResidueOnStartup` 顺序执行每玩家的 `resetBalanceAndUnfreeze`，大并发下拖慢启动。
