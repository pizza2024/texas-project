## 🔴 P0（立即修复）

### P0-E2E-FIX-001 — rakeback.e2e-spec.ts 测试配置错误（第3轮）

**问题:** `rakeback.e2e-spec.ts` 位于 `apps/backend/test/` 但 Jest 根配置 `testMatch` 仅扫描 `apps/backend/src/`，导致测试被忽略（0 matches）。

**根因:** Jest `testMatch` pattern 在 monorepo root `jest.config.ts` 中设为 `"**/?(*.)+(spec|test).ts"` 但 `testPathIgnorePatterns` 排除了 `/apps/`。Backend 独立 `jest.config.ts` 不存在。

```json
// Root jest.config.ts
testMatch: ["**/?(*.)+(spec|test).ts"],
testPathIgnorePatterns: ["/node_modules/", "/apps/", "/dist/"]  // ← 排除了 apps/backend/test/
```

**验证:**
```
$ cd apps/backend && npx jest src/rakeback/rakeback.controller.spec.ts
→ PASS (6 tests)

$ cd apps/backend && npx jest test/rakeback.e2e-spec.ts
→ No tests found, exiting with code 1
  testMatch:  - 0 matches
```

**修复方案（三选一）:**
1. **推荐:** 在 `apps/backend/` 创建独立的 `jest-e2e.json` 或 `jest.config.ts`，定义 `testMatch: ["**/test/**/*.e2e-spec.ts"]`
2. **备选:** 将 `rakeback.e2e-spec.ts` 移入 `src/rakeback/` 并重命名为 `.spec.ts`
3. **Root config:** 修改 monorepo root `jest.config.ts` 的 `testPathIgnorePatterns` 不再排除 `/apps/backend/test/`

**影响:** E2E 测试套件不完整，CI 无法验证 rakeback 端点。
