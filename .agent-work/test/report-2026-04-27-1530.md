### 发现：Frontend `emoji-reaction` 事件缺少 `roomId` 字段 ⚠️

**Backend 发送** (`game.handler.ts:597`):
```ts
gateway.server.to(roomId).emit('emoji-reaction', {
  id: crypto.randomUUID(),
  userId,
  username,
  emoji: validated.emoji,
  timestamp: Date.now(),
});
```

**Frontend `page.tsx` 监听期望** (`page.tsx:343`):
```ts
socket.on('emoji-reaction', (data: { roomId: string; userId: string; emoji: string }) => {
  if (data.roomId !== id) return; // ← roomId 在 payload 中
```

**影响**: 客户端依赖 `data.roomId !== id` 做过滤，但后端 payload 中没有 `roomId` 字段。所有客户端在非当前房间的场景下都无法正确过滤（当前房间逻辑仍正常，因为没有 roomId 时 `undefined !== id` 为 true，可能误拦截）。

**严重程度**: P2 — 仅影响多房间同用户场景，正常单房间游戏不受影响。
