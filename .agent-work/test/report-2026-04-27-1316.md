## 新发现 — 本轮

### P2-NEW-020 — Blast `activeGames` Map 无 TTL/持久化

**文件:** `blast.service.ts:84`

```typescript
private readonly activeGames = new Map<string, BlastGameRecord>();
```

**问题:**

- `activeGames` 是进程内内存 Map，无 TTL 清理
- `forfeitBlast` 从 Map 删除条目，但正常完成（无 forfeit）的游戏从不删除
- 如果游戏正常结束（所有玩家打完或提前 exit），`endBlastGame` 调用 `deleteActiveGame` 从 Map 移除 ✅
- **风险场景:** 服务器崩溃前正在 `startBlastGame` → 添加到 Map → 但 `endBlastGame` 未调用 → Map 条目泄漏（但进程重启时清空，风险低）

**评估:** P3 — 低优先级，Map 生命周期与进程绑定，进程重启清空
