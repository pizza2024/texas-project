## 🟡 P2 — 业务逻辑不完整

### P2-CLUB-003: `isClubOnly` DTO 字段未在 `createRoom` 中使用

**问题**: `CreateRoomDto` 定义了 `isClubOnly?: boolean` 和 `clubId?: string`，但 `RoomService.createRoom()` 未读取/使用这两个字段。

```typescript
// create-room.dto.ts — 有定义
clubId?: string;
isClubOnly?: boolean;

// room.service.ts createRoom — 未使用
async createRoom(dto: CreateRoomDto, userId: string) {
  // dto.clubId 和 dto.isClubOnly 从未被引用
  const room = await this.prisma.room.create({
    data: {
      name: dto.name,
      blindSmall: dto.blindSmall,
      // ... 其他字段
      // ❌ clubId 未设置
      // ❌ isClubOnly 未设置
    },
  });
}
```

**影响**: Club 专属房间功能无法通过 API 创建，仅数据库字段存在。

**修复建议**: 在 `room.service.ts` `createRoom` 的 `prisma.room.create` 中添加 `clubId: dto.clubId` 和 `isClubOnly: dto.isClubOnly ?? false`。

### P2-CLUB-004: `isClubOnly` 筛选逻辑缺失

**问题**: `RoomService.findAll()` 和 `findAllPaginated()` 仅按 `clubId` 过滤，未检查 `isClubOnly`。

```typescript
// room.service.ts
where: { isMatchmaking: false, ...(clubId && { clubId }) },
// ❌ 缺少: isClubOnly: true 时应同时验证用户 club membership
// ❌ 缺少: isClubOnly: false 时应排除 club-only 房间（对非成员隐藏）
```

**影响**: Club 专属房间对非成员可见，可被直接 join。

**修复建议**: `findAll` / `findAllPaginated` 在返回前需过滤 `isClubOnly` 房间——对非成员用户隐藏（isClubOnly=true 且用户非俱乐部成员）。
