### 🔴 P0 — 4个测试持续失败（app.gateway.spec.ts）

**文件**: `apps/backend/src/websocket/app.gateway.spec.ts`
**现状**: 7 passed, 4 failed / 11 total（较上轮5个失败有改善，但根因未解决）

#### 根因分析

TimerService mock 架构问题：`finalizeSettlement` / `finalizeActionTimeout` 等 mock 只调用 `table.begin*Countdown` 设置时间戳，但不推进游戏状态。导致 `jest.advanceTimersByTimeAsync()` 触发时，游戏状态未实际推进。

| #   | 用例                                                                      | 失败表现                          | 根因                                                                                        |
| --- | ------------------------------------------------------------------------- | --------------------------------- | ------------------------------------------------------------------------------------------- |
| 1   | `runs settlement countdown, enters ready countdown, and then auto-starts` | `table.resetToWaiting` 未被调用   | `finalizeSettlement` mock 只调用 `beginSettlementCountdown`，未推进状态                     |
| 2   | `rebuilds an in-progress settlement timer from restored state`            | `table.resetToWaiting` 未被调用   | 同上                                                                                        |
| 3   | `auto-checks on timeout when checking is allowed`                         | `table.getTimeoutAction` 未被调用 | `finalizeActionTimeout` mock 未调用 `getTimeoutAction`                                      |
| 4   | `auto-folds on timeout when checking is not allowed`                      | `table.processAction` 未被调用    | 同上，且 `scheduleActionTimeout` 已移至 TimerService，测试直接调用 gateway 方法不触发 timer |

#### 修复方案

TimerService mock 需要在调用 `begin*Countdown` 之后，**同步**触发状态推进回调：

```typescript
// schedulePostHandFlow mock 应改为：
schedulePostHandFlow: jest.fn().mockImplementation((_server: any, _roomId: string, table: any) => {
  if (table?.beginSettlementCountdown) {
    table.beginSettlementCountdown(TimerService.SETTLEMENT_DURATION_MS);
  }
  // FIX: 直接触发 finalizeSettlement 回调，模拟 timer 触发
  if (table?.processAction) {
    // 推进状态机：SETTLEMENT -> WAITING
    table.currentStage = GameStage.WAITING;
    table.settlementEndsAt = null;
    if (table?.resetToWaiting) table.resetToWaiting();
  }
}),
```

对于 `auto-fold` 测试，直接调用 `gateway.scheduleActionTimeout` 无效（方法已移至 TimerService），需改为通过 `timerService.finalizeActionTimeout` 触发。
