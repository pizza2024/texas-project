### P1-NEW-003: `setBalances` 原子化 ✅ 已修复

**修复前:** Phase 1 (`wallet.upsert`) 和 Phase 2 (`user.coinBalance.update`) 分离，Phase 2 用 `Promise.allSettled` 独立执行 — Phase 1 成功但 Phase 2 失败时数据不一致。

**修复后:** Phase 1 + Phase 2 合并为单一 `$transaction`，bot 用户跳过 coinBalance 同步（无 User 记录），real user 失败则整体回滚。

```ts
// 修复后: entries.flatMap() 合并 wallet + user update 到同一 transaction
await this.prisma.$transaction(
  entries.flatMap(({ userId, balance }) => {
    const walletOp = this.prisma.wallet.upsert({...});
    if (userId.startsWith(BOT_ID_PREFIX)) return [walletOp];
    const userOp = this.prisma.user.update({ where: { id: userId }, data: { coinBalance: normalized }});
    return [walletOp, userOp];
  })
);
```

### P1-NEW-004: `exchangeChipsToBalance` TOCTOU ✅ 已修复

**修复前:** `availableChips` 在事务外通过 `getAvailableBalance(userId)` 读取，并发双花攻击路径存在。

**修复后:** `availableChips` 计算移至事务内部，结合 `frozenChips` 扣减后检查，防止并发 double-spend。

```ts
// 修复后: balance 检查在 transaction 内部
const wallet = await tx.wallet.findUnique({ where: { userId } });
const currentBalance = wallet?.chips ?? ...;
const frozenChips = wallet?.frozenChips ?? 0;
const availableChips = Math.max(0, currentBalance - frozenChips);
if (chipsAmount > availableChips) {
  throw new BadRequestException('Insufficient available chips');
}
```
