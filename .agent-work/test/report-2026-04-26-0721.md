- **描述:** `table.processAction(userId, action, amount)` 直接接收客户端传来的 `amount`，游戏引擎内部虽有 `all-in-call-protection` 逻辑，但 **server-side 应该在调用前校验 `amount <= player.stack`**，否则恶意客户端可发送超大 `raise` 值尝试溢出。

### 🟡 P2 — 中优先级

**P2-ROOM-PASSWORD: 房间密码存 plaintext 在 sessionStorage**
- **文件:** `apps/web/app/room/[id]/page.tsx:328`, `rooms/page.tsx:1006`
- **描述:** `sessionStorage.setItem('room-password:${roomId}', password)` 明文存储房间密码。若 XSS 存在，攻击者可读取所有已访问房间密码。
- **建议:** 改用一次性 token 或内存态（不持久化）

**P2-JWT-LOCALSTORAGE: JWT 存于 localStorage — XSS 目标**
- **文件:** `apps/web/lib/auth-context.tsx:50`, `lib/api.ts:13`
- **描述:** `localStorage.setItem('token', jwt)` 可被任何页内 JS 读取。与 `httpOnly` cookie 相比，无法防御 XSS 盗取 token。
- **建议:** 迁移至 `httpOnly` cookie，或增加 CSP + CSRF token

**P2-CHAT-INJECTION: 聊天消息 `username` 来源于服务端但未验证**
- **文件:** `apps/web/components/chat/ChatPanel.tsx:68-71`
- **描述:** `socket.on('chat-message', (msg) => { render(msg.username) })` — 若服务端被攻陷或消息遭篡改，用户名可被伪造。
- **建议:** 客户端应仅信任 `userId`，并由前端映射到昵称显示

**P2-ROOM_UPDATE-NO-VALIDATION: `room_update` 事件无 Schema 校验**
- **文件:** `apps/web/app/room/[id]/page.tsx:335-337`
- **描述:** `socket.on('room_update', (data: TableState) => { setTable(data); })` 接收完整 TableState 直接设置状态，恶意/畸形数据可能破坏 UI。
- **建议:** 接入 Zod/Joi schema 校验

**P2-PROVIDER-ORDER: `SocketSessionProvider` 在 `AuthProvider` 之外初始化**
- **文件:** `apps/web/app/layout.tsx:62-69`
- **描述:** Provider 嵌套顺序可能导致 `SocketSessionProvider` 初始化时获取不到 auth context，socket 重连时 `handleExpiredSession` 行为不确定。
- **建议:** 确认 `AuthProvider` 包裹所有需要 auth 的 provider

**P2-AUTO-FOLD-DISCONNECT: 断线自动操作无二次确认**
- **文件:** `apps/web/app/room-mobile/[id]/page.tsx:179-202`
- **描述:** 网络抖动导致断线后自动执行 `fold`/`check`，玩家可能意外失去操作机会。
- **建议:** 显示断线遮罩 + 倒计时，而非静默自动操作
