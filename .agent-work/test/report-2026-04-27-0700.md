### P1 — 详情

**P1-DECORATOR-001: `@InjectServer()` 依赖注入缺失**

```typescript
// websocket/decorators.ts
export const InjectServer = createParamDecorator(
  (_data: unknown, ctx: ExecutionContext): Server => {
    const request = ctx.switchToHttp().getRequest();
    return request.ioServer; // ← 始终返回 undefined（未被任何中间件注入）
  },
);
```

**问题分析：**
- `@InjectServer()` decorator 设计用于在 HTTP 控制器中注入 Socket.io `Server` 实例
- `request.ioServer` 依赖某处 middleware 将 `ioServer` 附加到 request 对象
- **全项目搜索结果：** 没有任何 middleware、interceptor、guard 或 setup 代码将 `server`/`ioServer` 附加到 HTTP request
- `app.gateway.ts` 在 gateway 类上有 `@WebSocketServer() server: Server`（WS 层面），但 HTTP 路由控制器没有对应的 request 注入机制
- 搜索 `ioServer` 仅出现在 `decorators.ts` 一处
- 如果 `TournamentScheduleController` 或其他控制器尝试使用 `@InjectServer()`，传入的 `server` 参数为 `undefined`

**风险评估：**
- 当前控制器（`TournamentScheduleController`）**未使用** `@InjectServer()`
- `Controller` 通过 constructor 注入 `AppGateway`（含有 `server`）来广播 WS 事件
- **但是**：如果后续代码尝试使用 `@InjectServer()` 会遇到运行时 `undefined` 问题
- 属于"已创建但未连接"的死代码，存在未来误用风险

**建议修复方案（三选一）：**
1. **删除 `decorators.ts`**：如果 `@InjectServer()` 无用，删除它避免误导
2. **实现注入机制**：创建全局 middleware 将 `app.gateway.server` 附加到每个 HTTP 请求
3. **添加 useFactory provider**：在 NestJS 模块中正确配置 `ioServer` 的注入作用域
