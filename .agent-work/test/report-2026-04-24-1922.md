## 一、CodeReview 发现

### 🔴 P0 — 必须修复

#### P0-001: `straddle` 操作被验证层阻止（功能损坏）

| 项目       | 详情                                                     |
| ---------- | -------------------------------------------------------- |
| **文件**   | `packages/shared/src/validation.ts` (PlayerActionSchema) |
| **严重性** | P0 — 功能完全失效                                        |
| **影响**   | 客户端无法执行 straddle 动作                             |

**问题描述**：

`VALID_ACTIONS` 常量包含 `straddle`：

```ts
// apps/backend/src/websocket/constants.ts:26
export const VALID_ACTIONS = [
  "fold",
  "check",
  "call",
  "raise",
  "allin",
  "straddle",
  "sit-out",
] as const;
```

`table-engine` 完整实现了 straddle 逻辑（`straddle.spec.ts` 有大量测试，覆盖位置限制、重新开放行动、每手牌限制等）。

但 `PlayerActionSchema` 验证层**缺少 `straddle`**：

```ts
// packages/shared/src/validation.ts
export const PlayerActionSchema = z.object({
  action: z.enum(["check", "fold", "call", "raise", "allin", "sit-out"]), // ← 缺少 'straddle'
  amount: z.number().nonnegative().optional(),
  roomId: z.string().min(1).optional(),
});
```

**影响**：客户端发送 `straddle` 操作时，Zod 验证在 WebSocket 网关层直接拒绝，错误消息 `"Invalid request"`，**永远不会到达游戏引擎**。straddle 功能对玩家完全不可用。

**修复方案**：

```ts
// packages/shared/src/validation.ts
action: z.enum(['check', 'fold', 'call', 'raise', 'allin', 'straddle', 'sit-out']),
```

**建议**：同时在 `validate.ts` 的错误日志中加入 `straddle` 可用性提示。

### 🟡 P1 — 重要（需近期处理）

#### P1-001: 断线重连清理的竞态条件

| 项目         | 详情                                                     |
| ------------ | -------------------------------------------------------- |
| **文件**     | `apps/backend/src/websocket/connection-state.service.ts` |
| **严重性**   | P1 — 中等 — 可能导致房间状态不一致                       |
| **发现方式** | 代码审查                                                 |

**问题描述**：

`scheduleDisconnectCleanup` 中的定时回调传入空字符串作为 `socketId`：

```ts
// line ~180
const timeout = setTimeout(async () => {
  if (await hasOtherActiveSocketFn(userId, "")) {
    // ← 空字符串！
    return;
  }
  // ... cleanup logic
}, DISCONNECT_GRACE_PERIOD_MS);
```

当 timeout 触发时，`hasOtherActiveSocket(userId, '')` 用空字符串比对，**无法反映用户当前的真实 socket 连接状态**。可能导致：

- 用户重新连接后，断线清理仍被执行（错误将玩家踢出房间）
- 持有多个 socket 的玩家（多标签页），清理逻辑判断错误

**修复方案**：在 schedule 时捕获正确的 socketId，或在回调中重新查询当前状态。
