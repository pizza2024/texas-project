### W-006 — WebSocket 动作 rate-limit 降级问题 (P2)

**文件**: `apps/backend/src/websocket/app.gateway.ts` 行 118-140

**问题**:

- `checkRateLimit` 使用 Redis `incr` 做原子计数
- Redis 不可用时降级到内存 Map（非分布式安全）
- 降级后不同实例各有一份计数，可能被绕过

**风险**: 多实例部署时，用户每毫秒可在每实例触发10次，总计可超过限制

**建议**: Redis 不可用时，应拒绝超限用户（类似 W-001 的处理方式）

**优先级**: P2（仅多实例部署受影响）
