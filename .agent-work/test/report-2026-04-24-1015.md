## CodeReview — 10:15 检查

### 后端核心模块审查

#### `table-engine` — ✅ 健康

- **`table-game-logic.ts`**: Fisher-Yates shuffle 使用 `crypto.getRandomValues`，密码学安全。盲注分配、起手牌发牌逻辑正确。
- **`straddle.spec.ts`**: 测试覆盖 straddle 放置限制（PREFLOP 检查、已下注检查、hasActed 检查、stack 不足检查）和合法 straddle 场景，包括 ALLIN 边界。
- **`table-state.ts`**: GameStage 枚举完整（WAITING → SETTLEMENT），`HandResultEntry` 接口包含 bestCards（用于 showdown 显示）。

#### `websocket` — ✅ 健康

- **`game.handler.ts`**: 验证流程完整：
  - `join_room`: rate limit → brute-force 保护 → 密码校验 → 余额检查 → `withRoomLock` 串行化
  - `player_action`: rate limit → `withRoomLock` → `ensureRecoveredRoundFlow` → `processAction` → 持久化
  - `player_ready`: countdown 保护避免重复启动
- **`app.gateway.ts`**: `withRoomLock` 使用 Promise chain 吸收 rejection，防止 rejected promise 进入 Map 导致后续调用短路。`withUserLock` 同理。
- **`validate.ts`**: Zod schema 验证，失败时返回 `null` 并 emit `invalid_message`。
- **`app.gateway.spec.ts`**: 877 行全面测试，使用 `jest.useFakeTimers()` + sync `setTimeout` 回调模拟计时器，避免异步 microtask 时序问题。

#### `auth` — ✅ 健康

- **`auth.service.ts`**: bcrypt 密码校验，`SESSION_KEY_PREFIX` + Redis TTL 单设备登录，OTP 60s 限速，10分钟过期。
- **JWT payload** 包含 `sessionId`，WebSocket `handleConnection` 验证 Redis session 一致性，不一致则 `force_logout`。

#### `wallet` — ✅ 健康

- **`wallet.service.ts`**: `freezeBalance` 在玩家入桌时锁定全部 stack，防止双开。`getAvailableBalance` = `chips - frozenChips`。
- **`exchangeChipsToBalance`**: TRC20/ERC20 地址校验，原子性 `$transaction` 扣款 + 创建提现请求。

#### `withdraw` — ✅ 健康

- **`withdraw.service.ts`**:
  - `checkAndConsumeCooldown`: Redis SETNX 分布式锁，Redis 不可用时 **BLOCK 所有提现**（注释明确标注 `[SECURITY-WITHDRAW-BYPASS]`），不会因 in-memory fallback 而被绕过。
  - 提现创建在 `$transaction` 内完成：扣款 → 创建 `WithdrawRequest` → 记录 `Transaction`。
  - BullMQ retry 队列降级：队列不可用时 fallback 到 `executeChainWithdrawWithRetry`。
