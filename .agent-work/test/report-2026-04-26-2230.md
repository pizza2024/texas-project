## 本轮 CodeReview 发现

### 🔴 新增 P2 — `use-game-socket.ts` React ref 违规访问（2 errors）

**文件:** `apps/web/lib/use-game-socket.ts`

**问题:** 返回值 `activeFlights` 直接读取 `flightsRef.current`，这在 React 渲染阶段访问 ref 值，违反了 `react-hooks/refs` 规则。

```typescript
// Line 79 — 在渲染阶段读取 ref
const activeFlights = flightsRef.current; // ❌ Cannot access ref during render

// Line 85 — 在 return 中暴露 ref 值
return { emitEmoji, activeFlights, removeFlight };
//                                    ^^^^^^^^^^ flightsRef.current — 永远为空数组
```

**影响:**

1. ESLint `react-hooks/refs` 规则报告 2 个 error，CI 失败
2. `activeFlights` 永远为空 `[]`（ref 是可变容器，mutation 不会触发重渲染）
3. Emoji overlay 组件调用 `useGameSocket()` 后无法获取任何飞行中的表情——**所有 incoming emoji reaction 都不会显示在 UI 上**

**根因:** 设计者意图用 ref 而非 state 来避免每次表情触发都重渲染整个房间页面，但没有用 `useState` + `setState` 来触发更新。

**修复方向:** 用 `useState<EmojiFlight[]>([])` 替换 `useRef`，在 `socket.on` handler 中调用 `setFlights(prev => [...prev, flight])`。
