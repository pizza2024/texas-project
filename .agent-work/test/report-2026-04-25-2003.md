## 本轮 CodeReview 发现

### 新发现

**P2-TEST-001**: `withdraw.service.spec.ts:323` — `prisma.$transaction` mock 实现与实际 API 不兼容

```typescript
// 当前 mock（line 323）:
prisma.$transaction.mockImplementation(async (fn) => fn(prisma));

// 但 handleWithdrawFailure line 594 调用:
// await this.refundChips(...);           ← refundChips 内部不使用 $transaction
// await this.prisma.$transaction([...]) ← 数组形式的 callback-based API
// ↑ $transaction([ops]) — single callback 形式，不是 (fn) => fn(prisma)
```

`refundChips` 内部使用 `$transaction([...])`（数组语法），但 mock 实现只支持函数形式 `fn(prisma)`。当 `handleWithdrawFailure` 调用 `refundChips` 再触发 `$transaction` 时，mock 的 `fn` 是数组而非函数，导致 `TypeError: fn is not a function`。

该测试在完整套件运行时不会阻断其他测试，但 `handleWithdrawFailure` 的 refund 分支 **未被任何测试覆盖**。

**影响**: 低 — 该路径在 chain transfer 失败时触发，属于异常路径，且最终能正确退款。

**修复建议**: 更新 mock 支持两种调用形式，或在 spec 中为 `handleWithdrawFailure` 添加独立测试用例。
