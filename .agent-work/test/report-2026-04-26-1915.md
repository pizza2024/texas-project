### 🔴 P1 — Chat Username 身份冒充（建议升 P1）

**文件:** `apps/backend/src/websocket/game.handler.ts` 行 557-558

```typescript
const username =
  (client.data.user as { username?: string } | undefined)?.username ??
  "未知玩家";
gateway.server.to(roomId).emit("chat-message", {
  id: crypto.randomUUID(),
  userId,
  username, // ← 直接取 JWT payload，未与 DB 交叉验证
  content: trimmed,
  timestamp: Date.now(),
});
```

**问题:**

- `username` 来源于 JWT payload（`client.data.user.username`），在 `handleConnection` 行 290-294 由 `jwtService.verify(token)` 验证
- JWT 的 `username` 字段在用户注册时由客户端传入，未与 `User` 表 `username` 字段做交叉验证
- 攻击者注册时将 `username` 设为他人身份（如"管理员"），在聊天中可冒充他人
- 这是**身份冒充风险**，非单纯 XSS

**建议:** 在 `handleChatMessage` 中增加 DB lookup，从 `User` 表基于 `userId` 查询真实 `username`，而非信任 JWT payload 中的值。

**相关:** P2-CHAT-INJECTION（Productor 建议升 P1）
