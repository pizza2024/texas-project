#### ✅ P2-CLUB-007: Admin Room 一致性校验仍然缺失

- **位置**: `AdminRoomController` (admin-room.controller.ts) 使用 `CreateRoomDto`（admin/dto/create-room.dto.ts）
- **问题**: Admin `createRoom` 接口直接透传 DTO，**未复用** RoomController 中已修复的 `isClubOnly + clubId` 一致性校验
- **风险**: 管理员仍可通过后台 API 创建 `isClubOnly=true + clubId=null` 的非法房间
- **建议**: 在 `AdminService.createRoom()` 或 `AdminRoomController.createRoom()` 中添加相同校验

```typescript
// admin-room.controller.ts — createRoom 方法缺少校验
async createRoom(@Body() dto: CreateRoomDto, @Req() req: AdminRequest) {
  // ❌ 缺失: if (dto.isClubOnly === true && !dto.clubId) throw BadRequestException(...)
  const room = await this.adminService.createRoom(dto);
  // ...
}
```

#### 🔴 P2-CLUB-003: Club 专属房间前端权限过滤 + UI 缺失

- **状态**: 后端 RoomController 校验已修复，但前端房间列表页 `/rooms` **未过滤** `isClubOnly=true` 的房间
- **问题**: 普通用户可以看到 Club 专属房间，但无法加入（后端会拒绝）
- **建议**: 前端房间列表按 `isClubOnly + clubId` 做权限过滤，非俱乐部成员不可见

#### 🟡 P2-CLUB-004: Club 专属房间 WebSocket 加入权限未验证

- **位置**: `game.handler.ts` handleJoinRoom
- **问题**: `handleJoinRoom` 仅检查房间密码，未检查 `isClubOnly` 用户的 clubId 权限
- **风险**: 非俱乐部成员可能收到 "wrong_password" 之外的其他拒绝原因（逻辑不一致）
