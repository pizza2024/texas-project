### P0-DEPOSIT-E2E-001 — `deposit.e2e.spec.ts` E2E mock 缺失 `findFirst`（第2轮）

**状态:** 🔴 未修复（与上轮相同）

**问题:** `getOrCreateDepositAddress` 改用 `findFirst({ where: { userId, isDefault: true }})`，但 E2E mock 仅覆盖 `findUnique`，11个测试全部失败。

**根因:** `deposit.service.ts:63` 改用 `findFirst`，但 `deposit.e2e.spec.ts` 的 `mockPrisma.depositAddress` 只 mock 了 `findUnique`/`findMany`/`create`/`createMany`/`aggregate`，未 mock `findFirst`。

**修复方案:**
```typescript
// deposit.e2e.spec.ts — mockPrisma 初始化时添加：
mockPrisma.depositAddress.findFirst = jest.fn();
mockPrisma.depositAddress.findFirst.mockResolvedValue(null);
```
在所有 `findUnique.mockResolvedValue(...)` 处同步添加 `findFirst.mockResolvedValue(...)`。

**影响:** `npm run test` exit code 1，CI pipeline 阻塞。

**等待修复:** commit `a9b2b86` 的 P2-DEPOSIT-UX-002 功能正确，但测试覆盖未同步。

## P2 新发现问题

### P2-DEPOSIT-UX-004 — `SavedAddress.network` 后端从不返回

**状态:** 🔴 新发现（延续上轮 P2-DEPOSIT-UX-002 相关）

**前端定义** (`apps/web/app/deposit/page.tsx`):
```typescript
interface SavedAddress {
  id: string;
  label: string;
  address: string;
  network: string;   // ← 定义了
  createdAt: string;
}
```

**后端 `getSavedAddresses` 返回** (`deposit.service.ts:107`):
```typescript
return this.prisma.depositAddress.findMany({ where: { userId } });
// schema.prisma DepositAddress 模型无 network 字段 → 前端 addr.network 始终为 undefined
```

**Prisma schema:** `DepositAddress { id, userId, address, label?, isDefault, index, createdAt }` — 无 `network` 字段。

**影响:** 地址簿 UI 的 `addr.network` 显示为空白。

**修复方案:** 二选一
1. Schema 加 `network String?` 字段（充值地址来自同一链，网络固定）
2. 前端移除 `network` 字段（当前 TRON 网络，可写死显示 "TRON"）

### P2-DEPOSIT-UX-005 — `handleSaveAddress` / `handleDeleteAddress` 错误静默吞掉

**状态:** 🔴 新发现

**文件:** `apps/web/app/deposit/page.tsx`

```typescript
const handleSaveAddress = async () => {
  try {
    const res = await api.post<SavedAddress>("/deposit/addresses", { ... });
    setSavedAddresses((prev) => [res.data, ...prev]);
  } catch {
    // silently fail  ← 用户无任何反馈
  }
};

const handleDeleteAddress = async (id: string) => {
  try {
    await api.delete(`/deposit/addresses/${id}`);
    setSavedAddresses((prev) => prev.filter((a) => a.id !== id));
  } catch {
    // silently fail  ← 用户无任何反馈
  }
};
```

**影响:** 保存/删除失败时用户完全不知道操作失败，体验差。

**建议:** 显示 inline error message 或 toast。
