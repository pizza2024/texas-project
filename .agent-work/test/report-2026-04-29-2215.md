### B-INFO-003 安全审查

**问题:** `POST /auth/register-with-email` 无 rate limit，攻击者可批量注册账号。

**修复验证:**

```ts
// auth.controller.ts — 新增
@ApplyRateLimit(5次/300秒, emailOrIp)
@Post('register-with-email')
async registerWithEmail(...) { ... }
```

**结论:** ✅ 修复正确。5次/300秒（每5分钟5次）限制合理；支持按 email 或 IP 双重维度计数；与 `verifyEmailCode` 的 10次/300秒 策略保持一致。
