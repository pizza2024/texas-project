### 新发现问题

#### P2-ROOM-RETRY (待认领)

- **文件:** `apps/web/app/deposit/page.tsx:106`
- **问题:** 充值/操作重试逻辑无指数退避 (exponential backoff)
- **现状:** 使用固定间隔重试，可能导致大量无效请求
- **建议:** 实现指数退避重试 (e.g., 1s → 2s → 4s → max 30s)
- **优先级:** P2

#### P2-CODE-PATTERN (待认领)

- **文件:** `mission.service.ts` / `table-manager.service.ts`
- **问题:** `Promise.all` 在 mission/table-engine 中的使用模式可优化
- **现状:** 部分场景可改用 `Promise.allSettled` 避免单个 rejection 导致整体失败
- **建议:** 检查 `Promise.all` 调用点，确保错误处理完备
- **优先级:** P2

#### P2-WS-RATE-UNIT (待认领)

- **文件:** `connection-state.service.ts:107`
- **问题:** 时间单位注释混淆 (ms vs s)
- **现状:** 代码中混用毫秒/秒，可能导致 rate-limit 计算错误
- **建议:** 统一使用毫秒或添加明确注释
- **优先级:** P2
