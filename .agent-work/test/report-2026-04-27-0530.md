## 🔴 P0 — 新发现

### P0-E2E-002 — E2E moduleNameMapper 路径错误

**问题:** `jest-e2e.json` 中 `moduleNameMapper` 路径使用 `<rootDir>/../../packages/shared` 解析为 `/Users/pizza/workspace/texas-project/apps/packages/shared/src/validation.ts`（错误路径）。

**根因:** `<rootDir>` 在 `jest-e2e.json` 中指向 `apps/backend/test`，两级 `../` 之后变成 `apps/` 而非项目根。

**影响:** E2E 测试套件（app.e2e-spec.ts + rakeback.e2e-spec.ts）完全无法启动，CI/CD 发布验收阻断。

**受影响的映射:**

```
"^@texas/shared/validation$"     → 错误路径 (多了一层 apps/)
"^@texas/shared$"               → 同样错误
"^@texas/shared/types/tournament$" → 同样错误
```

**修复方案:** 改为 `<rootDir>/../../../packages/shared/src/validation.ts`（3层 `../`），或在 workspace 根添加 `ts-jest` 配置共用地。

**状态:** 🟡 待认领
