## 本轮 CodeReview 扫描

### 🔴 新增 P0 问题

#### 1. `deposit.e2e.spec.ts` — 11个 E2E 测试因 `findFirst is not a function` 全部失败

**文件:** `apps/backend/src/deposit/deposit.e2e.spec.ts`

**根因:** `getOrCreateDepositAddress` 改为 `findFirst({ where: { userId, isDefault: true }})`，但 E2E 测试 mock 仅覆盖 `findUnique`，未 mock `findFirst`。

**失败测试列表（`deposit.e2e.spec.ts`）:**
- `returns correct address when already cached`
- `creates new address when none exists`
- `creates new address with index 1 for first-time user`
- `uses nextIndex from aggregate when addresses exist`
- `handles concurrent requests correctly`
- `returns address from race winner`
- `returns correct address when another request inserted first (skipDuplicates)`
- ...（共11个）

**影响:** 测试套件返回 `npm run test` 失败 exit code 1，CI pipeline 阻塞。

**修复方案:**
```typescript
// deposit.e2e.spec.ts — mockPrisma 初始化时添加：
mockPrisma.depositAddress.findFirst = jest.fn();
mockPrisma.depositAddress.findFirst.mockResolvedValue(null);
```
所有使用 `findUnique` mock 的测试需要同步添加 `findFirst` mock（或在 `mockReset()` 后统一设置）。

**优先级:** P0（测试套件完全失败，CI 阻塞）

### ⚠️ 新增 P2 问题

#### 2. `SavedAddress.network` 字段 — 前端有定义但后端从不返回

**前端定义** (`apps/web/app/deposit/page.tsx`):
```typescript
interface SavedAddress {
  id: string;
  label: string;
  address: string;
  network: string;   // ← 定义了
  createdAt: string;
}
```

**后端 `getSavedAddresses` 返回** (`deposit.service.ts`):
```typescript
return this.prisma.depositAddress.findMany({ where: { userId } });
// Prisma DepositAddress 模型无 network 字段 → network 始终为 undefined
```

**影响:** 前端 UI 的 `addr.network` 显示为空白或 `undefined`。

**优先级:** P2（UI 显示异常）

#### 3. `handleSaveAddress` / `handleDeleteAddress` — 所有错误静默吞掉

**文件:** `apps/web/app/deposit/page.tsx`

```typescript
const handleSaveAddress = async () => {
  try {
    const res = await api.post<SavedAddress>("/deposit/addresses", { ... });
    setSavedAddresses((prev) => [res.data, ...prev]);
  } catch {
    // silently fail  ← 用户无任何反馈
  }
};

const handleDeleteAddress = async (id: string) => {
  try {
    await api.delete(`/deposit/addresses/${id}`);
    setSavedAddresses((prev) => prev.filter((a) => a.id !== id));
  } catch {
    // silently fail  ← 用户无任何反馈
  }
};
```

**影响:** 保存/删除失败时用户完全不知道操作失败了。

**建议:** 至少显示一个 toast 或 inline error message。

**优先级:** P2（用户体验差，但功能可工作）
