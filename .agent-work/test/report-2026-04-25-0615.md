## P0 — 未解决（本轮新发现）

> ⚠️ **Critical — 安全回归**

| ID | 问题 | 文件 | 严重性 |
|----|------|------|--------|
| P0-BRUTE-001 | `game.handler.ts:99` 调用 `checkPasswordAttemptLimit` **未 await** | game.handler.ts:99 | 🔴 Critical |

**详情：**

`app.gateway.ts:122` 的 `checkPasswordAttemptLimit` 是 `async` 方法：
```ts
async checkPasswordAttemptLimit(ip: string, roomId: string) {
    return this.connectionState.checkPasswordAttemptLimit(ip, roomId);
}
```

`connection-state.service.ts:60` 的同名方法也是 `async`（返回 `Promise<'banned' | 'rate_limited' | 'ok'>`）。

但 `game.handler.ts:99` 调用时**没有 await**：
```ts
const bruteForceResult = gateway.checkPasswordAttemptLimit(
    ipHash,
    roomId,
);
// ↑ 返回 Promise，永远不是字符串 'banned'
if (bruteForceResult === 'banned') {  // ← 永远为 false
```

**影响：** 密码暴力破解保护**完全失效**。攻击者可以无限制尝试房间密码，不会被封禁。

**task-queue.md 标注"待修复"但代码未实际修复**，说明要么是 Coding 代理误标，要么修复后被 revert。

**修复方案：**
```ts
const bruteForceResult = await gateway.checkPasswordAttemptLimit(
    ipHash,
    roomId,
);
```
