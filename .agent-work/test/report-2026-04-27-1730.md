## CodeReview — 专项深度验证

### P1-WALLET-001 — `exchangeBalanceToChips` 非原子（资金安全）

**状态: ❌ 未修复 — 仍存在**

`exchangeBalanceToChips` (wallet.service.ts:338-385) 仍为 3 次独立 DB 调用，无 `$transaction` 包裹：

```ts
// 1. wallet upsert — 无事务
await this.prisma.wallet.upsert({ ... });

// 2. user update — 分离调用，若失败则 wallet 已修改但 user 不同步
await this.prisma.user.update({ ... });

// 3. transaction log — 分离调用，若失败则审计日志丢失
await this.prisma.transaction.create({ ... });
```

对比：`exchangeChipsToBalance` (line 237-332) 正确使用 `$transaction(async (tx) => {...})`。

**影响:** 充值场景下若 `user.update` 或 `transaction.create` 在 `wallet.upsert` 后失败，wallet 余额已增加但 user.coinBalance 不同步，导致用户看到错误余额。

**修复建议:** 将 3 个调用移入 `this.prisma.$transaction(async (tx) => {...})`，参考 `exchangeChipsToBalance` 模式。

### P1-WITHDRAW-005 — `createWithdraw` transaction log 位置错误

**状态: ❌ 未修复 — 仍存在**

`createWithdraw` 中 `transaction.create` (WITHDRAW 审计日志) 在 `$transaction` 块**外部**：

```ts
const request = await this.prisma.$transaction(async (tx) => {
    // wallet upsert + user update + withdrawRequest.create — 原子内
    return tx.withdrawRequest.create({ ... });
});
// ★ 原子块关闭后
await this.prisma.transaction.create({   // ← 原子块外，崩溃可丢失
    data: { userId, amount: -dto.amountChips, type: 'WITHDRAW' }
});
```

**影响:** 若进程在 `$transaction` 提交后、审计日志写入前崩溃，WITHDRAW 流水丢失，无法对账。

**修复建议:** 将 `transaction.create` 移入 `$transaction` 回调内。

### P1-WITHDRAW-007 — `executeChainWithdraw` DB+链上非原子

// 1. 不可逆链上转账
tx = await contract.transfer(request.toAddress, amountInUnits);

if (payload.sessionId && this.redisService.isAvailable) {
