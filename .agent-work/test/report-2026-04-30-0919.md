### ⚠️ P2-NOTIFY-WS-BRIDGE — WebSocket 多实例广播风险

**严重程度：** P2（中）
**文件：** `apps/backend/src/websocket/app.gateway.ts`

**问题描述：**

当前 `notifyFriendsOfStatusChange` 中的在线通知通过 `this.wsManager.emitToUser()` 推送：

```typescript
this.wsManager.emitToUser(friend.friendId, 'notification', notif)
```

而 `wsManager.emitToUser` 的实现为：

```typescript
emitToUser<T = unknown>(userId: string, event: string, data: T): void {
  this.server.to(`user:${userId}`).emit(event, data);
}
```

**风险：**

NestJS 应用运行多实例（多进程 / 多 Pod）时，WebSocket 推送只覆盖**当前实例**连接的客户端。如果用户通过负载均衡连接到另一个实例，该用户**不会**收到 `notification` 事件。

**触发条件：**
- 生产环境使用 PM2 集群模式或 Kubernetes 多副本部署
- 用户当前 WS 连接路由到 instance-2，但通知从 instance-1 发出

**临时缓解：** 项目当前可能为单实例部署，风险暂不触发

**修复方案（P2-NOTIFY-WS-BRIDGE）：**
- 方案 A（推荐）：在 Redis Pub/Sub 基础上增加 `NotificationGateway` 的集群广播，参考现有的 `BroadcastService` 模式
- 方案 B：在 `NotificationService.create()` 内部直接调用 `emitToUser`，确保通知触发和推送都在同一模块内
- 方案 C：使用 Socket.io Adapter（`@socket.io/redis-adapter`）实现跨实例广播
