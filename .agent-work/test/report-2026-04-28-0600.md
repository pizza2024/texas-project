### 🔴 P1 遗留确认（待认领）

#### P1-NEW-003: `setBalances` Phase 2 非原子

**位置:** `wallet.service.ts:72–136`

```typescript
// Phase 1 — 原子（已提交）
await this.prisma.$transaction(entries.map(...));  // ✅

// Phase 2 — 非原子
const phase2Results = await Promise.allSettled(
  entries.map(({ userId, balance }) =>
    this.prisma.user.update({ where: { id: userId }, data: { coinBalance: normalized } })
  )
);
```

**风险：**

1. Phase 1 成功后 Phase 2 部分失败 → `wallet.chips` 已更新但 `user.coinBalance` 不一致
2. Phase 2 中每个 `user.update` 独立 auto-commit，崩溃后无法回滚 Phase 1
3. Phase 1 失败导致 Phase 2 从未执行，`wallet` 记录缺失但调用方无感知

**建议：** 将 Phase 1 + Phase 2 合并为单个 `$transaction`

#### P1-NEW-004: `exchangeChipsToBalance` TOCTOU — **确认 fund loss 漏洞**

**位置:** `wallet.service.ts:275–305`

```typescript
// Line 275 — 事务外读取
const availableChips = await this.getAvailableBalance(userId);  // T1 读取 1000
if (chipsAmount > availableChips) { ... }                       // T1 通过

// Line 285 — 事务内再次读取
const wallet = await tx.wallet.findUnique({ where: { userId } });
const currentBalance = wallet?.chips ?? ...;                    // T1 读取 1000
```

**攻击路径：并发双花**

1. T1: `getAvailableBalance` = 1000，通过检查
2. T2: `getAvailableBalance` = 1000，通过检查
3. T1: 事务内扣减至 0，提现 1000 成功
4. T2: 事务内扣减至 0，提现 1000 成功
5. **结果：** 2 笔 USDT 提现，DB 仅扣 1 笔 chips → **fund loss**

**修复：** 余额检查应移至 `$transaction` 内部

#### P2-NEW-016: `unfreezeBalance` TOCTOU

**位置:** `wallet.service.ts:192–207`

```typescript
const wallet = await this.prisma.wallet.findUnique({ where: { userId } }); // 事务外
const frozenAmount = wallet?.frozenChips ?? 0; // 冻结值在事务外捕获
await this.prisma.$transaction([
```
