## E2E 测试失败（P0 级阻断）

| ID | 问题 | 文件 | 详情 |
|----|------|------|------|
| **P0-E2E-001** | E2E 测试套件无法启动 — 模块解析失败 | tournament.service.ts | `Cannot find module '@texas/shared/types/tournament'` |

### 问题分析

```
Cannot find module '@texas/shared/types/tournament' from '../src/tournament/tournament.service.ts'
Require stack:
  tournament.service.ts → app.gateway.ts → websocket.module.ts → app.module.ts → app.e2e-spec.ts

at Resolver._throwModNotFoundError
```

**根因：** `packages/shared/package.json` exports 字段未包含 `./types/tournament` 路径解析，但 `apps/backend/tsconfig.json` 的 `paths` 映射 ` "@texas/shared/*": ["../../packages/shared/src/*"]` 理论上应能解析。

**注意：** 单元测试（31 suites / 410 tests）全部通过，说明运行时 module 加载正常，仅 jest-e2e 的 ts-jest 转换链在此路径解析上失败。这可能与 jest ts-jest 的模块解析缓存或 exports 字段冲突有关。

### 建议

1. 在 `packages/shared/package.json` 的 exports 字段添加显式 tournament 导出：
   ```json
   "./types/tournament": "./src/types/tournament.ts"
   ```
2. 或在 jest-e2e.json 的 `moduleNameMapper` 中显式映射
3. 验证 `npm run test:e2e` 在 CI 环境中是否同样失败（如果 CI 通过而本地失败，则可能是 node_modules 缓存问题）
