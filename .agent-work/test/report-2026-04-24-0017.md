## 🆕 新发现

### T-003: 洗牌算法使用 Math.random()（P2）

**文件**: `apps/backend/src/table-engine/table-game-logic.ts:30`

```ts
const j = Math.floor(Math.random() * (i + 1));
```

**分析**: Fisher-Yates  shuffle 使用 `Math.random()`，非密码学安全随机数。

**风险评估**:
- 扑克游戏洗牌不需要密码学级别的随机性（参考：多数在线扑克平台使用 Mersenne Twister 或类似）
- `Math.random()` 可预测性在服务端上下文中风险有限（客户端无法观察内部状态）
- 但若攻击者能注入 seed 或观察早期牌序，仍存在理论风险

**建议**: 考虑替换为 `crypto.getRandomValues()`，但非 P0 紧急

**Bot 同样使用 Math.random()**: `bot.service.ts:136,159,183,194,205,217` — 影响 Bot 对战公平性质疑（低风险）
