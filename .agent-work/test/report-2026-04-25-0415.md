### P2 问题

#### P2-NEW-002: `creditRakeback` 使用 `$transaction([op])` 数组语法 ⚠️ 低优先级

**文件**: `apps/backend/src/rakeback/rakeback.service.ts:160`

```ts
// 当前实现（数组语法）
await this.prisma.$transaction([
  this.prisma.user.update({
    where: { id: userId },
    data: { rakebackBalance: { increment: rakebackAmount } },
  }),
]);
```

**问题**: Prisma 的 `$transaction([...])` 语法用于**批量操作**（parallel/batched），各操作在事务中执行但彼此隔离独立。与 `$transaction(async (tx) => ...)` 的**交互式事务**（共享同一 `tx` 客户端）语义不同。

**对比**: `claimRakeback` 使用了正确的 callback 语法：

```ts
const result = await this.prisma.$transaction(async (tx) => {
  const wallet = await tx.wallet.update({...});
  await tx.user.update({...});
  return {...};
});
```

**风险**: 当前功能正常（单操作在事务中执行），但模式不一致，未来若需要多操作原子性时可能出错。

**建议**: 统一为 callback 语法：

```ts
await this.prisma.$transaction(async (tx) => {
  await tx.user.update({
    where: { id: userId },
    data: { rakebackBalance: { increment: rakebackAmount } },
  });
});
```

**优先级**: P2（代码一致性，无实际 bug）
