## CodeReview 发现

### W-001 — Withdraw 分布式 cooldown 绕过 (P0)

**文件**: `apps/backend/src/withdraw/withdraw.service.ts` 行 51, 114-143

**问题确认**: 
- 行 51: `private readonly cooldowns = new Map<string, number>()` 内存 Map 作为 Redis 降级
- 行 114-117: Redis 不可用时仅记录 WARN 日志，降级到内存 Map
- 行 142: `this.cooldowns.set(userId, Date.now())` 仅更新本进程内存
- **多实例部署时**: 不同实例的 Map 不同步，可绕过 60 秒冷却

**代码证据**:
```
// 行 114-117
this.logger.warn(
  `[SECURITY] Withdraw cooldown using in-memory Map — not safe for multi-instance deployment`,
);
```

**建议修复**: 已在任务队列，建议 Coding 代理优先处理

### W-002 — JWT Redis 降级安全日志缺失 (P0)

**文件**: `apps/backend/src/auth/jwt.strategy.ts` 行 22-31

**问题确认**:
- 行 26-31: Redis 不可用时跳过 session 验证，记录 `[SECURITY-AUTH-BYPASS]` 前缀 WARN 日志
- 注释中有 TODO: `// TODO(@security): set a metrics counter here`
- 缺少: 监控/告警机制、metrics counter、安全事件告警

**代码证据**:
```typescript
this.logger.warn(
  `[SECURITY-AUTH-BYPASS] Redis unavailable — session validation SKIPPED for user ${payload.sub}. ` +
    `Single-device login protection is temporarily disabled. ` +
    `Timestamp: ${new Date().toISOString()}. ` +
    `Source: ${payload.username ?? 'unknown'}`,
);
```

**建议**: 需接入 metrics 系统（如 Prometheus）并配置安全告警

### W-003 — 房间密码暴力破解防护 (P1→P2)

**文件**: `apps/backend/src/websocket/game.handler.ts` 行 100-108

**问题确认**:
- `wrong_password` 事件仅返回给客户端，无日志记录
- 无尝次计数（passwordAttempts）
- 无临时封禁机制
- 攻击者可无限尝试密码

**代码证据**:
```typescript
if (!passwordMatch) {
  client.emit('wrong_password', { roomId });
  return { event: 'wrong_password', data: { roomId } };
}
```

**建议**: 添加 IP/用户维度的尝次计数，超限后临时封禁
