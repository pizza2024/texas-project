### 🔴 P2 — 新发现 / 未修复

#### P2-NEW-001: `checkAddressDeposits` 非原子读-改-写竞态条件
**文件:** `apps/backend/src/deposit/deposit.service.ts:362-369`
```ts
const currentBalance = await this.walletService.getBalance(userId);  // READ
const newBalance = new BigNumber(currentBalance).plus(chips);       // MODIFY
await this.walletService.setBalance(userId, newBalance.toNumber());  // WRITE
```
**风险:** 同一地址并发收到两笔充值，第二个请求可能覆盖第一个的余额更新（非幂等），存在资金损失风险。
**建议:** 包裹在 `$transaction` 内，或使用 `wallet.increment('chips', chips)` 原子操作。
**优先级:** P2（需 Coding 实施）

#### P2-NEW-002: `JSON.parse` OTP 响应无 try/catch
**文件:** `apps/backend/src/auth/auth.service.ts:200`
```ts
const { code: storedCode, expiresAt } = JSON.parse(stored);
```
**风险:** 如果 Redis 数据损坏（非有效 JSON），会导致未捕获异常。Redis 数据通常可信，但防御性编程应捕获。
**建议:** 包裹在 `try/catch` 内，无效数据按"验证码无效"处理。
**优先级:** P2

#### P2-NEW-003: JWT Token 存储于 localStorage（XSS 攻击目标）
**文件:** `apps/web/lib/auth.ts:16`, `apps/web/lib/auth-context.tsx:50`, `apps/web/lib/api.ts:13`
```ts
localStorage.setItem('token', token); // XSS 可直接读取
```
**风险:** 任何 XSS 漏洞可立即窃取用户 JWT。localStorage 不支持 httpOnly，无法防御。
**建议:** 迁移至 httpOnly Secure Cookie（P2-JWT-LOCALSTORAGE 遗留问题）。
**优先级:** P2（安全相关，建议升 P1）

#### P2-NEW-004: `getOrCreateDepositAddress` 索引分配存在竞态窗口
**文件:** `apps/backend/src/deposit/deposit.service.ts:62-77`
```ts
const aggregate = await this.prisma.depositAddress.aggregate({ _max: { index: true } });
const nextIndex = Math.max(1, (aggregate._max.index ?? 0) + 1);
// ↑ 并发请求可能分配相同 index
```
**缓解:** 有 P2002 catch 处理重复创建，但会导致一个请求失败并重查。
**建议:** 对 `index` 字段加唯一约束 + `SELECT FOR UPDATE` 避免竞态。
**优先级:** P2
