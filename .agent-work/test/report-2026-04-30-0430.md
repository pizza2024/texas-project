### ⚠️ 遗留未修复问题

#### 1. `client.leave()` 在 `leaveCurrentRoom` 之前 — 状态漂移窗口仍存在

**文件:** `apps/backend/src/websocket/game.handler.ts:389`

```typescript
return gateway.withRoomLock(roomId, async () => {
  // Leave the Socket.io room FIRST to avoid race conditions with DB state
  client.leave(roomId);                        // ← 在 DB 更新之前
  const result = await gateway.tableManager.leaveCurrentRoom(userId);  // ← DB 之后
```

**问题:** 注释说 "FIRST to avoid race conditions"，但实际上 `client.leave()` 在锁内、DB 操作之前是正确的（锁内无竞态），但注释与实际意图不符。真正的问题是在 `client.leave()` 和 `leaveCurrentRoom()` 之间如有异常，socket 已离开 room 但 DB 未更新。

**建议:** 添加 try/finally 确保一致性，或将 `client.leave()` 移至 `leaveCurrentRoom` 成功之后。

**优先级:** P2（race window 极小，需特定故障场景才触发）

#### 2. `BOT_ID_PREFIX` 前缀碰撞 — 高风险未关闭

**文件:** `apps/backend/src/wallet/wallet.service.ts:95`

```typescript
if (userId.startsWith(BOT_ID_PREFIX)) {
  return [walletOp]; // Skips coinBalance sync for "bots"
}
```

**问题:** 用户 ID 以 `"bot_"` 开头的真实用户，其 `User.coinBalance` 永远不会被 `setBalances` 同步，导致 `getAvailableBalance` 返回错误的可用余额。

**建议:** 使用 `User.role === 'BOT'` 判断，而非字符串前缀匹配。

**优先级:** P1（用户资金正确性受损，无 P0/P1 当前项，可安排）
