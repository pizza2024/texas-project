## 新发现：E2E 测试配置问题（持续）

### 问题

`test/jest-e2e.json` 缺少 `setupFiles` 加载 `.env.test`，导致 `JWT_SECRET` 未注入，E2E 测试在启动时即失败：

```
Error: JWT_SECRET is not defined
at getJwtSecret (../src/config/jwt.config.ts:9:11)
```

### 根因

`jest-e2e.json` 配置：
```json
{
  "moduleFileExtensions": ["js", "json", "ts"],
  "rootDir": ".",
  "testEnvironment": "node",
  "testRegex": ".e2e-spec.ts$",
  "transform": {
    "^.+\\.(t|j)s$": "ts-jest"
  }
}
```

缺少 `setupFiles` 字段来加载 `.env.test`。

### 修复方案

在 `jest-e2e.json` 中添加：

```json
{
  "setupFiles": ["<rootDir>/test/setup-e2e.ts"],
  "globalSetup": "<rootDir>/test/global-e2e-setup.ts"
}
```

或更简单地在 `jest-e2e.json` 添加：

```json
{
  "setupFiles": ["./test/load-env.ts"]
}
```

其中 `test/load-env.ts` 内容：
```ts
import { config } from 'dotenv';
config({ path: '.env.test' });
```

- 阻塞 P1-RAKEBACK-001（rakeback E2E 验证）
