## 代码审查 — P2-WITHDRAW-UX-001（新实现）

### Backend (`apps/backend/src/withdraw/`)

#### Prisma Schema — `WithdrawAddress` 模型

```prisma
model WithdrawAddress {
  id        String   @id @default(uuid())
  userId    String
  address   String
  label     String?
  isDefault Boolean  @default(false)
  createdAt DateTime @default(now())
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  @@unique([userId, address])
  @@map("withdraw_addresses")
}
```

| 检查项 | 结果 | 说明 |
|--------|------|------|
| `onDelete: Cascade` | ✅ | 用户删除时自动清理地址簿 |
| `@@unique([userId, address])` | ✅ | 防止同一用户重复保存同一地址 |
| `isDefault` 默认 `false` | ✅ | 第一条地址自动设为默认（service 层处理） |
| `userId` 上有索引 | ✅ | Prisma 默认为主键和外键添加索引 |

**⚠️ 注意:** `prisma migrate` 尚未运行。数据库需执行迁移才能创建 `withdraw_addresses` 表。发布前必须运行迁移。

#### `WithdrawService` — 地址簿方法

| 方法 | 逻辑 | 安全检查 |
|------|------|---------|
| `getSavedAddresses` | 按 `isDefault desc, createdAt desc` 排序 | ✅ 仅返回同用户地址 |
| `saveAddress` | 重复地址更新 label；首个自动设默认 | ✅ `userId_address` unique 约束 |
| `deleteAddress` | 删除默认地址时自动升格下一条 | ✅ ownership 检查 |
| `setDefaultAddress` | 同一用户只能有一个默认 | ✅ `userId_address` unique + transaction |

**`setDefaultAddress` 原子性:**
```typescript
await this.prisma.$transaction([
  this.prisma.withdrawAddress.updateMany({
    where: { userId, isDefault: true },
    data: { isDefault: false },
  }),
  this.prisma.withdrawAddress.update({
    where: { id: addressId },
    data: { isDefault: true },
  }),
]);
```
✅ 事务正确，防止 race condition。

**`deleteAddress` 默认地址升格逻辑:**
```typescript
if (wasDefault) {
  const next = await this.prisma.withdrawAddress.findFirst({
    where: { userId },
    orderBy: { createdAt: 'asc' },
  });
  if (next) {
    await this.prisma.withdrawAddress.update({
      where: { id: next.id },
      data: { isDefault: true },
    });
  }
}
```
✅ 逻辑正确，但**非原子**（查询+更新分离）。低风险因为单用户操作，暂不视为问题。
