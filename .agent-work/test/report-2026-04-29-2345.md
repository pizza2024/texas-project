### P0-TS-001 详情

**问题:** HEAD commit `f61746c` ("fix(P2): move ACTIVE_BETTING_STAGES to module level") 引入的 TypeScript 编译错误。

```ts
// 当前代码 (line 35):
const ACTIVE_BETTING_STAGES = ['PREFLOP', 'FLOP', 'TURN', 'RIVER'] as const;

// Line 95 和 753:
ACTIVE_BETTING_STAGES.includes(table.currentStage) && ...
//                    ^^^^^^^^^^^^^^^^^^^ TS2345: string 不能赋值给 "PREFLOP" | "FLOP" | "TURN" | "RIVER"
```

**根因:** `TableState.currentStage` 定义为 `string`（`packages/shared/src/types/game.ts:95`），而 `ACTIVE_BETTING_STAGES` 是 `readonly ["PREFLOP", "FLOP", "TURN", "RIVER"]` 元组类型。`Array.includes()` 收窄了参数类型，导致类型不匹配。

**修复方案（二选一）:**

1. `const includesStage = (s: string): boolean => (['PREFLOP','FLOP','TURN','RIVER'] as const).includes(s as any);`
2. 将 `TableState.currentStage` 类型改为 `currentStage: 'PREFLOP' | 'FLOP' | 'TURN' | 'RIVER' | 'WAITING' | 'SETTLEMENT' | 'DEALING'`（推荐，统一类型）

**影响:** `npx tsc --noEmit -p apps/web/tsconfig.json` 编译失败，阻塞 Web 构建。
