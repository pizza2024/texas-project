#### P0-TYPE-001 — Rakeback 利率单位错误（金融显示 Bug）

**文件:** `packages/shared/src/types/rakeback.ts:20-24` vs `apps/backend/src/rakeback/rakeback.service.ts:6-10`

前端定义 tier 利率为百分比整数：
```ts
// rakeback.ts
rate: 15 | 22 | 30 | 40 | 50  // 解释为 15%=15
```

后端存储为小数：
```ts
// rakeback.service.ts
rate: 0.15 | 0.22 | 0.3 | 0.4 | 0.5  // 解释为 15%=0.15
```

**影响:** 前端 UI 显示 Rakeback 利率时，数值会是实际的 100 倍（如 DIAMOND 显示 50% 而非 0.5%）。涉及真实的用户金融数据展示。

**文件:** `apps/backend/src/wallet/wallet.service.ts:355-371`

```ts

await this.prisma.user.update({ ... });          // 失败 → wallet 更新了，user 没更新
```
