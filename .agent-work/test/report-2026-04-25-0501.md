## P0 验证结果（复审）

### P0-GAME-002 修正 ⚠️

**文件:** `table-round.ts:289-316`

**上轮描述有误，重新核实：**

```typescript
// table-round.ts:295-301
const rake = Math.min(
  Math.floor(this.table.pot * tierConfig.rate),  // rake = floor(pot * rate)
  tierConfig.cap,
);
const winAmount = this.table.pot - rake;  // ✓ winAmount 已扣 rake
winner.stack += winAmount;               // ✓ winner 拿到的已扣 rake
this.table.pot = 0;                      // ⚠️ 但 this.table.pot 未扣 rake
```

**实际确认的 bug：** `this.table.pot -= rake` 从未执行。rake 虽然从 `winAmount` 中扣除了（winner 拿 `pot - rake`），但 `this.table.pot` 本身未被扣减。如果后续有任何代码读取 `this.table.pot`（如审计日志、历史记录），会看到完整 pot 而非扣税后金额。但从资金流角度 winner 确实少拿了 rake，**P0 仍成立**。

**建议：** 在 `winner.stack += winAmount` 后添加 `this.table.pot -= rake`（使 pot 余额与实际一致），或统一使用 `this.table.pot -= rake` 风格（参考 performShowdown 第174行）。
