## 新发现

### P1 — missions/page.tsx 缺少类型导入

**严重程度:** P1 — 编译失败导致 missions 页面不可用

**问题描述:**
`apps/web/app/missions/page.tsx` 使用了 `MissionStatus` 和 `MissionType` 类型，但这两个类型**从未被导入**：

```
app/missions/page.tsx(22,11): error TS2552: Cannot find name 'MissionStatus'. Did you mean 'PermissionStatus'?
app/missions/page.tsx(36,3): error TS2304: Cannot find name 'MissionType'.
app/missions/page.tsx(375,12): error TS2304: Cannot find name 'UserAvatar'.
```

**根因分析:**
- `MissionStatus` 和 `MissionType` 是**字符串字面量联合类型**（`ACTIVE | COMPLETED | CLAIMED` 和 `ONE_TIME | DAILY | WEEKLY`），在后端 `mission.service.ts` 中以字符串硬编码使用，但**未导出到 `@texas/shared` 包**
- `UserAvatar` 组件存在于 `@/components/user-avatar`，但 `missions/page.tsx` 第375行使用了它而**未 import**
- 对比 `deposit/page.tsx` 和 `rakeback/page.tsx` — 它们正确 import 了 `UserAvatar`

**影响范围:**
- `/missions` 页面**无法通过 TypeScript 编译**
- `npx tsc --noEmit` 返回3个错误

**修复方案:**

1. 添加缺失的 UserAvatar import:
```typescript
import { UserAvatar } from '@/components/user-avatar';
```

2. 在 Mission 接口上方定义缺失的类型（或从 shared 包导出）:
```typescript
type MissionStatus = 'ACTIVE' | 'COMPLETED' | 'CLAIMED' | 'EXPIRED';
type MissionType = 'ONE_TIME' | 'DAILY' | 'WEEKLY';
```

**涉及文件:**
- `apps/web/app/missions/page.tsx`
