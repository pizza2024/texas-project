## 本轮 CodeReview 审查

### ✅ P1-CHAT-INJECTION 重审 — 确认为 False Positive

**文件:** `apps/backend/src/websocket/game.handler.ts` 行 557-559

**之前判定:** username 取自 JWT payload 而非 DB 查询，可能身份冒充。

**重审结论:** 这是 False Positive，理由如下：

1. JWT payload 中的 `username` 来自 `auth.service.ts:81`，在用户**登录时**从 DB 写入并由 JWT 签名保护
2. `handleConnection` (`app.gateway.ts:290`) 调用 `jwtService.verify(token)` 验证签名，验证通过后才设置 `client.data.user = payload`
3. `handleChatMessage` 使用的是经过 JWT 验证的 `client.data.user.sub`（userId）作为身份标识，`username` 仅用于**展示**
4. 攻击者若要冒充他人身份，需同时满足：(a) 盗取受害者 JWT，(b) 该 JWT 的 username 恰好是要冒充的名字 — 实质等同于账号盗用，与 username 字段无关

**结论:** 无需修复，维持 P2 观察即可。
