## P1 — 新发现（2项）

### P1-NEW-001: `buildPots` 使用 stale `allPlayers` 导致 all-in 边池计算错误

**文件:** `apps/backend/src/table-engine/table-round.ts`

**问题:** `buildPots` 方法接收 `allPlayers` 参数（含所有玩家，包括已 FOLD 的），但在确定 `eligiblePlayerIds` 时，它用 `p.status !== PlayerStatus.FOLD` 来过滤。然而，调用方传的是原始 `allPlayers` 数组，而玩家的 `status` 字段在 all-in 后可能已变为 `ALLIN`。更关键的是，`buildPots` 被 `performShowdown` 调用时，`allPlayers` 中可能包含在之前 round 已经 FOLD 的玩家 — 他们的 `totalBet > 0`（贡献到底池），但状态已经是 `FOLD`。若这些玩家的 `totalBet` 产生了独立的边池级别，而该级别所有玩家都 FOLD 了（只有 ALLIN 玩家还在），则会产生错误分配。

**代码片段（当前工作区）:**

```typescript
const eligiblePlayerIds = atThisLevel
  .filter(
    (p) => p.status === PlayerStatus.ACTIVE || p.status === PlayerStatus.ALLIN,
  )
  .map((p) => p.id);

if (eligiblePlayerIds.length > 0) {
  pots.push({ amount, eligiblePlayerIds });
} else if (pots.length > 0) {
  // All contributors at this level folded — add to previous pot
  pots[pots.length - 1].amount += amount; // 错误：可能把其他玩家的有效注码加到前一个 pot
}
```

**风险:** 在有多人 all-in 且有边池的场景下，可能出现边池金额错误分配。

**建议:** 在构建 pot 之前，按当前游戏阶段（`preflop`/`flop`/`turn`/`river`）过滤出实际参与该轮的玩家，而非用 `allPlayers`。

**优先级:** P1
