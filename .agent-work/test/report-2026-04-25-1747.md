## P0 问题（3个 — 均未修复）

### P0-BACKEND-001: `resolveFoldWin()` 缺少 `bestCards`
- **文件:** `table-round.ts:293–321`
- **问题:** Fold 获胜时 `lastHandResult` 条目缺少 `bestCards: []` 字段，客户端手牌复盘渲染时 `bestCards` 为 `undefined`，会导致 ReplayModal 渲染异常
- **修复建议:** 在 `table-round.ts:311–317` 的 `map()` 中添加 `bestCards: []`
- **影响:** 手牌复盘 UI 错误

### P0-BACKEND-002: Straddle 后 `calledAllIn` 未重置
- **文件:** `table-round.ts:attemptStraddle()`
- **问题:** `this.table.calledAllIn` 在 Straddle 操作后未被重置为 `null`，后续玩家无法 raise（因为 `calledAllIn` 非 null 时 raise 逻辑受限）
- **复现场景:** UTG straddle → CO raise → BTN all-in（此时 calledAllIn 仍为前一轮遗留值）
- **修复建议:** 在 `attemptStraddle()` 末尾（`this.table.activePlayerIndex = ...` 之后）添加 `this.table.calledAllIn = null`
- **影响:** Straddle 后 all-in/raise 逻辑异常

### P0-BACKEND-003: `persistSettlementRecords()` catch 块中 `handId` 未定义
- **文件:** `table-manager.service.ts:305–312`
- **问题:** Phase 1 `prisma.hand.create` 失败时 `handId` 未定义，`adminLogSettlementFailure(roomId, handResult, err)` 第四参数 `handId` 传 `undefined`（应为可选参数但函数签名要求 string）
- **风险:** 如果 `adminLogSettlementFailure` 内部访问 `handId.toString()` 会抛出 `TypeError`
- **修复建议:** Phase 1 catch 中初始化 `let handId: string`，并传入 `adminLogSettlementFailure` 前检查
- **影响:** Admin 告警逻辑可能崩溃
