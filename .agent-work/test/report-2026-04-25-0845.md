## 新增发现

### P1-RAKEBACK-001: E2E 测试配置缺失 — `.env.test` 未加载

**问题**: `rakeback.e2e-spec.ts` 运行时报错：

```
FATAL: JWT_SECRET environment variable is not set.
```

**根因**: `test/jest-e2e.json` 缺少 `setupFiles` 或 `dotenv` 配置来加载 `.env.test` 文件。

```json
// test/jest-e2e.json — 当前缺少 dotenv 配置
{
  "moduleFileExtensions": ["js", "json", "ts"],
  "rootDir": ".",
  "testEnvironment": "node",
  "testRegex": ".e2e-spec.ts$",
  "transform": {
    "^.+\\.(t|j)s$": "ts-jest"
  }
  // ❌ 缺少 "setupFiles" 或 "globalSetup" 加载 .env.test
}
```

`.env.test` 文件存在且包含 `JWT_SECRET`：

```
NODE_ENV=test
PORT=4001
DATABASE_URL="postgresql://texas:***@localhost:5432/texas_test"
JWT_SECRET="***"
REDIS_URL="redis://localhost:6379/1"
```

**修复方案**: 在 `jest-e2e.json` 添加 `setupFiles`：

```json
{
  "setupFiles": ["<rootDir>/../test/.env.e2e.ts"]
}
```

或使用 `dotenv` 加载。

**影响**: 所有 E2E 测试（不只是 rakeback）都会因同样问题失败。
