## 本轮处理任务

### ✅ P1-NEW-001 — `activeGames` Map 无 TTL 清理（内存泄漏）

**问题:** `BlastService.activeGames` 是纯 in-memory `Map<tableId, BlastGameRecord>`，无任何 TTL 清理机制。当游戏正常结束时 `cleanupGame()` 会删除条目，但若 `onBlastHandComplete` 未被调用（race condition、异常、节点宕机等），条目会永久留在 Map 中，造成内存泄漏。

**修复 (commit `d951350`):**

1. **`implements OnModuleDestroy`** — NestJS 生命周期钩子，模块销毁时清理 interval
2. **定时 sweep (`setInterval`)** — 每 `BLAST_TOTAL_DURATION_MS`（3 分钟）执行一次 `sweepExpiredGames()`，扫描所有条目并删除已过 `endsAt` 的 stale 记录
3. **`sweepExpiredGames()`** — 防御性清理：遍历 Map，若 `now >= game.endsAt` 则删除并打 warn 日志（可见性），记录被清理数量
4. **`onModuleDestroy()`** — `clearInterval(cleanupInterval)` 防止进程退出后 timer 泄漏

```ts
// 构造时启动定期清理
this.cleanupInterval = setInterval(() => {
  this.sweepExpiredGames();
}, BLAST_TOTAL_DURATION_MS); // 3 分钟扫一次

// 清理逻辑
private sweepExpiredGames(): void {
  const now = Date.now();
  for (const [tableId, game] of this.activeGames) {
    if (now >= game.endsAt) {
      this.activeGames.delete(tableId);
    }
  }
}
```
