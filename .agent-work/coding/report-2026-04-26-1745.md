### P1-WS-001 🟡 待评估（无新提交）

`app.gateway.ts:292-294` — `client.data.user` 在 JWT verify 后立即设置，先于 Redis 会话校验。

**分析:**

```typescript
const payload = this.jwtService.verify(token);
client.data.user = payload; // ← 立即设置

if (payload.sessionId && this.redisService.isAvailable) {
  const stored = await this.redisService.get(`user_session:${payload.sub}`);
  if (stored !== payload.sessionId) {
    client.data.user = null; // ← 校验失败后清空
    client.emit("force_logout", { reason: "SESSION_REPLACED" });
    client.disconnect();
    return;
  }
}
```

**风险:** 如果 Redis 校验在设置 `client.data.user` 之后、`client.disconnect()` 之前，有极短时间窗口（毫秒级），其他事件处理器可能读取到未校验的用户。

**实际风险:** 低。Redis 校验本身是毫秒级操作，且 `client.data.user` 在校验失败后立即被清空。真正的问题在于是否有其他 `beforeJoin` 或类似 hook 依赖这个未校验的数据。需 Game 代理确认。

**状态:** 🟡 标记为低风险，观察中。
