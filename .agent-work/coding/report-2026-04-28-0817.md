## 本轮任务

### P2-NEW-012 — `resolveFoldWin` 绕过 side-pot ✅

**问题根因:**
`resolveFoldWin()` 使用 `this.table.pot` 单一总值分配给赢家，而非使用 `buildPots()` 构建的边池结构。当出现 all-in 玩家产生 side-pot 时，fold-win 路径会错误地把所有池（含其他玩家无权获得的 side-pot）全部分配给获胜者。

**修复内容** (`table-round.ts:295`):

1. 调用 `buildPots(allPlayers)` 构建 main + side pot 结构（与 `performShowdown` 一致）
2. 遍历每个 pot，只向 `eligiblePlayerIds` 包含的非fold玩家分配
3. 通过 `winAmounts` Map 累加每个 pot 的份额（处理多人情况下的余数分配）
4. 统一在循环结束后 `table.pot = 0`
5. 同步修复：winner.stack 也不再直接修改，而通过 Map 统一结算
