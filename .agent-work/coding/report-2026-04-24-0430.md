### Pre-existing 测试问题根因

所有 5 个失败测试都涉及 `jest.useFakeTimers()` + `jest.advanceTimersByTimeAsync` 场景：

**问题 A：`scheduleDisconnectCleanup` mock 不执行 callback**
- `scheduleDisconnectCleanup` 的 mock 是 `jest.fn()`（空函数）
- 真实实现：存储 `onTimeout` callback，在 timer 到期时执行
- mock 不存储/不执行 callback，导致 `jest.runOnlyPendingTimersAsync()` 后 `leaveCurrentRoom` 未被调用

**问题 B：所有 timer 相关的 `finalizeXxx` 方法需要被调用才能触发后续逻辑**
- `finalizeActionTimeout` / `finalizeReadyCountdown` / `finalizeSettlement` 的 mock 是 `mockResolvedValue(undefined)`
- 调用链：`scheduleXxx` → setTimeout → `finalizeXxx`（需要 mock 触发）
- 测试需要重新设计 mock 来模拟 timer 触发后的 callback 执行
