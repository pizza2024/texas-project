### 2. `table-state.ts` — Tier-based Rake 配置

```typescript
export const TIER_RAKE_CONFIG: Record<string, { rate: number; cap: number }> = {
  MICRO:   { rate: 0.05, cap: 0.30 },
  LOW:     { rate: 0.04, cap: 0.50 },
  MEDIUM:  { rate: 0.035, cap: 1.00 },
  HIGH:    { rate: 0.03, cap: 2.00 },
  PREMIUM: { rate: 0.025, cap: 3.00 },
};
```

新增 `tier`（房间级别）和 `rakeAmount`/`rakePercent`（本局抽水）字段到 `TableSnapshot` 接口。

### 3. `table.ts` — Table 类新增字段

- `tier: string` — 房间级别（默认 `'LOW'`）
- `rakeAmount: number` — 本局已扣除抽水
- `rakePercent: number` — 本局抽水比例

在 `toSnapshot()` / `fromSnapshot()` 中正确序列化/反序列化。

### 4. `table-round.ts` — Tier-based Rake 计算

**`performShowdown()`**:
```typescript
const tierConfig = TIER_RAKE_CONFIG[this.table.tier] ?? { rate: RAKE_RATE, cap: RAKE_CAP };
const rake = Math.min(Math.floor(this.table.pot * tierConfig.rate), tierConfig.cap);
this.table.pot -= rake;
this.table.rakeAmount = rake;
this.table.rakePercent = tierConfig.rate;
```

**`resolveFoldWin()`**（弃牌赢）: 同样使用 tier-based rake。

同时移除了无用的 `FOLD_RAKE_RATE` / `FOLD_RAKE_CAP` 局部常量（与 showdown 使用相同费率）。

### 5. `table-player-ops.ts` — 清理无用 import

移除了 `RAKE_RATE` / `RAKE_CAP` 的无用 import。

### 6. `table-game-logic.ts` — 重置抽水状态

### 7. `table-manager.service.ts` — 持久化抽水记录

**`persistSettlementRecords()`** 新增逻辑：

