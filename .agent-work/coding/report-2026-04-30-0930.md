## 处理的任务

### P2-NOTIFY-WS-BRIDGE — WebSocket 多实例广播问题 ✅ 已修复

**来源:** Test r53 发现的新问题（优先级 P2）

**问题描述:**
- `WebSocketManager.emitToUser()` 通过遍历 `server.sockets.sockets` 查找目标用户的 socket
- `server.sockets.sockets` 仅包含**当前 NestJS 实例**上的连接
- 多实例部署（PM2 cluster / K8s replicas）时，通知只推送给连接在**当前实例**的用户
- 其他实例上的用户完全收不到通知（`friend_status_update`、`notification`、`deposit_confirmed`、`blast_game_*` 等事件均受影响）

**修复方案:** Redis Pub/Sub Bridge

**改动文件:**
1. `apps/backend/src/websocket/websocket-manager.ts` — 重构
   - 新增 `localSockets: Map<string, Set<string>>` — 用户 ID → Socket ID 集合的本地索引，O(1) 查找
   - 新增 `redisSub` / `redisPub` — 两个独立的 ioredis 客户端（subscriber + publisher）
   - 新增 `initRedis(redisUrl)` — 在 `AppGateway.onModuleInit()` 调用，使用 `REDIS_URL` 环境变量
   - 新增 `registerSocket(userId, socketId)` / `unregisterSocket(userId, socketId)` — 连接/断开时注册
   - `emitToUser` 改为：本地立即送达 → Redis 发布（跨实例）；Redis 不可用时降级为纯本地（单实例行为不变）
   - 导出 `WS_REDIS_CHANNEL = 'ws:broadcast'` 常量

2. `apps/backend/src/websocket/app.gateway.ts` — `onModuleInit()` 中调用 `wsManager.initRedis(redisUrl)`

**技术细节:**
- Redis pub/sub channel: `ws:broadcast`
- 消息格式: `JSON.stringify({ userId, event, data })`
- `redisSub.on('message')` 回调处理跨实例消息，调用 `deliverToLocalUser()` 送达本地 socket
- `initRedis` 内部有 `if (this.redisSub) return` 防止重复初始化
- Redis 连接错误 → `warn` 日志但不阻断服务（fail-loud 但 non-fatal）
- 单实例部署：完全降级为原行为，无 Redis 依赖
