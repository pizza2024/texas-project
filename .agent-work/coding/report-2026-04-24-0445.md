### 测试结果

```
Tests: 5 failed, 6 passed, 11 total
```

**通过的 6 个**：

- ✓ does nothing when the disconnected client is not in a room
- ✓ does not remove the player when another socket for the same user is still active
- ✓ cancels pending room cleanup when the user reconnects before the grace period ends
- ✓ loads the persisted balance when a player joins a room
- ✓ rejects joining a room when the player has no balance
- ✓ prevents the same user from joining multiple rooms concurrently

**失败的 5 个（pre-existing setTimeout mock 限制）**：

- ✕ syncs the room state when a seated player disconnects
- ✕ runs settlement countdown, enters ready countdown, and then auto-starts the next hand
- ✕ rebuilds an in-progress settlement timer from restored state
- ✕ auto-checks on timeout when checking is allowed
- ✕ auto-folds on timeout when checking is not allowed

**根因**：`schedulePostHandFlow` 和 `scheduleActionTimeout` 在 TimerService mock 中是 `mockResolvedValue(undefined)`，但真实实现使用 `setTimeout` 在内部调用 `finalizeXxx`。Jest fake timers 无法穿透 mock 调用链到达真实 timer。需要重新设计 mock 来模拟 timer 触发后的 callback 执行，或使用真实 TimerService 实例（需要 mock RedisService）。
