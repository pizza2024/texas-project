## 处理的任务

### P0-NEW-TABLE-STUCK ✅

**文件:** `apps/backend/src/table-engine/table-player-ops.ts`
**问题:** `checkAndAutoFoldSittingOut` 在 `consecutiveTimeouts >= MAX_CONSECUTIVE_TIMEOUTS` (3次) 时直接 `return false`，不推进 turn，导致 `activePlayerIndex` 仍指向该玩家且 `hasActed=true`，牌桌永久卡死。
**修复:** 当 MAX 达到时，同样执行 fold + fold-win 检查 + turn 推进，与正常路径完全一致。仅区别是玩家状态保持 `FOLD`（而非 `SITOUT`），且 `consecutiveTimeouts` 重置为0。
**测试:** `sit-out-detector.spec.ts` — PASS

### P0-NEW-WITHDRAW-DOUBLE-REFUND ✅

**文件:** `apps/backend/src/withdraw/withdraw.service.ts` + `withdraw.service.spec.ts`
**问题:** `handleWithdrawFailure` 内两次并发调用均先 `findUnique` 读 `status=PENDING`，然后 `update`（无 WHERE 条件），两次都进入退款分支，导致双重退款。
**修复:** 将 `update` 改为 `updateMany` 加 `WHERE status='PENDING'` 条件 — 第二次调用时 `count=0`，早退不退款。这是标准的幂等原子操作模式。
**测试:** `withdraw.service.spec.ts` — 全部 15 PASS

### P0-NEW-TRC20-CHARS ✅

**文件:** `apps/backend/src/wallet/wallet.service.ts`
**问题:** TRC20 正则 `/^T[A-HJ-NP-Za-km-z1-9]{33}$/` 排除了合法 Base58 字符（0, I, O, l），导致部分有效 Tron 地址被拒绝。
**修复:** 改为 `/^T[A-Za-z0-9]{33}$/`，接受所有字母数字字符（34-char T-prefix 地址）。
**测试:** `wallet.service.spec.ts` — PASS

### P0-NEW-OTP-NORATELIMIT ✅ (已验证无需修改)

**文件:** `apps/backend/src/auth/auth.service.ts`
**问题:** `verifyEmailCode` 无 rate limit。
