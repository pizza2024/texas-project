## 代码改动

### 1. `apps/backend/src/matchmaking/matchmaking.service.spec.ts` (新增)

**33 个测试用例**，覆盖：

| 方法                          | 测试场景                                                                                        |
| ----------------------------- | ----------------------------------------------------------------------------------------------- |
| `hashIp`                      | 相同 IP 一致性，不同 IP 不同，SHA256 格式                                                       |
| `getPlayerElo`                | 用户存在返回 elo，不存在返回默认值 1000                                                         |
| `recordPlayerJoined`          | ELO map 和 IP set 正确写入                                                                      |
| `recordPlayerLeft`            | 正确删除，最后一个玩家清空 map/set                                                              |
| `findOrCreateMatchmakingRoom` | 找到合适房间/满员创建新房/按 createdAt 排序/tier+isMatchmaking+status 过滤/ELO 范围匹配         |
| `isRoomSuitable`              | ELO 差异超限拒绝，ELO 范围内存接受                                                              |
| `updateElo`                   | 手牌数<2 无操作，空 entries 无操作，零下注跳过，胜者/败者 ELO 增量计算，ELO 地板/天花板强制     |
| `checkCollusion`              | 空 otherUserIds 返回 false，24h 无手牌返回 false，共用手牌数低于阈值返回 false，超阈值返回 true |
| `onModuleInit`                | 清理过期 matchmaking 房间，活跃房间无操作                                                       |
| `BLIND_TIERS`                 | 5 个 tier 完整定义，结构正确，购买范围排序                                                      |

**关键 Mock 修复:**

- `$transaction`: 实现 `async fn => Array.isArray(fn) ? Promise.all(fn) : fn(mockPrisma)` — 支持数组和回调两种调用形式
- `user.updateMany`: 默认返回 `{ count: 0 }` — 避免 floor/ceiling 测试中 undefined
- `handAction.findMany` / `groupBy`: 默认返回 `[]` — 避免 `checkCollusion` 调用时报错

### 2. `apps/backend/src/wallet/wallet.service.spec.ts` (新增)

**13 个测试用例**，覆盖：

| 方法                     | 测试场景                                                                               |
| ------------------------ | -------------------------------------------------------------------------------------- |
| `getBalance`             | 钱包存在返回 chips，不存在回退 user.coinBalance，两者不存在返回 STARTING_CHIPS (10000) |
| `freezeBalance`          | 正确 upsert frozenChips 并扣减 coinBalance，负数归零                                   |

**关键 Mock 修复:**

- `$transaction`: 支持数组形式 `[[upsert, update, create]]` 和回调形式 `async tx => ...`
