## 本轮处理

### T-009 — app.gateway.spec.ts 修复

**问题**: `rebuilds an in-progress settlement timer from restored state` 测试失败。

**根因**: `handleJoinRoom` 调用 `gateway.ensureRecoveredRoundFlow()` 将控制权转给 `timerService.ensureRecoveredRoundFlow()`。测试 mock 的 `ensureRecoveredRoundFlow` 是空函数 `jest.fn()`，导致 `schedulePostHandFlow` 从未被调用，timer 链没有启动，`resetToWaiting` 和 `beginReadyCountdown` 永远不会被调用。

**修复**: 为 `ensureRecoveredRoundFlow` mock 添加实现逻辑，当 `table.currentStage === GameStage.SETTLEMENT` 且 `settlementEndsAt` 有值时，调用 `schedulePostHandFlow` 启动 mock timer 链。

**改动文件**: `apps/backend/src/websocket/app.gateway.spec.ts`

```typescript
// 修复前
ensureRecoveredRoundFlow: jest.fn(),

// 修复后
ensureRecoveredRoundFlow: jest
  .fn()
  .mockImplementation(
    (_server: any, _roomId: string, table: any) => {
      if (table.currentStage === GameStage.SETTLEMENT && table.settlementEndsAt) {
        const remainingMs = table.settlementEndsAt - Date.now();
        if (remainingMs > 0) {
          (gateway as any).schedulePostHandFlow(
            timerMockServer ?? gateway.server,
            _roomId,
            table,
            remainingMs,
            false,
          );
        }
      }
    },
  ),
```

**测试结果**: 11/11 通过（197/197 全量通过）
