## 处理的任务

### P1-BLAST-TIMEOUT — BLAST matchmaking timeout 未调用 ✅ 已修复

**问题描述:**
- `TournamentService` 已实现 `startMatchmakingTimeout` / `clearMatchmakingTimeout` 方法（30秒倒计时，超时触发 `matchmaking_timeout` WS 事件）
- 但 `createBlastLobby` / `joinBlastLobby` 从未调用这些方法
- 导致 lobb 永远不满人时无通知、无清理

**根本原因:**
- `startMatchmakingTimeout(server)` 需要 `socket.io.Server` 实例
- `TournamentService` 已有 `BroadcastService`，但没有获取 `Server` 的途径
- `WebSocketManager` 有 `getServer()` 方法，但未注入 `TournamentService`

**改动文件:**

1. **`tournament.service.ts`**
   - 新增 import: `WebSocketManager` from `../websocket/websocket-manager`
   - Constructor 新增 `@Inject(forwardRef(() => WebSocketManager)) private readonly wsManager: WebSocketManager`
   - `createBlastLobby` 末尾调用:
     ```ts
     const server = this.wsManager.getServer();
     if (server) {
       this.startMatchmakingTimeout(id, server);
     }
     ```
   - `joinBlastLobby` 中 `BLAST_MAX_PLAYERS` 满人时调用 `this.clearMatchmakingTimeout(lobbyId)` 取消倒计时

2. **`tournament.service.spec.ts`**
   - 新增 import: `WebSocketManager`
   - 新增 `mockWebSocketManager = { getServer: jest.fn().mockReturnValue(null), emitToUser: jest.fn() }`
   - Module providers 新增 `{ provide: WebSocketManager, useValue: mockWebSocketManager }`
