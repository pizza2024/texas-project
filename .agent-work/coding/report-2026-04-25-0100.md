## 处理的任务

### P2-NEW-001: calledAllIn re-entry raise 保护缺陷 ✅ 修复

**问题描述**:

- 原代码: `if (this.table.calledAllIn !== null && amount >= this.table.calledAllIn) return false;`
- 边界 case: Alice all-in 100，Bob call (calledAllIn=100)，Carol raise to 120 → `120 >= 100` → raise 被允许（错误）
- 正确行为: calledAllIn 已设置后，任何 raise 都应被阻止

**修复内容** (`apps/backend/src/table-engine/table-player-ops.ts`):

```typescript
// 修复前:
if (this.table.calledAllIn !== null && amount >= this.table.calledAllIn) {
  return false;
}

// 修复后:
if (this.table.calledAllIn !== null) {
  return false;
}
```

- 严格检查: 一旦有人 all-in 且被 call，calledAllIn 被设置后，本 street 禁止任何 raise
- 符合德州扑克规则: all-in 后只能 call/fold，不能再 raise

### P2-NEW-004: roomId 来源未验证 ✅ 修复

**问题描述**:

- `handlePlayerAction` 接受客户端提供的 `roomId`，但没有验证用户是否真的在该房间
- 恶意用户可以伪造 `roomId` 对其他房间发起操作

**修复内容** (`apps/backend/src/websocket/game.handler.ts`):

```typescript
// 获取用户当前房间
const userCurrentRoomId =
  await gateway.tableManager.getUserCurrentRoomId(userId);

// 如果客户端提供了 roomId，交叉验证
if (
  validated.roomId &&
  userCurrentRoomId != null &&
  validated.roomId !== userCurrentRoomId
) {
  return { event: "error", data: { message: "Invalid roomId" } };
}

// 如果未提供 roomId，从用户当前房间推导
let roomId = validated.roomId ?? null;
if (!roomId) {
  roomId = userCurrentRoomId;
}
```

- 安全检查: 仅当 userRooms index 已填充时（userCurrentRoomId != null）才验证
- 允许刚加入房间但 index 尚未更新的边界情况
