## 本轮完成

### ✅ P2-NEW-003 — 浮点数芯片金额验证

**文件**: `packages/shared/src/validation.ts` line 10

**改动**: `amount: z.number().nonnegative().max(1_000_000_000).optional()` → `amount: z.number().int().optional()`

**原因**: 芯片必须为整数，浮点数可能导致渲染/结算问题。

---

### ✅ P2-NEW-001 — calledAllIn 重入保护缺陷

**文件**: `apps/backend/src/table-engine/table-player-ops.ts` line 319

**改动**: `activePlayer.bet >= this.table.calledAllIn` → `amount >= this.table.calledAllIn`

**原因**: 原代码在 raise 流程中，`activePlayer.bet` 尚未更新（在下注操作后才更新），导致检查总是失败。修复后使用 `amount`（目标加注额）与 `calledAllIn` 比较，正确阻止超额加注。

**场景**: Alice all-in 100 → Bob call (calledAllIn=100) → Carol raise to 150，现在正确拒绝。

---

### ✅ P2-NEW-004 — roomId 类型不准确

**文件**: `apps/backend/src/websocket/game.handler.ts` line 240-241

**改动**: `data: { action: unknown; amount?: unknown; roomId?: unknown }` → `data: PlayerActionInput`

**原因**: 使用 `PlayerActionInput` 类型（从 `PlayerActionSchema` 推断）替代 `unknown`，提供正确的类型安全。`roomId` 现在正确类型为 `string | undefined`。
