## 处理的任务

### P1-NEW-003 — `setBalances` Phase 1+2 非原子（财务一致性）

**严重度:** HIGH — wallet.chips 和 user.coinBalance 可能不一致

**问题:** Phase 1（wallet upsert）和 Phase 2（user.coinBalance update）是两个独立 `$transaction`。Phase 1 成功后 Phase 2 失败 → DB 状态不一致。

**修复:** 将 Phase 1 + Phase 2 合并为单一 `$transaction`，使用 `flatMap` 把每个 entry 的 wallet op 和 user op 放入同一事务数组。Bot 用户（`BOT_ID_PREFIX`）只执行 wallet op。

**文件:** `apps/backend/src/wallet/wallet.service.ts`
**Commit:** `3cafbb2`

### P1-NEW-004 — `exchangeChipsToBalance` TOCTOU 余额检查（资金安全）

**严重度:** HIGH — 并发请求可绕过余额检查导致双花

**问题:** `availableChips` 在事务外读取（第275行），两个并发请求可以同时通过余额检查，然后都执行扣款，导致超额提现。

**修复:** 移除事务外的 `getAvailableBalance` 调用，将余额检查移至事务内部。在事务内读取 `wallet.frozenChips`，计算 `availableChips = currentBalance - frozenChips`，通过后才执行扣款。

**文件:** `apps/backend/src/wallet/wallet.service.ts`
**Commit:** `3cafbb2`
