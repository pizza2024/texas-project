## 本轮处理

### P2-NOTIFY-001 Phase 2 — 通知前端 UI

**问题:** P2-NOTIFY-001 Phase 1 后端已完成（commit `4994a23`），但前端 UI 缺失。

**实施内容:**

#### 1. `packages/shared/src/socket.ts` — 通知类型 + Socket 事件
- 新增 `NotificationType` 联合类型（friend_online/.../system）
- 新增 `NotificationPayload` 接口（id, userId, type, title, body, metadata, read, createdAt）
- `ServerToClientEvents` 新增 `notification` 事件
- 新增 `setNotificationHandler()` 导出函数
- `getSocket()` 中注册 `socket.on("notification", ...)`

#### 2. `apps/web/hooks/use-notifications.ts` — 通知状态 Hook（新建）
- `useNotifications()` — 完整的通知管理 Hook
  - `notifications[]` — 通知列表
  - `unreadCount` — 未读数（Badge 显示）
  - `markRead(ids)` — 单条/批量标已读
  - `markAllRead()` — 全部标已读
  - `refresh()` — 重新拉取
- 初始加载：GET `/notifications`
- 实时推送：通过 `setNotificationHandler` 接收 WS `notification` 事件，新通知 prepend 到列表
- 非阻塞设计：错误不打断用户体验

#### 3. `apps/web/components/notification/notification-bell.tsx` — Bell 按钮（新建）
- 🔔 Bell 图标按钮（`relative w-10 h-10`）
- 未读数红色渐变 Badge（右上角，圆角，`99+` 截断）
- 点击 toggle 下拉面板
- Outside-click / Escape 关闭

#### 4. `apps/web/components/notification/notification-dropdown.tsx` — 通知面板（新建）
- 固定宽度 `w-80`，最大高度 `max-h-[400px]`，滚动
- 暗色主题（金色边框，匹配德州扑克设计语言）
- 每条通知：图标（按 type 映射）/ 标题 / 正文（2行截断）/ 时间（相对时间）/ 未读蓝色左边框 + 橙色圆点
- 顶部：「通知中心」+「全部标为已读」按钮
- 底部：「关闭」按钮
- 空状态：🔔 + "No notifications yet"

#### 5. `apps/web/components/lobby/lobby-header.tsx` — 集成
- `import { NotificationBell } from "@/components/notification/notification-bell"`
