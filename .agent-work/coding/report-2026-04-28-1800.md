## P2-TEST-006 — ActionBar 无重复提交保护 ✅ 已修复

**问题:** `handleAction` (PC) 和 `emit` (Mobile) 直接调用 `socket.emit('player_action', ...)` 无防重放保护。快速连续点击（网络延迟 + 用户手速）可能导致服务器收到多次相同操作。

**修复:** 引入 `actionPendingRef` 布尔标志，1 秒窗口内屏蔽重复提交：

### PC (`apps/web/app/room/[id]/page.tsx:608-617`)

```typescript
const handleAction = (action: string, amount?: number) => {
  // P2-TEST-006: debounce duplicate submissions
  if (actionPendingRef.current) return;
  const socket = getAuthorizedSocket();
  if (!socket) return;
  actionPendingRef.current = true;
  socket.emit("player_action", { roomId: id as string, action, amount });
  // Reset when player's turn ends (next state update or timeout)
  setTimeout(() => {
    actionPendingRef.current = false;
  }, 1000);
};
```

### Mobile (`apps/web/app/room-mobile/[id]/page.tsx:386-395`)

```typescript
const emit = (action: string, amount?: number) => {
  // P2-TEST-006: debounce duplicate submissions
  if (actionPendingRef.current) return;
  const token = getStoredToken();
  if (!token) return;
  actionPendingRef.current = true;
  getSocket(token).emit("player_action", {
    roomId: id as string,
    action,
    amount,
  });
  setTimeout(() => {
    actionPendingRef.current = false;
  }, 1000);
};
```

**设计决策:**

- 1 秒 reset 窗口足够覆盖绝大多数网络延迟重试场景
- `setTimeout` reset 不依赖服务器响应 — 防止服务器响应慢时用户被永久卡住
- 比 `useState` 更合适：`useRef` 更新不同步触发 re-render，无 UI 闪烁风险
