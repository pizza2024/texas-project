### P0-SEC-001 — Redis Session 验证绕过（Critical）
**文件**: `apps/backend/src/auth/jwt.strategy.ts`

**问题**: Redis 不可用时，认证被完全绕过（fail-open），允许任何持有效 JWT 的用户通过验证，存在严重安全隐患。

**修复**:
- 移除了允许认证 bypass 的代码逻辑
- 改为 fail-closed 策略：Redis 不可用时抛出 `UnauthorizedException('AUTH_SERVICE_UNAVAILABLE')`
- 日志改为 `[SECURITY] Redis unavailable — session validation FAILED...`

**改动**:
```diff
- // Security trade-off: Redis down + reject all = total service outage.
- // Instead we allow auth but log a CRITICAL-severity error...
- this.logger.error(`[SECURITY-AUTH-BYPASS] CRITICAL...`);
- // (no throw — allowed auth through)

+ // Redis unavailable + sessionId present = fail-closed (deny auth).
+ this.logger.error(`[SECURITY] Redis unavailable...`);
+ throw new UnauthorizedException('AUTH_SERVICE_UNAVAILABLE');
```
