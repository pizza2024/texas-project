### ✅ W-001: Withdraw 分布式 cooldown 绕过修复

**文件**: `apps/backend/src/withdraw/withdraw.service.ts`

**改动**:

- Redis SETNX 分布式锁机制已正确实现
- **关键修复**: 当 Redis 不可用时（setNX 返回 null），不再使用不安全的 in-memory Map 降级，而是直接拒绝提现请求
- 添加 `[SECURITY-WITHDRAW-BYPASS]` 前缀的错误日志

**修复前** (不安全的降级):

```typescript
// Redis unavailable — fall back to in-memory (dev/test single-instance only)
this.logger.warn(...); // 警告但允许
// ... in-memory cooldown 逻辑，允许攻击者绕过
```

**修复后** (安全的阻断):

```typescript
// Redis unavailable — BLOCK all withdrawals to enforce cooldown globally
this.logger.error(`[SECURITY-WITHDRAW-BYPASS] CRITICAL: Redis unavailable...`);
throw new BadRequestException("提现服务暂时不可用，请稍后再试");
```
