### P1-WITHDRAW-005 — createWithdraw transaction log 在原子块外 ✅

**修复:** 移除原子块外的 `transaction.create` 调用（该日志在结算时由 `processWithdraw` 或 `handleWithdrawFailure` 统一记录，无需在创建时单独记录）。

**文件:** `apps/backend/src/withdraw/withdraw.service.ts`

### P1-WALLET-002 — setBalances Phase 2 失败静默吞掉 ✅

**问题:** Phase 2 (更新 `User.coinBalance`) 若 `PrismaClientKnownRequestError` 对非 bot 用户失败，仅 warn 并 return，不抛出异常。导致 `wallet.chips` 已更新但 `user.coinBalance` 不同步。

**修复:** 
- 对非 bot 用户：抛出异常 (`throw error`)
- 改用 `Promise.allSettled` 收集所有结果
- 统一日志记录失败数量 (`logger.error`)

**文件:** `apps/backend/src/wallet/wallet.service.ts`

---

### P1-WITHDRAW-007 — executeChainWithdraw 链上确认后 DB 更新无重试 ✅

**问题:** `executeChainWithdraw` 在链上交易 `tx.wait(1)` 确认后，直接执行 `withdrawRequest.update` 更新 `txHash/status`。若此时 DB 不可用，链上已确认但 DB 状态滞留在 `PROCESSING`。

**修复:** 
- 链上确认后 DB 更新包入 3 次重试循环
- 指数退避: 500ms × attempt
- 全部重试失败时 `logger.error` + 抛出 `Error`（链上已确认，DB 需人工介入）

**文件:** `apps/backend/src/withdraw/withdraw.service.ts`
