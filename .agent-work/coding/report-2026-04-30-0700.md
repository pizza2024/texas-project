## 本轮处理

### P2-WITHDRAW-UX-001 — 提现地址簿 ✅

**任务来源:** Productor 第400轮优先队列
**优先级:** P2（竞品对照：GGPoker/WSOP/CoinPoker/ACR 均已有此功能）

**实现内容:**

#### 1. Prisma Schema — `apps/backend/prisma/schema.prisma`
- 新增 `WithdrawAddress` model：
  - `id` (uuid, PK)
  - `userId` (String, FK → User)
  - `address` (String)
  - `label` (String?, optional)
  - `isDefault` (Boolean, default false)
  - `createdAt` (DateTime)
  - `@@unique([userId, address])` — 防止重复地址

#### 2. 后端 Controller — `withdraw.controller.ts`
4个新端点：
- `GET /withdraw/addresses` — 获取用户所有保存的地址
- `POST /withdraw/addresses` — 保存新地址（支持 label）
- `DELETE /withdraw/addresses/:addressId` — 删除地址
- `PATCH /withdraw/addresses/:addressId/default` — 设为默认

#### 3. 后端 Service — `withdraw.service.ts`
4个新方法：
- `getSavedAddresses(userId)` — 按 isDefault desc, createdAt desc 排序
- `saveAddress(userId, address, label?)` — 创建或更新地址；首个地址自动设为默认
- `deleteAddress(userId, addressId)` — 删除并重新指定默认地址
- `setDefaultAddress(userId, addressId)` — 清除现有默认，设为新默认

#### 4. 前端 — `apps/web/app/withdraw/page.tsx`
地址簿 UI 集成：
- `SavedAddress` 接口（id, label, address, isDefault, createdAt）
- 状态管理：savedAddresses, addressBookOpen, showAddAddress, newAddressLabel, newAddressValue, addressLabelError, addAddressLoading
- 地址簿列表 UI（可折叠）
- "Use" 按钮 → 自动填充 toAddress 表单
- "Set as Default" / "Delete" 按钮
- 内联新增地址表单（ETH 地址格式校验 + label 输入）
- API 调用：GET/POST/DELETE/PATCH `/withdraw/addresses`

#### 5. i18n — `en.json` + `zh-CN.json`
新增 key:
- `withdraw.addressBook`
- `withdraw.addAddress`
- `withdraw.deleteAddress`
- `withdraw.setDefault`
- `withdraw.addressLabel`
- `withdraw.noSavedAddresses`
