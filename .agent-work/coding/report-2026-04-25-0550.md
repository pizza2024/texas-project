### ✅ P1-003: 密码暴力破解保护 in-memory Map — 迁移至 Redis

**问题**: `checkPasswordAttemptLimit()` 使用进程内 `Map<string, {count, windowStart}>`，多实例部署中攻击者可绕过限制。

**修复文件**: `apps/backend/src/websocket/connection-state.service.ts`

**修复方案**: 将 `passwordAttempts` Map 完全迁移到 Redis：

- Redis key 格式：`brute:{ip}:{roomId}`，TTL = `PASSWORD_ATTEMPT_WINDOW_MS`（5分钟）
- `checkPasswordAttemptLimit()` 改为 `async`，使用 `redisService.incr(key, ttl)` 原子操作
- `clearPasswordAttempts()` 改为 `async`，使用 `redisService.del(key)`
- Redis 不可用时 fail-closed（返回 `'banned'`）— 安全关键路径必须严格

保留原有行为：5次错误 → 5分钟封禁，第3/4次失败时 warn 日志。
