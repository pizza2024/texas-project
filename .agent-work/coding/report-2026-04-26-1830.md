### P1-MOBILE-RECONNECT ✅

**问题:** Mobile 端完全缺失 `rejoin_available` handler，断线后无法重新加入牌桌。

**改动:** `apps/mobile/app/room/[id].tsx`
- 在 `useEffect` 中 socket 连接后注册 `rejoin_available` handler
- Alert 弹窗告知用户检测到重新连接
- 点击"确定"后自动 emit `join_room` 恢复游戏状态

```tsx
socket.on('rejoin_available', ({ roomId }) => {
  if (!mounted) return;
  Alert.alert('重新连接', '检测到您已重新连接，正在恢复游戏...', [
    { text: '确定', onPress: () => socket.emit('join_room', { roomId: id }) },
  ], { cancelable: false });
});
```

### P1-JWT-LOCALSTORAGE ✅ (Backend additive)

**问题:** JWT 存 localStorage，XSS 可窃取 Token。

**改动:**

#### `apps/backend/src/auth/auth.controller.ts`
- `login` 端点：登录成功后在 Response 上设置 httpOnly cookie（与返回 body 同时进行）
- `register` 端点：同样设置 httpOnly cookie

```ts
res.cookie('access_token', token, {
  httpOnly: true,
  secure: process.env.NODE_ENV === 'production',
  sameSite: 'lax',
  maxAge: 7 * 24 * 60 * 60 * 1000,
  path: '/',
});
```

#### `apps/backend/src/auth/auth.service.ts`
- 新增 `generateTokenForUser(userId)` 方法，供 register 在设置 cookie 时生成 Token

#### `apps/backend/src/auth/jwt.strategy.ts`
- `jwtFromRequest` 优先读 Authorization Bearer header，fallback 读 httpOnly cookie
- 适配 `passReqToCallback: true`

**说明:** 此为 additive change，localStorage 仍正常工作（向后兼容）。Web 前端迁移留待后续 P2。
