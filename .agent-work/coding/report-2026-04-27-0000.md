## 本轮处理

### P2-CHAT-IDEMPOTENCY ✅ 已修复（commit 770ddcf）

**问题:** 聊天室消息无幂等键，网络重试时同一消息可能被处理多次

**方案:** 客户端发送可选 `clientMessageId`（UUID），服务端用 Redis `SET NX EX 60` 原子去重

**改动文件（4个）:**

| 文件                            | 改动                                                                         |
| ------------------------------- | ---------------------------------------------------------------------------- |
| `connection-state.service.ts`   | 新增 `isMessageProcessed(clientMessageId)` — Redis SET NX EX 60，fail-closed |
| `app.gateway.ts`                | 新增 `checkMessageProcessed()` 委托到 connectionState                        |
| `game.handler.ts`               | `handleChatMessage` 在处理前检查 `clientMessageId`，重复则返回 `chat_error`  |
| `packages/shared/src/socket.ts` | `ClientToServerEvents.chat-message` 添加 `clientMessageId?: string`          |

**去重逻辑:**

```
客户端: socket.emit('chat-message', { roomId, content, clientMessageId: crypto.randomUUID() })
服务端: SET chat_idem:{clientMessageId} 1 NX EX 60
        → OK（新建）→ 处理消息
        → null（已存在）→ client.emit('chat_error', { message: '消息已发送，请勿重复提交' })
```

**Fail-closed:** Redis 不可用时 `isMessageProcessed` 返回 `false`，视为重复并拒绝（安全策略，与其他限流一致）
