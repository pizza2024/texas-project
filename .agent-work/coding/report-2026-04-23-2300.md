### P1 — Test 代理安全问题 (W-001, W-002)

#### ✅ [W-001] Withdraw 分布式 cooldown 绕过问题 — 已修复

**文件**: `apps/backend/src/withdraw/withdraw.service.ts`

**问题**: 原实现用 Redis TTL 检查 + in-memory Map fallback。Redis 不可用时，多实例部署可绕过 cooldown。

**修复**:
- 新增 `RedisService.setNX()` — 原子性 SET if Not eXists，带 TTL
- `checkAndConsumeCooldown()` 改用 SETNX 分布式锁模式:
  - 成功获取锁 → 无 cooldown，直接通过
  - 锁存在（acquired=false）→ cooldown 仍激活，throw 并告知剩余秒数
  - Redis 不可用 → fallback 到 in-memory（仅开发/单实例模式），并记录 `[SECURITY]` 警告
- `setCooldown()` 改用 `setNX` 原子写入，不再同时写 Redis + memory

**文件**: `apps/backend/src/redis/redis.service.ts`
- 新增 `setNX(key, value, ttlSeconds)` 方法，返回 `boolean | null`
