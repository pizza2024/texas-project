## 本轮调研 — PokerCraft / HUD 玩家数据统计功能缺口

### 核心发现

本项目目前 **完全没有玩家数据统计（HUD）功能**。竞品已将此类功能作为核心差异点。

### 竞品 HUD 功能对比

| 平台      | 产品名        | 核心 HUD 统计                    | 特色                             |
| --------- | ------------- | -------------------------------- | -------------------------------- |
| GGPoker   | PokerCraft    | VPIP/PFR/AF/WTSD/W$SD/3-Bet/limp | 实时 HUD + 长期数据库 + 牌局回放 |
| ACR       | Poker Tracker | VPIP/PFR/AF/CBet/FChip           | 第三方 HUD 支持                  |
| CoinPoker | 内置统计      | VPIP/PFR/AF + 区块链验证         | 透明化数据                       |
| 888poker  | pokerTracker  | 基础统计                         | 简洁 HUD                         |

### 核心 HUD 指标定义

| 指标      | 全称                         | 计算公式                            | 参考值（9人桌） |
| --------- | ---------------------------- | ----------------------------------- | --------------- |
| **VPIP**  | Voluntarily Put Money In Pot | ` voluntaryPots / totalHands * 100` | 15-25%          |
| **PFR**   | Pre-Flop Raise               | `preflopRaises / totalHands * 100`  | 10-18%          |
| **AF**    | Aggression Factor            | `(bets+folds) / calls` (postflop)   | 1.5-4.0         |
| **WTSD**  | Went to Showdown             | `showdowns / totalHands * 100`      | 20-30%          |
| **W$SD**  | Won Money at Showdown        | `winsAtShowdown / showdowns * 100`  | 48-55%          |
| **3-Bet** | 3-Bet %                      | `re3bets / opps3bet * 100`          | 4-8%            |
| **CBet**  | Continuation Bet %           | `cbets / opportunities * 100`       | 50-70%          |

### GGPoker PokerCraft 详细功能

**数据采集层：**

- 每手牌记录：动作序列、筹码变化、摊牌手牌、底池大小
- 存储：云端长期数据库，支持多年历史

**HUD 显示层：**

- 玩家头像旁 overlay 显示 2-4 个关键指标（可自定义）
- 颜色编码：红色=紧/蓝色=松
- 点击玩家名称展开完整统计面板

**分析功能：**

- 牌局回放（带 HUD 数据叠加）
- 对手倾向标记（LAG/TAG/NIT/FISH）
- 漏斗分析（where money comes from）

### 本项目缺口分析

**当前数据基础（已有）：**

- `HandAction` — 每手牌操作记录（FOLD/CHECK/CALL/RAISE/ALLIN/STRADDLE）
- `Hand` — potSize, winnerId, timestamps
- `Settlement` — 每局结算金额
- `User.elo` — ELO 评分（但非 poker-specific）

**缺失的关键统计计算：**

| 统计  | 缺失原因                                                | 实现难度 |
| ----- | ------------------------------------------------------- | -------- |
| VPIP  | 未追踪 voluntary put（check-call vs bet-call 无法区分） | 中       |
| PFR   | 未记录 preflop raise 次数                               | 低       |
| AF    | 未区分 bet/raise vs call                                | 中       |
| WTSD  | 需要追踪 showdown 参与                                  | 低       |
| 3-Bet | 需要记录 3-bet 机会和次数                               | 高       |
| CBet  | 需要记录 flop 发起 cbet 次数                            | 高       |

**最关键的缺口： HandAction 缺少 `betType` 字段**

当前 `HandAction` 仅有 `type: FOLD/CHECK/CALL/RAISE/ALLIN/STRADDLE/SIT-OUT`，无法区分：

- Call 之前是否有过 bet/raise（影响 AF 计算）
- Raise 是 3-bet 还是 4-bet+
- Straddle 后动作类型

### 推荐实施路径

**Phase 1 — 基础统计（推荐优先）**

```
1. 扩展 HandAction 添加字段：
   - isVoluntary: boolean（bet/raise/straddle 算 voluntary，check/call 不算）
   - street: 'preflop' | 'flop' | 'turn' | 'river'
   - betSize?: number

2. 新增统计服务：
   - PlayerStatsService
   - 计算 VPIP/PFR/WTSD/W$SD/AF

3. WebSocket 推送 player-stats 事件
```

**Phase 2 — HUD UI**

```
4. 玩家头像旁显示 2-3 个核心指标
5. 玩家详情弹窗显示完整统计
```

**Phase 3 — 高级分析**

```
6. 对手倾向分类（TAG/LAG/NIT/FISH）
7. 牌局回放 HUD 叠加
```
