### P2-CLUB-002: Club 排行榜

#### API 设计

**Endpoint**: `GET /clubs/:clubId/leaderboard`

**Query Params**:
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `period` | `daily` \| `weekly` \| `monthly` \| `all` | `weekly` | 排行周期 |
| `metric` | `elo` \| `chips` \| `hands` \| `winrate` | `elo` | 排行维度 |
| `limit` | number (1-100) | 20 | 每页数量 |
| `cursor` | string | — | 分页游标 |

**Response Schema**:
```json
{
  "data": [
    {
      "rank": 1,
      "userId": "uuid",
      "username": "PlayerOne",
      "avatar": "https://...",
      "value": 1542,
      "change": 3,
      "badge": "DIAMOND"
    }
  ],
  "meta": {
    "total": 150,
    "nextCursor": "opaque_cursor_string"
  }
}
```

**Backend 实现要点**:
1. `Hand` + `Settlement` 按 `clubId` 聚合 → 通过 `Table.Room.clubId` join
2. `period` 映射到时间范围过滤（`createdAt >= periodStart`）
3. `elo` 直接读 `User.elo`；`chips` = SUM(`Settlement.amount`)；`hands` = COUNT；`winrate` = wins/played
4. Redis 缓存排行榜，TTL = 5分钟（避免每次请求重算）

#### Frontend 实现要点

- Club 页面新增 "Leaderboard" Tab
- 维度切换 Tab：`ELO` | `筹码盈亏` | `手牌数` | `胜率`
- 周期切换：`日` | `周` | `月` | `总览`
- 前3名特殊样式（金/银/铜牌）
- 用户当前排名高亮（如果是 Club 成员）

### P2-CLUB-003: Club 专属房间

#### 房间关联模型

```typescript
// Room model 已有
Room {
  clubId?: string  // nullable，外键 → Club.id
  isClubOnly: boolean  // 是否仅俱乐部成员可见
}
```

#### API 设计

**创建 Club 专属房间** `POST /rooms`
```json
{
  "name": "VIP 俱乐部专属",
  "clubId": "uuid",
  "isClubOnly": true,
  "smallBlind": 100,
  "bigBlind": 200,
  "maxPlayers": 6,
  "buyInMin": 1000,
  "buyInMax": 10000,
  "password": "optional_password"
}
```

**房间列表过滤** `GET /rooms`
新增 Query Param: `clubId=uuid` — 返回该 Club 专属房间

**权限控制**:
1. `isClubOnly=true` + `clubId` 设置 → 仅 Club 成员可见
2. Club Owner/ADMIN 可创建/编辑/删除 Club 专属房间
3. Club Member 可见但进入时校验成员身份

#### Frontend 实现要点

- Club 页面新增 "Rooms" Tab
- 显示 Club 专属房间列表（含状态：WAITING/PLAYING）
- "创建专属房间" 按钮 → 弹窗表单（仅 Owner/ADMIN 可见）
- 房间 Card 显示当前人数和盲注级别

### P2-CLUB-004: Club 战绩统计

#### API 设计

**Endpoint**: `GET /clubs/:clubId/stats`

**Query Params**:
| 参数 | 类型 | 说明 |
|------|------|------|
| `userId` | uuid (optional) | 特定成员，不传则返回 Club 总览 |

**Response Schema**:
```json
{
  "clubId": "uuid",
  "userId": null,
  "period": {
    "from": "2026-04-01",
    "to": "2026-04-25"
  },
  "totalHands": 15420,
  "totalChips": 2456000,
  "netChips": 456000,
  "avgPot": 159,
  "topPlayers": [
    {
      "userId": "uuid",
      "username": "PlayerOne",
      "netChips": 250000,
      "winRate": 0.42,
      "handsPlayed": 892
    }
  ],
  "popularRooms": [
    {
      "roomId": "uuid",
      "roomName": "VIP 1",
      "handsPlayed": 3240
    }
  ],
  "memberCount": 48,
  "activeMembers": 32
}
```

**聚合维度**（基于 `Settlement` + `Hand` + `Room.clubId`）:
- 总手牌数、总局数
- 总筹码流动（流入/流出）
- 净筹码（赢 - 输）
- 平均彩池大小
- 活跃成员数（近30天有手牌）
- 热门房间（按手牌数排序）

#### Frontend 实现要点

- Club 页面新增 "Stats" Tab
- 顶部卡片：总手牌、总盈亏、活跃成员
- 折线图：每日手牌数趋势
- 柱状图：TOP 10 成员盈亏排行
- 时间范围选择器：近7天/30天/90天/全部

参考 GGPoker / 888poker 的 Squash/Replayer 功能，设计方向：

1. **手牌历史页** (`/hands`) 新增 "复盘" 按钮
2. **复盘面板**（Modal 或全屏）：
   - 左侧：牌桌俯视图（5张公共牌 + 2张手牌分列）
   - 右侧：操作时间线（每位玩家的 Fold/Call/Raise 节点）

3. **GGPoker Squash 特色**：多桌同步复盘，适合教练分析

**Backend 数据需求**:
- `Hand.stateSnapshot` 已存储完整牌桌状态（JSON）
- `HandAction` 已记录所有操作 + 时间戳
- 前端直接使用现有数据，无需新增 API
