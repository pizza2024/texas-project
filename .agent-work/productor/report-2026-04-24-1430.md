## MVP 方案（Phase 1 — 社交功能）

### 1. 玩家笔记系统

```typescript
model PlayerNote {
  id          String   @id @default(uuid())
  ownerId     String   // 笔记所有者
  targetId    String   // 被记录的玩家
  content     String   @db.VarChar(500)
  tags        String[] // ["nit", "fish", "reg", "whale"]
  color       String   // 标签颜色
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@unique([ownerId, targetId])
}
```

**API 端点：**
```
GET    /api/notes          — 获取我的所有笔记
POST   /api/notes          — 创建笔记
PATCH  /api/notes/:id      — 更新笔记
DELETE /api/notes/:id      — 删除笔记
GET    /api/notes/:userId  — 获取对特定玩家的笔记
```

### 2. 桌内表情系统

```typescript
// 表情枚举
const TABLE_EMOJIS = ['👏', '🔥', '😂', '🤔', '😎', '🙏', '😱', '💪'] as const;
type TableEmoji = typeof TABLE_EMOJIS[number];

// WebSocket 事件
interface EmojiReaction {
  senderId: string;
  targetSeat?: number; // 可选，发送给特定座位
  emoji: TableEmoji;
  handId: string;
}
```

**UI 设计建议：**
- 位于玩家头像旁或底部操作栏
- 每手牌限发 1 次表情
- 表情 3 秒后自动消失（不干扰牌局）
- 表情气泡样式，不遮挡牌面

### 3. 桌内聊天系统

```typescript
// 聊天消息
interface ChatMessage {
  id: string;
  roomId: string;
  senderId: string;
  content: string; // max 200 chars
  createdAt: DateTime;
}

// 安全措施
// • 敏感词过滤（服务端 + 客户端双重）
// • 同一消息 1 秒内不可重复发送
// • 每分钟最多 10 条消息（rate limit）
// • 管理员可禁言玩家
```

**MVP 实现范围：**
- 仅同桌玩家可见
- 200 字符限制
- 敏感词过滤
- 可关闭聊天选项
