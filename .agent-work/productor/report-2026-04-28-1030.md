## 本轮调研专题：Blast 即时赛事深度解析与 Phase 1 实施路径

### GGPoker Blast 机制拆解

GGPoker 的 Blast 是其核心变现产品之一，机制如下：

```
参赛费: $5 / $10 / $25 / $50 / $100
座位: 3人（固定）
奖池倍数: 1x ~ 15,000x（开赛时随机抽取）
基础奖池: 参赛费 × 3 × 倍数
```

**关键机制：**
1. **3人即开** — 凑齐3人立即开始，无需等待
2. **倒计时池** — 若3人未凑齐，每90秒注入 prize wheel 旋转（制造期待感）
3. **倍池加速** — 时间越长，奖池倍数上限越高
4. **盲注加速** — 每2分钟涨盲（快节奏，压榨玩家）

### Phase 1 实施路径（数据库模型 + 基础 API）

**第一步：数据库模型扩展（Prisma）**

```prisma
model BlastTournament {
  id            String   @id @default(uuid())
  roomId        String   @unique
  multiplier    Int      // 2x ~ 10000x
  status        BlastStatus // WAITING / STARTED / COMPLETED
  startedAt     DateTime?
  completedAt   DateTime?
  entries       BlastEntry[]
  createdAt     DateTime @default(now())
}

model BlastEntry {
  id              String   @id @default(uuid())
  tournamentId    String
  userId          String
  buyin           Int      // chips 扣款
  position        Int      // 座位号
  finalMultiplier Int?     // 结算时的倍数
  prize           Int?     // 获得的 chips
  createdAt       DateTime @default(now())
}
```

**第二步：Room 扩展**

- `room.isBlast = true` 标识即时赛事房间
- `room.blastConfig` JSON 字段存储配置（minPlayers=3, buyin tiers, maxMultiplier）

**第三步：BlastService 核心逻辑**

```
createBlastRoom()         // 创建 BLAST 房间
joinBlast(tournamentId)   // 加入赛事（扣 chips，检查人数）
startBlastIfFull()        // 3人齐则开赛
distributePrizes()        // 结算，按排名分配
```

**第四步：WebSocket 事件扩展**

```
客户端 → 服务器:
  join-blast-tournament
  leave-blast-tournament

服务器 → 客户端:
  blast-tournament-state
  blast-starting
  blast-result
```

### Blast vs 普通 Cash Game 差异点

| 维度 | Cash Game | Blast Tournament |
|------|-----------|-----------------|
| 时长 | 无限 | 固定3分钟 |
| 人数 | 2-9人 | 固定3人 |
| 涨盲 | 可选 | 无 |
| 奖池 | 动态 | 固定（倍数×买入×3） |
| 变现 | Rake | 门票 + 随机奖池 |
| 社交 | 高 | 中（3人更亲密） |

### Blast 的商业价值

1. **快速循环** — 3分钟一局，高频消费 chips
2. **随机刺激** — 奖池倍数制造"差一点赢"的成瘾性
3. **低门槛高回报** — $5 参赛可能赢得 $50,000（10000x）
4. **天然话题性** — 玩家会分享"刚在 Blast 赢得 8000x"
