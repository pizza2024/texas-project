## 本轮深度调研 — 俱乐部/公会系统竞品分析

### 概述

俱乐部系统（Club/Guild）是德州扑克游戏中最重要的社交功能和留存机制之一。竞品分析显示，所有主流平台均实现了俱乐部系统。

### 详细分析

#### PokerStars Club（行业标杆）

**创建门槛**：
- 需要 VIP 等级或付费
- 防止垃圾俱乐部泛滥

**核心功能**：
1. **私有桌**：俱乐部成员专享，需密码或俱乐部邀请才能加入
2. **俱乐部排行榜**：周/月战绩排名，激励竞争
3. **俱乐部锦标赛**：定期举办的私人比赛，奖品池来自俱乐部成员贡献
4. **聊天功能**：俱乐部成员内部聊天，社交核心
5. **徽章系统**：成员等级（Bronze/Silver/Gold/Platinum）

**技术实现要点**：
- 俱乐部 ID 关联 Room 私有标志
- 俱乐部专属 ELO 排名
- 邀请链接生成（一次性有效）

#### WSOP Championship Club

**特点**：
- 与真实 WSOP 赛事挂钩
- 俱乐部成员可获得真实赛事门票
- 社交化元素最丰富

**功能**：
- 俱乐部专属徽章（虚拟收藏品）
- 成员互动表情
- 俱乐部挑战任务

#### 888poker Club

**特点**：
- 简化版俱乐部系统
- 主要聚焦于私有牌桌
- 轻社交，重游戏

### 俱乐部系统数据模型建议

基于竞品分析和现有 CHIPS 数据模型，建议增加以下表结构：

```prisma
model Club {
  id          String   @id @default(uuid())
  name        String   @unique
  description String?
  ownerId     String
  inviteCode  String   @unique  // 一次性邀请码
  isPublic    Boolean  @default(true)
  minBuyIn    Int?     // 俱乐部最低买入
  createdAt   DateTime @default(now())
  
  members     ClubMember[]
  rooms       ClubRoom[]
  chatMessages ClubChat[]
}

model ClubMember {
  id        String     @id @default(uuid())
  clubId    String
  userId    String
  role      ClubRole   @default(MEMBER)  // OWNER/ADMIN/MEMBER
  joinedAt  DateTime   @default(now())
  
  club      Club      @relation(fields: [clubId], references: [id])
  user      User      @relation(fields: [userId], references: [id])
  
  @@unique([clubId, userId])
}

enum ClubRole {
  OWNER
  ADMIN
  MEMBER
}

model ClubRoom {
  id        String  @id @default(uuid())
  clubId    String
  roomId    String  @unique
  
  club      Club    @relation(fields: [clubId], references: [id])
  room      Room    @relation(fields: [roomId], references: [id])
}

model ClubChat {
  id        String   @id @default(uuid())
  clubId    String
  userId    String
  content   String
  createdAt DateTime @default(now())
  
  club      Club    @relation(fields: [clubId], references: [id])
}
```

## 项目体验记录

### Pot-Relative Raise 实现验证 ✅

通过代码审查 `ActionBar.tsx` 工作区变更，确认实现如下：

**快捷按钮行**（第 235-280 行区域）：
| 按钮 | 计算 | 样式 |
|------|------|------|
| Min | `min(minRaiseTo, myPlayerStack)` | 琥珀色边框 |
| ½ Pot | `min(floor(pot × 0.5), myPlayerStack)` | 琥珀色边框 |
| ¾ Pot | `min(floor(pot × 0.75), myPlayerStack)` | 琥珀色边框 |
| All-in | `myPlayerStack` | 红色边框（激活时） |

**Raise Input 上限保护**：
- `Math.min(Number(e.target.value), myPlayerStack)` 防止超注
- `value={Math.min(raiseAmount || minRaiseTo, myPlayerStack)}` 显示 clamped 值

**结论**：实现正确，竞品标配功能已在 PC 端实现。移动端 `room-mobile/[id]` 尚未集成。
