### 三、Blast 优先级改进建议

#### P0（阻塞体验）

**1. BLAST 轮盘概率加权（P2-BLAST-001）**
- 当前：等概率随机（1/9 ≈ 11%）
- 应有：概率按稀有度分层（1X~2X 高频，1000X 极稀有）
- 参考方案：服务端计算 multiplier，投递到 Wheel 时直接使用
- 数据存储：`BlastSpin { id, tournamentId, playerId, multiplier, timestamp }`

**2. BLAST 匹配倒计时（P2-BLAST-002）**
- 当前：`MatchingOverlay` 无超时机制
- 应有：30秒倒计时 + 超时取消 + 自动退款
- 与 RoomEngine 联动：`lobbyExpiry` 事件 → 自动解散

#### P1（核心体验）

**3. BLAST 买入下限扩展（P2-BLAST-003）**
- GGPoker 最低 $0.25，CoinPoker 最低 $0.10
- 本项目最低 $5 偏高，建议增加 $1, $2 选项
- 低买入 → 用户获取成本低 → 提高赛事参与率

**4. BLAST 轮盘音效设计（P2-BLAST-004）**
- 旋转音效（渐快 + 渐慢）
- 停止音效（普通/稀有/ jackpot 三种）
- GGPoker 音效分层清晰，本项目完全无声
- 技术方案：Web Audio API + preload

**5. BLAST 奖池上限（Jackpot Cap）（P2-BLAST-005）**
- GGPoker $25 场最高 12,000X = $300,000 cap
- 本项目 $100 场 1000X = $100,000（已有上限）
- 建议：$5/$10 场 cap 100X，$25+ 场 cap 可更高
- Cap 作用：控制平台风险 + 制造"大奖"稀缺感

#### P2（优化体验）

**6. BLAST 赛事大厅统计（P2-BLAST-006）**
- 每级别显示：当前等待人数、平均等待时间
- 轮盘预展示（无需进入赛事即可查看倍数分布）
- 参考 GGPoker：点击买入级别 → 显示奖池轮盘 + 历史大奖

**7. BLAST 玩家槽位（3人→4人）讨论（P2-BLAST-007）**
- GGPoker 用 4人，本项目用 3人
- 3人优势：更快的游戏节奏 + 更强的相关性（每位玩家参与度高）
- 4人优势：更像传统德州 + 更多对抗空间
- 建议：**保持 3人设计**（差异化 + 速度优势），但需在 UI 上明确宣传

**8. BLAST 加速盲注结构（P2-BLAST-008）**
- 参考 GGPoker：第1级 2分钟 → 第2级 90秒 → 第3级 30秒
- 超时强制 all-in 决胜机制需明确
- 建议在 Room.tournamentConfig 中增加 `blindDurationSchedule`
