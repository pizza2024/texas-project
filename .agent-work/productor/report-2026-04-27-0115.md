## 本轮调研 — 竞品差异化：链上可验证洗牌（Verifiable Shuffle）

### 背景

本项目已有 USDT 充值 + HD 钱包充值地址生成（`deposit/` 模块），但**尚未利用区块链信任机制作为产品差异化点**。竞品 CoinPoker 已将"链上可验证"作为核心营销点。

### CoinPoker 的 Verifiable Shuffle 机制

CoinPoker 主页标注的核心卖点：
- **Blockchain-verified shuffles** — 手牌洗牌在链上可验证
- **Provably fair RNG** — 公开算法，玩家可验证随机性
- **Crypto-native** — USDT 存取，TRC20 网络

CoinPoker 的实现方式：
```
1. 服务端生成随机种子（commitment scheme: hash(random) 公开后reveal）
2. 客户端提供随机种子（client seed）
3. 洗牌 = SHA256(server_seed + client_seed)
4. 链上存储 hash(server_seed) 用于事后验证
```

### 建议：P2-VERIFIABLE-SHUFFLE 规格

**目标：** 在不改变当前游戏引擎架构的前提下，增加"可验证公平"模块。

```
Phase 1: 种子承诺系统（不影响现有游戏逻辑）
- 新增 /api/v1/verify/shuffle-commit 生成 hash(serverSeed)
- 新增 /api/v1/verify/reveal-seed 公开种子
- 新增 /api/v1/verify/verify-hand 验证历史牌局洗牌结果

数据模型扩展：
model ShuffleCommit {
  id          String   @id
  handId      String   @unique
  serverSeed  String   // 原始种子（仅事后公开）
  seedHash    String   // SHA256(serverSeed) — 公开的承诺
  clientSeed  String?  // 客户端种子
  revealedAt  DateTime?
  createdAt   DateTime
}
```

**与现有系统的边界：**
- 游戏引擎 `TableEngine` 保持不变
- `ShuffleCommit` 仅用于**事后验证**，不影响实时游戏
- 前端 UI：玩家可查看当前牌局的 seed hash，游戏结束后验证

**实施优先级：** P3（现有 P0/P1 已清零，可在下一迭代规划）
