### 核心发现：留存钩子矩阵

```
GGPoker 的留存机制：
├── Play Okay (新手引导) → 降低上手门槛
├── Fish Buffet (Rakeback 可视化) → 长期留存核心
├── Rush & Cash (类似 Zone Poker) → 高频快玩
├── Achievements + Badges → Gamification
└── Friend invites → 社交裂变

CoinPoker 的差异化：
├── 区块链Provably Fair → 信任建立
├── USDT 直充 → 0 摩擦充值
└── 匿名/低 KYC → 隐私友好

本项目的独特优势（竞品无或弱）：
├── USDT 充值/提现（法币平台无）
├── 区块链透明（vs GGPoker/Ignition 黑盒）
└── ELO 匹配（技术先进）
```

### 本项目留存问题根因分析

```
问题1: 无 Rakeback 可见性
→ 玩家不知道每次局后获得了什么好处
→ 长期玩家缺乏"工资"感知
建议: 添加 Rakeback 页面，展示本周/本月 Rake + 可兑换筹码

问题2: 无 Tutorial
→ 新用户首次进入房间后不知道操作
→ 高跳出率
建议: 首次进入牌桌时，Overlay 引导 highlight Fold/Check/Call/Raise 按钮

问题3: 无 Gamification
→ 缺乏短期成就目标
→ 纯游戏体验难以维持长期兴趣
建议: 添加简单徽章系统（首胜/百局/连胜5局等）

问题4: 好友邀请奖励缺失
→ 缺乏病毒式传播钩子
→ 只能靠 ELO 匹配结识陌生人
建议: Referral 奖励（双方各得 100 chips）
```

## 本轮新增发现

### 关键发现：Rakeback 是留存关键缺失

竞品分析显示 GGPoker 的 **Fish Buffet**（可视化 Rakeback）是其最强留存工具之一。玩家每次打完牌都能看到"我积累了 X Rake，下周可兑换 Y 筹码"。本项目完全没有 Rakeback 展示，这可能是长期玩家流失的一个原因。

**建议快速实现路径**：
- 计算公式：`rake = potSize × rakeRate`（可配置）
- 每周 Rake 累计展示
- 兑换比率（如 100 Rake = 1 chip）

### 关键发现：Tutorial 缺失影响新用户

本项目完全没有新手引导。对比 GGPoker Play Okay（互动式引导覆盖所有操作），新用户首次进入牌桌会面临"Fold/Check/Call/Raise 全是灰色，不知道该干什么"的情况。5档 ELO 房间也没有说明匹配逻辑。

**建议快速实现路径**：
- 首次进入 `/room/[id]` 时检测 `localStorage['has_seen_tutorial']`
- 无标记则展示半透明 Overlay，逐步 highlight 操作按钮
- 每个阶段配文字说明（3-4 步，30 秒完成）
