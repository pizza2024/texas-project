### GGPoker SPINS 规格分析（参考 P1-BLAST-001）

SPINS 是 GGPoker 的三人即时赛（Sit&Go变种），核心特征：

| 维度 | 规格 |
|------|------|
| 人数 | 3人 |
| 参赛费 | $0.25 / $1 / $3 / $5 / $10 / $25 |
| 随机奖池 | 2x-12,000x买入 |
| 时间 | 无报名费，30秒开赛 |
| 结构 | 初始筹码 500，单次重买 |
| 加速规则 | 盲注每3分钟涨，all-in或fold很快 |

**奖池分布（GGPoker SPINS）：**
- 50% 进入奖池
- 抽水 8%
- 奖池随机抽签（2x/6x/10x/120x/240x/12000x）

## 遗留 P2 问题评审

### P2 → P1 建议升档评估

#### 1. P2-JWT-LOCALSTORAGE（建议升 P1）

**风险:** JWT存 localStorage，任何XSS漏洞可直接盗窃token并劫持账户

**修复成本:** 中等
- 需修改 `packages/shared/src/auth.ts` 的 StorageAdapter
- 后端 `auth.service.ts` 需支持 httpOnly cookie
- 需处理 SSR（Next.js server components 不能访问 cookie）

**建议:** 升 P1，安全性优先

#### 2. P2-DEPOSIT-TOCTOU

**风险:** `getOrCreateDepositAddress` 并发请求可能分配相同 HD wallet index

**修复成本:** 低
- 使用唯一约束 + upsert 模式替代先查后创
- Prisma `upsert` 本身是原子操作

**建议:** 维持 P2，下一轮推进
