## P0 事项（已清零）

### P0-E2E-FIX-001 — rakeback.e2e.spec.ts 模块初始化失败 ✅ 已解决

**根因:** 多重问题叠加：
1. `AppModule` 循环依赖：`TableEngineModule → DepositModule → WebsocketModule`
2. `$transaction` mock 每次调用创建新对象，而非传递同一引用
3. `mockResolvedValue` 在创建时捕获值，而非 `mockImplementation` 每次重新读取

**修复方案（Test agent）：**
- 不再加载 `AppModule`，直接 mock `RakebackController` + `RakebackService` + `PrismaService`
- `$transaction: jest.fn().mockImplementation(async (cb) => cb(mock))` 传递同一引用
- `findUnique` 改用 `mockImplementation` 而非 `mockResolvedValue`

**测试结果:** 6/6 通过
```
✓ GOLD tier for 6000 totalRake
✓ BRONZE tier for 100 totalRake
✓ GOLD tier for 3000 totalRake
✓ 400 when rakeback balance is 0
✓ Successfully claim rakeback
✓ Prevent double-claim
```
