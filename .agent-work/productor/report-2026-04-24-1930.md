## 本轮竞品调研：移动端扑克 UX 设计

### 调研背景

本项目拥有 React Native (Expo SDK 55) 移动端。本轮聚焦**移动端扑克 UX 设计模式**，为移动端体验优化提供参考。

#### 移动端扑克 UX 核心设计模式

##### 1. 手势交互设计

| 手势 | 竞品实现 | 描述 |
|------|----------|------|
| 滑动弃牌 | WSOP, Zynga, 888poker | 水平滑动 = fold，卡片飞出动画 |
| 点击跟注/加注 | 全部 | tap 按钮操作 |
| 滑动加注 | PlayWSOP, ClubGG | 垂直滑动筹码条，快速调整加注额 |
| 长按查看手牌 | WSOP, Zynga | 按住手牌放大显示，适合暗牌场景 |
| 双指缩放 | ClubGG | 缩放牌桌视角 |

##### 2. 单手 vs 双手操作

主流移动扑克 app 分两派：

**单手派（拇指操作）**:
- Zynga Poker: 所有操作按钮集中在屏幕底部，单拇指可覆盖
- WSOP: 底部操作区，Bet/Fold/Check 三大按钮
- 设计原则：常用操作在屏幕下 1/3 区域

**双手派（需要双手持机）**:
- ClubGG: 两侧对称布局，需要双手操作
- 888poker: 更多功能按钮，需要双手配合

##### 3. 竖屏 vs 横屏

| 模式 | 竞品 | 优点 | 缺点 |
|------|------|------|------|
| 竖屏 | Zynga, WSOP, PlayWSOP | 单手操作，地铁/躺着用 | 视野受限 |
| 横屏 | 888poker（可选）, ClubGG | 视野更宽，信息更多 | 必须双手持机 |
| 自适应 | PlayWSOP | 竖屏聊天，横屏打牌 | 开发成本高 |

##### 4. 移动端专用功能

| 功能 | 竞品 | 说明 |
|------|------|------|
| 快速操作 | WSOP, Zynga | 一键全压、快速跟注按钮 |
| 手势密码 | 多数主流 | 图案解锁进入 app |
| 推送通知 | WSOP, 888poker | 轮到你时推送（后台也能收到） |
| 离线缓存 | ClubGG | 缓存牌局数据，断线重连不丢状态 |
| 省电模式 | Zynga | 降低动画帧率，延长续航 |

##### 5. 竞品移动端评分对比（App Store/Google Play）

| App | iOS | Android | 移动端特色 |
|-----|-----|---------|-----------|
| WSOP | 4.8★ | 4.7★ | 品牌授权 + 跨平台同步 |
| 888poker | 4.6★ | 4.5★ | 横屏模式，精准控制 |
| Zynga Poker | 4.7★ | 4.6★ | 单手操作，社交导入 |
| ClubGG | 4.5★ | 4.4★ | Club 社交 + 私人桌 |

### 移动端 UX 问题（需人工体验确认）

1. **盲区**: 目前不了解 `/room-mobile/[id]` 实际 UI 布局
2. **操作延迟**: WebSocket 延迟在移动端影响更大
3. **键盘行为**: 聊天输入时游戏状态是否冻结？
4. **后台通知**: 玩家离开 app 后还能收到轮到自己通知吗？

## 竞品调研小结

**移动端扑克 UX 核心洞察**:
- 单手拇指操作是主流（Zynga/WSOP），按钮集中在屏幕底部
- 滑动弃牌是行业标配，本项目缺失
- 竖屏优先是趋势，但横屏提供更宽视野
- 移动端推送通知是 DAU 提升的关键功能
- Club 系统（ClubGG 模式）已在 Club Phase 1 完成，移动端需同步开发

**本项目差异化**: Web3 原生 + 无 KYC + Club 社交，三位一体，移动端体验与竞品对齐后，差异化优势更明显。
