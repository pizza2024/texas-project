## 本轮调研主题：德州扑克游戏 UX 设计最佳实践

### 一、牌桌布局与信息层级

#### 竞品布局模式对比

| 竞品            | 牌桌形态     | 公共牌位置 | 动作栏位置 | 玩家信息            |
| --------------- | ------------ | ---------- | ---------- | ------------------- |
| **PokerStars**  | 椭圆形经典桌 | 顶部居中   | 底部固定   | 玩家头像+筹码在桌沿 |
| **WSOP**        | 圆形/椭圆形  | 顶部       | 底部大按钮 | 紧贴轮盘边缘        |
| **GGPoker**     | 3D 圆桌视角  | 顶部       | 右侧滑出   | 玩家列表侧栏        |
| **888poker**    | 经典椭圆     | 顶部居中   | 底部       | 简洁水平排列        |
| **本项目 (PC)** | 桌面布局     | 待确认     | 底部固定   | 牌桌上环绕          |

#### 关键 UX 发现

1. **Pot 大小展示**：所有竞品均将 Pot 放在桌面正中央、字号最大
2. **当前玩家高亮**：Active 玩家有明确的光标/箭头/发光指示
3. **动作按钮优先级**：Fold/Check/Call 三大按钮始终最醒目，Raise 是secondary
4. **倒计时视觉**：WSOP 使用红色脉冲，PokerStars 使用圆形进度环

#### 本项目 ActionBar 当前实现 (PC + Mobile 共用)

```

```

**当前按钮布局**：


**✅ 做得好的地方**：

- 结算阶段的 Show/Muck 选择

**⚠️ 可改进的地方**：

### 三、Mobile 端 UX 专项

#### 本项目 Mobile 页面

```
路径: apps/web/app/room-mobile/[id]/page.tsx
当前状态: 816 行，PC 和 Mobile 共用一套组件（ActionBar）
```

**Mobile 特有设计**：

- CardDisplay 使用更大字号（`w-16 h-[5.5rem]`）提升可读性
- 筹码飞行动画 (`CHIP_FLIGHT_MS = 620ms`, `CHIP_FLIGHT_STAGGER_MS = 70ms`)
- Payout 结算飞筹码动画
- 使用 `canvas-confetti` 庆祝效果

**竞品 Mobile UX 参考**：

| 竞品            | 手势设计      | 特点                       |
| --------------- | ------------- | -------------------------- |
| **WSOP Mobile** | 滑动弃牌/跟注 | 大按钮 + 手势双重确认      |
| **PokerStars**  | 点击为主      | 紧凑布局，滑动展开更多选项 |
| **GGPoker**     | 滑动 + 点击   | 右侧抽拉动作面板           |

**本项目 Mobile 改进建议**：

1. **滑动操作**：加入滑动手势（向左=Fold，向右=Call）作为高级选项
2. **单手模式**：移动端考虑将所有操作集中在屏幕一侧
3. **下注转盘**：类似 WSOP 的旋转下注转盘（高客制化需求，优先级低）
