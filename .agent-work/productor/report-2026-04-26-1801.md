### P2-JWT-LOCALSTORAGE XSS 风险升级建议

Test 代理建议将 `P2-JWT-LOCALSTORAGE` 升为 P1：

**理由：**
- JWT 存储在 localStorage 是经典 XSS 攻击向量
- 攻击者可通过 `eval(atob(token.split('.')[1]))` 伪造任意用户身份
- 本项目已有 HttpOnly cookie 方案（`@texas/shared/auth`），但前端可能未全面迁移

**建议 Coding 下一轮处理：**
1. 确认 `apps/web` 登录后 JWT 存储位置
2. 如使用 localStorage，迁移至 HttpOnly cookie
3. 或确认已有防护机制，关闭此 Issue

**理由：** 竞品全线具备此功能，属于基础体验。本项目 WebSocket 重连逻辑缺失导致移动端用户可能在比赛中意外断线。

**建议实现：**
1. Socket.io `reconnection` 事件监听
2. 重连期间 UI 显示"正在重连..."遮罩
3. 重连成功后恢复 `game-state`
4. 断线期间暂停出牌计时器

### P2-PUSH-NOTIFICATION（新增）

**理由：** 竞品全线具备推送通知（入座提醒、赛事开始、好友上线）。本项目 notification 模块已有 Resend 集成，可扩展到移动推送。

**建议方案：**
- 使用 Expo Notifications
- 触发场景：入桌邀请、好友申请、提现状态更新
