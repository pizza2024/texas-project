## Hand Replay Phase 2 详细规格（完善版）

> 延续上一轮草案，本轮补充 equity % 数据接口与后端改造方案

### 核心功能规格

#### P1 — 胜率曲线（Equity Curve）

**前端组件需求:**

- 在 `ReplayTimeline` 底部叠加 SVG/Canvas 折线图
- X 轴 = 时间轴节点（每 action 一个点）
- Y 轴 = 0-100% equity
- 每玩家一条线，颜色从牌桌玩家配色继承
- 悬停（hover）显示：`节点时间 | 玩家名: XX%`

**后端接口变更 — `GET /hands/:id/replay` 响应:**

```ts
interface ReplayNode {
  stage: "PREFLOP" | "FLOP" | "TURN" | "RIVER" | "SHOWDOWN";
  communityCards: string[]; // ['AH', 'KD', '7C', ...]
  pot: number; // 当前 pot 大小
  actions: Action[];
  // Phase 2 新增:
  playerEquity: Record<string, number>; // { userId: equityPercent }
  // 例如: { "user_abc": 35.2, "user_xyz": 64.8 }
}
```

**后端改造 — `hand-history.service.ts`:**

1. `buildReplayTimeline()` 每节点调用 `HandEvaluator.evaluate(communityCards, holeCards)` 计算各玩家胜率
2. 预计算所有节点 equity，避免前端实时计算开销
3. 缓存：`ReplayData` 结果可 Redis 缓存 1 小时

#### P1 — Pot Odds 计算器

**交互:**

- 点击时间轴任意节点 → 弹出浮窗
- 显示：Pot / Call Amount / Pot Odds % / 决策提示（COLOR/FD/OO/GP）
- 复用现有 `PotOddsCalculator` 服务（`table-engine/pot-odds.ts`）
- 关键节点（all-in / call > 50% pot）高亮标注

**浮窗布局:**

```
┌─────────────────────────────┐
│  Pot Odds Calculator        │
│  ─────────────────────────  │
│  Pot:     $1,200            │
│  Call:    $300              │
│  Odds:    25%                │
│  ─────────────────────────  │
│  你的牌: A♠ K♠               │
│  提示: 底池赔率充足，可跟注   │
└─────────────────────────────┘
```

#### P1 — 播放速度控制

- 按钮组：`0.5x` / `1x` / `2x` / `4x`
- 默认 `1x`
- 实现：前端 `setInterval` 调速，控制每节点展示时长
- 状态存储在 `ReplayContext`（React Context）

#### P2 — 自动播放

- 「自动播放」开关按钮
- 开启后：每节点停留当前速度对应时长，自动跳转下一节点
- 到达 `SHOWDOWN` 后停止，可选「循环」从头播放
- 进度条实时更新

#### P2 — 关键决策提示

- 在 all-in / big call 节点标注 💡 图标
- 浮窗显示：决策分析文字（如 "对手 all-in，你持有顶对，赔率计算显示 +EV"）
- 预置文案模板 + 动态赔率数据

## Tournament SNG Phase 1 详细规格（完善版）

### 游戏规则

| 字段     | 值                                      |
| -------- | --------------------------------------- |
| 参赛人数 | 固定 8 人                               |
| Buy-in   | 500 / 1000 / 5000 chips（从钱包扣）     |
| 座位     | 随机分配，1-8号                         |
| 开始条件 | 8人满 / 60秒倒计时（任意人数≥2）        |
| 盲注结构 | 固定 100/200，每 5 分钟涨盲（无涨盲表） |
| 时限     | 单局操作 30 秒超时                      |
| 淘汰     | 0 chips 立即离开桌子                    |

### 奖金分配

| 名次  | 分配     |
| ----- | -------- |
| 第1名 | 60% 奖池 |
| 第2名 | 30% 奖池 |
| 第3名 | 10% 奖池 |

### Phase 1 实施范围

**后端需要:**

1. `TournamentSngService` — 创建/加入/开始/结算
2. `TournamentSng` Prisma model — 独立于 `Room/Table`
3. `RoomType.SNG` 枚举值
4. 独立于 cash table 的 WebSocket handler
5. 自动出桌逻辑（0 chips → 淘汰）

**前端需要:**

1. Tournament Lobby 页面 (`/tournaments`)
2. Tournament 牌桌页面 (`/tournament/[id]`) — 可复用现有 `room/[id]`
3. 倒计时 UI（满8人或定时）

**不在 Phase 1 范围:**

- 重新参赛（淘汰后不可重新买入）
- 奖金领取 UI（自动结算到钱包）
- 排行榜
