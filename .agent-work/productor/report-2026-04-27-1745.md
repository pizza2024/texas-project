### Blast Phase 3 — 规格草案

**背景：** Blast 是 SNG（Sit & Go）变体，3人桌，2分钟基础时长，随机乘数（SpinWheel）决定奖池。

**当前 Phase 1-2 已实现：**

- Phase 1: BLAST Lobby + 创建房间 + 加入匹配 ✅
- Phase 2: 随机 SpinWheel 乘数 + 游戏逻辑 ✅

**Phase 3 缺失功能（待实现）：**

#### 3.1 `endsAt` 倒计时机制

```
场景：Blast 游戏开始时，server 计算 endsAt = startTime + BLAST_TOTAL_DURATION_MS

后端：
- Table.stateSnapshot.blastEndsAt: number (Unix ms)
- 每局结算后，若 now >= endsAt，游戏强制结束
- 游戏结束触发 `endBlastGame(force=true)`

前端：
- MatchingOverlay 显示倒计时（2:00 格式）
- BLAST 游戏中显示 "Time Left: 1:32"
- endsAt <= 30s 时，桌面红色边框闪烁警告
```

#### 3.2 超时惩罚机制

```
规则：
- BLAST_TOTAL_DURATION_MS = 120000ms（2分钟）
- 超时玩家自动 forfeit，游戏继续
- forfeit 记录到 HandAction，类型 FORFEIT_BLAST
- 连续 2 次超时 = 强制离开牌桌（kick）

UI 反馈：
- 超时前 10s：黄色警告 "即将自动弃牌"
- 超时后：红色提示 "你超时了"
```

#### 3.3 提前退款（Early Cashout）

```
场景：游戏剩 30s 时，玩家可选择"提前退出"拿保底

规则：
- endsAt - now <= 30000ms 时可用
- 保底金额 = entryFee × 0.5（固定 50% 返还）
- 点击"提前退出"后，立即结算离桌
- 剩余玩家继续，直到 endsAt

实现：
- WebSocket 新事件：`blast-early-cashout` (client→server)
- Server 检查：(endsAt - now) <= 30000ms && player.chips > 0
- 结算：chips -= entryFee × 0.5，退还钱包
```

#### 3.4 持久化与数据库模型

```
新增 Prisma 字段（Table 表）：
- blastStartedAt: DateTime?  // 游戏开始时间
- blastEndsAt:    DateTime?  // 强制结束时间
- blastMultiplier: Float?    // 本局 SpinWheel 乘数

HandAction 新增类型：
- FORFEIT_BLAST    // 超时判负
- EARLY_CASHOUT    // 提前退出

Settlement 新增标记：
- fromBlast: Boolean @default(false)
- cashoutMultiplier: Float?  // 提前退出乘数（0.5）
```
