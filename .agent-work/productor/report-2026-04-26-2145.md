## 二、竞品深度调研

### GGPoker SPINS 奖池随机算法机制

**什么是 SPINS？**
- 3人桌即时赛（Sit & Go），买入 $1/$3/$5/$10/$25/$50/$100
- 报名后奖池随机决定（入场时抽签）
- 最高可中 10,000x 奖池（$100 买入 → $1,000,000 奖池）

**奖池层级（USD）:**

| 买入 | 100x | 25x | 10x | 5x | 3x | 2x |
|------|------|-----|------|----|----|-----|
| $1 | $100 | $25 | $10 | $5 | $3 | $2 |
| $3 | $300 | $75 | $30 | $15 | $9 | $6 |
| $10 | $1,000 | $250 | $100 | $50 | $30 | $20 |
| $25 | $2,500 | $625 | $250 | $125 | $75 | $50 |

**随机算法设计推测：**
1. **种子生成：** 报名时服务端生成不可预测的随机种子（crypto RNG）
2. **链上参考（如 CoinPoker）：** 使用区块 hash 或 Chainlink VRF
3. **奖池概率分布（非均匀）：**
   - 2x-3x：高频（60-70%）— 保证小额返还
   - 5x-10x：中频（20-30%）— 中等刺激
   - 25x-10000x：低频（5-10%）— 头部大奖，制造病毒传播
4. **临界值：** $50/$100 档位才有 10,000x

**本项目参考价值：**
- P1-BLAST-001 (Blast 即时赛) 可借鉴此奖池设计
- 建议采用服务端种子 + 客户端可信随机（防作弊）
- 3人桌是标准配置，减少 AA 对决的无聊感

### PokerCraft 手牌分享技术实现

**功能描述：**
每局结束后生成一张高质量手牌图片，包含：
- 背景：牌桌局部截图（带品牌水印）
- 手牌：两张底牌（彩色、高清）
- 公共牌：5张公共牌（如有）
- 玩家信息：昵称、赢取筹码
- 赛事信息：盲注级别、赛事名称、时间戳

**技术实现方案（推测）：**

**方案 A: Canvas 服务端渲染**
```
1. Node.js canvas 模块在服务端渲染
2. 牌面使用 SVG 或 PNG sprite sheet
3. 渲染完成后上传 CDN，返回 URL
4. 前端直接展示图片 URL（无需前端 canvas）
优点：一致性好，防作弊（服务端控制）
```

**方案 B: 前端 Canvas 渲染**
```
1. 游戏结束后服务端下发牌面数据
2. 前端使用 <canvas> 渲染
3. 调用 Web Share API 分享（移动端）
   或 生成 blob URL 复制图片（PC端）
优点：减少服务端负载
```

**方案 C: 混合方案（推荐）**
```
1. 服务端生成基础图片（牌桌+手牌）
2. 前端叠加玩家昵称等动态信息
3. 减少网络传输，保留个性化
```

**竞品细节：**
- GGPoker PokerCraft：每局生成，48小时内可回放
- WSOP Hand Share：赛后分享到 Facebook/Twitter
- 分享卡片设计风格：暗黑主题 + 金色点缀

### CoinPoker 链上结算 vs 链下结算对比

**CoinPoker 简介：**
- 区块链德州扑克，CHP (CoinPoker Token) 为平台币
- USDT 充值 + CHP 结算

| 维度 | 链上结算 (CoinPoker) | 链下结算（本项目） |
|------|---------------------|------------------|
| 透明度 | 所有交易上链，可验证 | 依赖平台信誉 |
| 提现速度 | 依赖区块链确认（5-30分钟） | 即时（银行转账） |
| 手续费 | Gas 费用（波场链低） | 无（平台补贴） |
| 匿名性 | 伪匿名（不需 KYC） | KYC 实名 |
| 合规风险 | 监管灰色地带 | 更合规（地区依赖） |
| 抗审查 | 强（无单点故障） | 弱（平台可冻结） |
| 玩家信任 | 技术信任（代码开源） | 机构信任 |

**本项目定位：**
- 采用链下结算更符合当前监管环境
- USDT 充值（链上）+ 平台内流转（链下）= 混合模式
- 竞品优势：CoinPoker 对加密原生玩家有吸引力
