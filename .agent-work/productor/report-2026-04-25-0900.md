## 三、P1 设计：Rakeback Tier 扩展

### 3.1 扩展背景

当前 3 tier（BRONZE/SILVER/GOLD → 15%/22%/30%）与 GGPoker 差距达 30 个百分点。

### 3.2 目标模型：5 Tier

| Tier     | Rakeback | 日均 Rake | 累计条件           |
| -------- | -------- | --------- | ------------------ |
| BRONZE   | 15%      | $0+       | 注册即得           |
| SILVER   | 22%      | $500+     | 月累计抽水 $500    |
| GOLD     | 30%      | $2,000+   | 月累计抽水 $2,000  |
| PLATINUM | 40%      | $10,000+  | 月累计抽水 $10,000 |
| DIAMOND  | 50%      | $50,000+  | 月累计抽水 $50,000 |

### 3.3 实现要点

```
DB 变更:
- RakebackTier enum: BRONZE | SILVER | GOLD | PLATINUM | DIAMOND
- User.rakebackTier 字段（按月计算，非固定）
- User.monthlyRake 总计（每月 1 日重置）

计算逻辑:
- creditRakeback() 内按 tier 查表计算百分比
- Tier 升降判定: 每月 1 日跑 cron job
- PLATINUM+: 需要人工审核或自动+邮件通知

API:
- GET /user/rakeback → 返回 { tier, percentage, monthlyRake, nextTier, progress }
- 前端显示进度条: "距离 PLATINUM 还差 $8,000"

前端:
- /rakeback 页面: 5 tier 进度展示
- 升级动画 + 邮件通知
```
