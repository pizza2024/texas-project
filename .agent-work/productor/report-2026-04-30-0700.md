### GGPoker 提现 UX 详细拆解（最佳实践）

**Step 1 — 选择提现方式**
- 显示所有可用方式（Crypto / Bank / Wallet）
- 每种显示处理时间、费用、限额
- 不可用方式灰显+tooltip 说明原因

**Step 2 — 地址选择/输入**
- 下拉选择已存地址（标签+地址末4位）
- "Add New Address" 单独按钮（非表单必填）
- 新地址：标签（自定义）+ 网络（USDT-TRC20等）+ 地址验证
- 地址格式实时校验（截断/错误提示）

**Step 3 — 金额**
- 显示可用余额
- 滑块选择金额
- **净额公示**："你将收到 $980（已扣 $20 手续费）"
- Wagering 进度（若有）

**Step 4 — 确认**
- 摘要卡片：地址/金额/费用/净额/预计到账
- 邮件/SMS 验证码二次确认
- "提交"后即时反馈："请求已提交，预计 30 分钟内处理"

**Step 5 — 追踪**
- 提现记录页：每笔显示状态时间线
- 状态：Pending → Processing → Confirmed / Rejected
- Rejected 原因明确展示（可申诉）

### CoinPoker Crypto 提现 UX 亮点

1. **实时 Gas 估算** — 提交时显示当前网络费用
2. **TRON 20分钟确认** — 时间承诺明确
3. **链上自动通知** — txHash 发送后 WS push
4. **地址标签** — 避免复制粘贴错误

4. **Blast 模式:** SpinWheel 动画 + 匹配倒计时 ✅
5. **好友系统:** 申请/接受/拉黑 ✅

### ⚠️ 提现 UX 严重缺失（最高优先修复）
基于竞品分析，当前提现 UX 差距：

1. **无地址簿** — 用户每次提现需重新输入长地址（安全隐患）
2. **无手续费公示** — "净额 = 申请金额 - 手续费" 不可见
3. **无到账时间承诺** — 用户无法预期等待时长
4. **无提现历史** — 无法追踪过去提现记录和状态
5. **无 WS 推送** — 提现状态变更无实时通知

**参考 GGPoker 最佳实践，建议实施路径：**
- Phase 1: 地址簿 + 手续费透明（当前 P2-WITHDRAW-UX-001/002）
- Phase 2: 到账时间承诺 + 历史记录（P2-WITHDRAW-UX-003/004）
- Phase 3: 实时状态推送 + on-chain 追踪

### ⚠️ 房间 UX 待完善

2. **空状态引导** — 无房间时 CTA（"成为第一位玩家"或"创建房间"）
