## 二、扑克平台 UX 设计最佳实践

### 1. 专业扑克 UI 的核心要素

**视觉层级（GGPoker / WSOP 为例）：**

- 牌桌占据主视觉中心（60%+ 屏幕空间）
- 玩家信息以紧凑卡片形式围绕牌桌边缘
- 动作按钮大而醒目（最小 48x48dp touch target）
- 盲注/底池信息顶角固定显示，不干扰游戏

**竞品共性：**
| 要素 | GGPoker | WSOP | 888poker |
|------|---------|------|----------|
| 底池放大镜 | ✅ | ✅ | ✅ |
| 手牌历史浮层 | ✅ hover | ✅ | ✅ |
| 快速动作按钮 | Fold/Call/Raise 三键 | 同左 | BLAST 简化版 |
| 玩家计时器 | 红色倒计时 + 警告色 | 同左 | 加速模式更短 |
| 牌桌声音反馈 | 可开关 | 同左 | 同左 |

### 2. 玩家留存机制（行业标准）

**GGPoker 的留存体系：**

- **Daily Challenge** — 每日任务（完成X手牌/赢X个底池）
- **Achievement System** — 解锁式成就（首胜/连胜/特定牌型）
- **Rake Race** — 每周排行榜，top players 分成rake
- **Bad Beat Protection (GGCare)** — Bad Beat 时获赔奖金，降低挫败感

**WSOP 的社交留存：**

- **Bracelet/Ring 成就系统** — 21款收藏品，纯装饰性成就感
- **Clubhouse** — 玩家俱乐部，私人牌桌
- **Social Feed** — 手牌精彩瞬间分享

**888poker 的入门友好：**

- **不得弃牌惩罚** — 前20手不得弃牌（仅限新手）
- **Blitz Blackjack** — 跨游戏积分统一
- **SnapCam** — 表情包反应

### 3. 移动端 UX 关键差异

| 场景     | PC 设计          | 移动端设计              |
| -------- | ---------------- | ----------------------- |
| 动作按钮 | 底部居中，大尺寸 | 拇指区（屏幕下半部分）  |
| 玩家信息 | 横向展开         | 纵向折叠，点击展开      |
| 牌桌视图 | 铺开式           | 单桌沉浸式 + 缩放手势   |
| 聊天     | 侧边栏           | 收起式，@提及高亮       |
| 充值入口 | 顶部banner       | 浮动按钮 + 安全二次确认 |
