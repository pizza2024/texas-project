## 本轮主题：断线重连机制深度分析

### 发现：Reconnection 基础设施存在，但 Mobile 端未激活

**Socket.io 底层 reconnect 已配置：**

```typescript
// packages/shared/src/socket.ts
reconnection: true,
reconnectionDelay: 500,
reconnectionDelayMax: 5000,
reconnectionAttempts: 10,
```

**`visibilitychange` 前台检测已实现：**

```typescript
// 移动端切回前台时立即触发 reconnect
document.addEventListener("visibilitychange", () => {
  if (document.visibilityState === "visible" && socket && !socket.connected) {
    socket.connect();
  }
});
```

**`rejoin_available` 事件已定义（packages/shared/src/socket.ts:20）：**

```typescript
rejoin_available: (data: { roomId: string }) => void;
```

**问题：Mobile Room 页面未使用 `setRejoinAvailableHandler`**

检查 `apps/mobile/app/room/[id].tsx`：

- ✅ 监听 `room_update`（游戏状态同步）
- ✅ 监听 `left_room`（离开房间）
- ❌ **未注册 `setRejoinAvailableHandler`** — 断线重连后，客户端不知道可以回桌
- ❌ **未处理 `rejoin_available` 逻辑** — 无 UI 提示、无自动回桌

**对比 Web 端：** Web 端的 `socket-session-provider.tsx` 可能有更完整的处理，但 Mobile 完全缺失。

### 影响

| 场景               | 结果                                   |
| ------------------ | -------------------------------------- |
| 移动网络抖动       | 玩家被踢出牌桌                         |
| App 切后台超过 10s | socket 断开，玩家丢桌                  |
| 通话后切回游戏     | 需要手动重新加入房间，筹码状态可能丢失 |

**所有主流竞品（WSOP/GGPoker/888poker/CoinPoker）均标配断线重连。**

### 根因

`setRejoinAvailableHandler` 在 `mobile/lib/socket.ts` 已导出，但 `room/[id].tsx` 根本没有调用它。
