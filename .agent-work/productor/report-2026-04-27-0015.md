### GGPoker SPINS 机制

#### 什么是 SPINS

SPINS 是 GGPoker 2019年推出的彩票式单桌赛（Jackpot Sit-and-Go），3人桌，买入从 $1 到 $25 不等，奖池在比赛开始前**随机抽取**，最高可达买入的 12,000 倍（$25 买入最高 $300,000）。

#### 奖池分布（以 $1 SPINS 为例）

| 奖池倍数           | 概率   |
| ------------------ | ------ |
| 2×（$2）           | 75.00% |
| 5×（$5）           | 16.00% |
| 25×（$25）         | 5.00%  |
| 120×（$120）       | 2.50%  |
| 360×（$360）       | 1.00%  |
| 12,000×（$12,000） | 0.025% |

**期望值:** 约 1.15×（rake 约 13%）  
**中靶率:** 每 4,000 场有 1 场出现最高奖池

#### 速度变体

| 模式              | 盲注提速    | 推荐玩家 |
| ----------------- | ----------- | -------- |
| **SPINS**         | 标准        | 初学者   |
| **SPINS Hyper**   | 每 2.5 分钟 | 激进快速 |
| **SPINS Express** | 每 2 分钟   | 超快节奏 |
| **SPINS Super**   | 每 1.5 分钟 | 专业     |

#### SPINS 核心设计原则

1. **奖池随机性**: 比赛开始瞬间才揭示奖池，防止提前策略调整
2. **低 rake**: 相比普通 SNG，SPINS rake 约 8-10%
3. **高波动性**: 大量资金集中在小概率大奖
4. **快速节奏**: 10-20 分钟一局，适合移动端

### 关键差距分析

#### 1. Jackpot 累池 vs 即开型

本项目采用 **累池型 Jackpot**（每次买入抽取 % 进入奖池），而 GGPoker SPINS 是 **即开型**（奖池完全由当场比赛买入组成，随机放大）。

| 特性               | 累池型               | 即开型                 |
| ------------------ | -------------------- | ---------------------- |
| 大奖期望           | 稳定但较低           | 更高（但方差大）       |
| 玩家感知           | 累积感、期待感       | 即时惊喜               |
| 技术实现           | 需全局奖池状态       | 更简单                 |
| 对项目 Crypto 友好 | ✅（链上可验证累池） | ⚠️（纯随机需可信 RNG） |

**本项目建议:** 保留累池型，Blast Jackpot 已有规格优势（crypto 可验证透明性）。

#### 2. 奖池揭示时机

- **GGPoker SPINS:** 比赛开始前揭示奖池倍数
- **本项目:** 应在 "开始" 按钮点击后、盲注开始前揭示

**建议:** Blast 游戏开始流程：玩家准备 → 轮盘动画 → 揭示奖池倍数 → 盲注开始

#### 3. 速度变体缺失

本项目目前无速度变体（Hyper/Express）。考虑到移动端用户占比，**建议 Phase 2 Jackpot SNG 补充 Hyper 速度**：

- 盲注计时器从标准 60s → 30s
- 允许 late re-entry（延迟加入）
- UI 添加速度选择器（普通/Hyper）

#### 4. 第三方 HUD 可见性

| 问题                     | 建议                                    |
| ------------------------ | --------------------------------------- |
| SPINS 轮盘结果是否应公开 | **否** — 比赛开始前保密，防止多账号协作 |
| 玩家统计 HUD             | **赛后可见** — 避免实时情报优势         |
| 区块链验证奖池           | **加分项** — 链上随机数 + 可验证公平性  |
