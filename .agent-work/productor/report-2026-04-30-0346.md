### 二、竞品充值/提现 UX 对标

#### GGPoker

**充值：**
- 银行卡 / WebMoney / Skrill / Neteller / Crypto 多支付方式
- 即时到账（充值成功即显示余额）
- 最低充值 $10

**提现：**
- 银行转账为主（3-7 个工作日）
- 需要 KYC 验证
- 提现限额根据 VIP 等级不同

**关键 UX 亮点：**
- 充值界面显示「预计到账时间」和「手续费」
- 余额变动实时推送 WebSocket 通知

#### 888poker

**充值：**
- CREDIT CARD / DEBIT CARD / NETELLER / SKRILL / CRYPTO
- 即时到账
- 首充 bonus 弹窗引导

**提现：**
- 同样的支付方式（对称性）
- 提现页面显示「处理中」动画
- 提现确认邮件通知

**关键 UX 亮点：**
- 「充值引导」弹窗（首次充值时）
- 支付方式推荐（突出显示最佳选项）

#### CoinPoker（Crypto 原生）

**充值：**
- 内置 Web3 钱包（MetaMask / WalletConnect 一键连接）
- TRC20 / ERC20 双链支持
- 链上确认即自动到账（无人工审核）

**提现：**
- 30 分钟平均到账时间（核心竞争力）
- 实时链上状态追踪
- 透明费率（无隐藏费用）

**关键 UX 亮点：**
- **地址簿功能**：保存常用提现地址，再提现无需重复输入 ✅
- **充值确认动画**：链上确认后播放成功动画（不是单纯文字）
- **TRX 能量费说明**：Gas 费说明，帮助用户理解链上成本

### 四、P2-DEPOSIT-UX / P2-WITHDRAW-UX 规格建议

基于竞品分析，以下是建议优先实施的 UX 改进：

#### P2-DEPOSIT-UX-001：充值成功确认动画

**当前问题：** 充值到账后无明确反馈，用户需手动检查余额。

**建议方案：**
- 区块确认完成后（WebSocket 通知），显示 ✅ 绿色成功动画
- 显示：到账金额 / 汇率 / 预计到账时间
- 自动刷新余额

**竞品参考：** CoinPoker — 链上确认后播放绿色光效动画

#### P2-DEPOSIT-UX-002：充值地址簿（可选）

**当前问题：** 用户每次充值需复制/粘贴长地址。

**建议方案：**
- 保存用户最近 3 个充值地址（本地存储）
- 下拉选择 + 支持新增
- 首次充值默认当前地址

**竞品参考：** CoinPoker 地址簿

#### P2-DEPOSIT-UX-003：充值限额与手续费说明

**当前问题：** 无最低/最高充值限额说明，用户可能在链上损失手续费后才发现不够。

**建议方案：**
- 在地址卡下方增加「充值说明」区：
  - 最低充值：10 USDT
  - 链上 Gas 费由用户承担（约 1-5 USDT）
  - 预计到账时间：2-5 分钟（3 个区块确认）

**竞品参考：** GGPoker 充值页「预计到账」

#### P2-WITHDRAW-UX-001：提现地址簿

**当前问题：** 提现地址每次需手动输入，容易出错。

**建议方案：**
- 支持保存多个提现地址（数据库存储）
- 地址标签（如「我的 TP 钱包」）
- 提现历史中可直接「复用」上次地址

#### P2-WITHDRAW-UX-002：提现手续费/限额说明

**当前问题：** 提现页面无手续费说明，用户对实际到账金额不清楚。

**建议方案：**
- 提现前显示：「手续费：X USDT，预计到账：Y USDT」
- 显示每日/每月提现限额
- VIP 等级影响限额

#### P2-WITHDRAW-UX-003：提现进度追踪

**当前问题：** 提现提交后状态不透明，用户不知道进展。

**建议方案：**
- PENDING → PROCESSING → ON-CHAIN → CONFIRMED 完整状态流
- 每个状态显示时间戳
- PROCESSING 阶段显示预计完成时间
- 链上 txHash 出来后直接链接到 explorer

#### P2-WITHDRAW-UX-004：提现二次确认

**安全建议：**
- 提现前显示：「您即将提现 X USDT 到地址 0x...」
- 建议绑定 Google Authenticator 二次验证（大额提现）
