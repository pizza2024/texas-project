### 代码走查：桌内移动端适配

检查 `apps/web/app/room/[id]/page.tsx` 发现移动端自动跳转逻辑：

```ts
const isMobile =
  /Android|webOS|iPhone|iPad|iPod|BlackBerry|IEMobile|Opera Mini/i.test(
    navigator.userAgent,
  ) ||
  (navigator.maxTouchPoints > 1 && window.innerWidth < 1024);
if (isMobile) {
  router.replace(`/room-mobile/${id}`);
}
```

**评估**：移动端检测逻辑健壮，区分 `room/[id]` (PC) 和 `room-mobile/[id]` (Mobile) 两套 UI 路径。但 `maxTouchPoints > 1` 可能误判桌面触控屏用户。**P3 优先级**建议。
