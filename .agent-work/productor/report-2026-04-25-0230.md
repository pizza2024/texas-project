### P0-NEW-001 深度分析

**问题确认**：
```ts
// auth.service.ts:5
import { TooManyRequestsException } from '@nestjs/common'; // ❌ undefined

// auth.service.ts:185
throw new TooManyRequestsException('Too many failed attempts...');
```

**验证方法**：
```bash
node -e "const m = require('@nestjs/common'); 
console.log('TooManyRequestsException:', typeof m.TooManyRequestsException)"
# 输出: undefined

# 对比
node -e "const m = require('@nestjs/common'); 
console.log('BadRequestException:', typeof m.BadRequestException)"
# 输出: function ✅
```

**根本原因**：`@nestjs/common@11.1.14` 版本未导出 `TooManyRequestsException`，这是一个 NestJS 框架层的问题，而非业务代码逻辑问题。

**修复方案**（3种）：
| 方案 | 优点 | 缺点 |
|------|------|------|
| 1. 替换为 `BadRequestException` | 简单，零依赖 | HTTP 状态码不同（400 vs 429） |
| 2. 自定义 `RateLimitException` | 语义准确，可携带 retry-after | 需要创建新异常类 |
| 3. 使用 `HttpException` 手动构造 429 | 无需新增导入 | 代码略冗长 |

**推荐**：方案2 — 创建自定义 `RateLimitException`，语义准确且可携带 `retryAfter` 信息。

**风险评估**：生产部署会失败（P0 阻塞），必须本轮或下轮修复。
