## 竞品调研 — 移动端操控交互深度分析

### 本轮主题：移动端加注交互对比

移动端德州扑克的核心交互差异在于**加注（Raise）控制**的设计。竞品对此投入了大量 UX 优化。

#### 竞品加注交互模式

| App            | 加注控制方式                                | 特色               |
| -------------- | ------------------------------------------- | ------------------ |
| **PokerStars** | 滑块 + 预设按钮（1/2 pot, 3/4 pot, All-in） | 快速选择，无需拖动 |
| **WSOP**       | 滑块 + 数字输入                             | 支持键盘精确输入   |
| **888poker**   | 滑块 + 快速按钮（Min, 1/2, 2x, All-in）     | 5档快速选择        |
| **CoinPoker**  | 纯滑块 + 预设                               | 简洁但功能完整     |

#### CHIPS 当前实现

当前 `room-mobile/[id]/page.tsx`（758行）的加注控制：

```jsx
// Step-down / Step-up 按钮
<button onClick={() => setRaiseAmount(Math.max(minRaise, raiseAmount - step))}>−</button>
<input type="range" min={minRaise} max={myPlayer?.stack} />
<button onClick={() => setRaiseAmount(Math.min(stack, raiseAmount + step))}>+</button>
<div>${fmt(raiseAmount)}</div>
```

**现状评估**：

- ✅ 基础滑块控制已实现

- ✅ 触控区域（h-14 buttons, h-3 slider）基本合理

- ❌ **无最低加注/最大加注快捷入口**

## 竞品加注交互最佳实践

### 推荐的移动端加注 UI

参考 888poker / PokerStars 的设计：

```
┌──────────────────────────────────────────┐
│  FOLD      CHECK/CHECK   CALL $45        │
├──────────────────────────────────────────┤
│  [Min $10] [1/2 $22] [All-In] [RAISE ▶]  │  ← 快速按钮行
│  ━━━━━━━━━●━━━━━━━━━━━━━━━━━━━━━━━      │  ← 滑块
│            Raise: $45                    │
└──────────────────────────────────────────┘
```

**关键设计点**：

1. **Min / 1/2 Pot / All-In** 三档快速按钮，减少滑块拖动
2. **滑块精确定位**到按钮对应位置
3. **All-In 按钮醒目**（红色背景）
4. **同时支持滑块和按钮**，给用户选择权
