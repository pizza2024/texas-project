### 本项目 Phase 3 规格建议

#### 1. 延迟注册（Late Registration）

**数据库扩展（Room/MTTConfig）：**

```prisma
model MttConfig {
  id                  String   @id @default(uuid())
  roomId              String   @unique
  lateRegLevels       Int      @default(3)    // 允许注册的最后级别
  lateRegCutoff       DateTime?              // 或用级别数计算截止时间
  allowLateReg        Boolean  @default(true)
}
```

**逻辑实现：**

- 注册时检查当前级别 ≤ `lateRegLevels`
- 延迟注册玩家起始筹码 = `startingChips * lateRegStackPercent(level)`
- 延迟注册金 = 报名费（无额外费用，保持公平性）

#### 2. Re-Entry

**数据库扩展：**

```prisma
model MttEntry {
  id          String   @id @default(uuid())
  handId      String?
  reEntryNum  Int      @default(0)  // 0=首次，1=第一次重买，...
  isActive    Boolean  @default(true)
}
```

**逻辑实现：**

- 淘汰时 `isActive = false`，允许重新买入
- `reEntryNum++`，生成新 entry
- 最多重买次数：`mttConfig.maxReEntries`（0=禁止，null=无限）
- 多-entry 同时排名，只取最优名次

#### 3. 暂停机制（Pause）

**状态机扩展：**

```typescript
enum MttState {
  // ... existing
  PAUSED = "PAUSED", // 新增
  BREAK = "BREAK", // 新增
}
```

**暂停规则：**

- 私人 MTT：房主可随时暂停（最多 N 分钟）
- 公众 MTT：仅在 break 时可暂停
- 暂停时所有玩家 action 超时计时器暂停
- 暂停广播：`mtt-pause-state` WS 事件

#### 4. 多桌管理（Multi-Table）

**分桌规则：**

- 每桌人数：`mttConfig.seatsPerTable`（常见 6/8/9/10）
- 桌数 = `ceil(remainingPlayers / seatsPerTable)`
- 新桌成立条件：人数达到 `seatsPerTable - 1` 时拆分
- 牌桌分配算法：随机 + 桌平衡（避免某桌过强）

**合并规则：**

- 有人淘汰时触发合并检查
- 合并优先：同级别 + 筹码量相近的桌
