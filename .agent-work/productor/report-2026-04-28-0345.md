## 本轮调研 — Run It Twice（双开牌）机制深度分析

### 什么是 Run It Twice？

当所有玩家全下后，为降低运气成分，允许**发两次公共牌**（flop/turn/river 各发两份），**底池均分到两次开牌中**，各算一次独立结果后合并。

### 竞品实现对比

| 竞品       | Run It Twice | 说明                        |
| ---------- | ------------ | --------------------------- |
| GGPoker    | ✅           | 最低$0.25/$0.50，部分高额桌 |
| PokerStars | ✅           | 部分房间可开启              |
| 888poker   | ✅           | 仅特定 Tournament           |
| WSOP       | ❌           | 主打现场感，禁用此机制      |
| CoinPoker  | ❌           | 注重可验证随机性            |

### Run It Twice 的产品价值

**核心价值 — 降低波动，提升游戏体验**

- 运气成分减少约 50%（两次独立结果）
- 玩家更愿意参与高额桌（心理压力降低）
- 减少 bad beat 投诉（运气摊薄）
- 增加游戏时长（玩家留存率↑）

**商业价值 — Rake 增加**

- 每次 Run It Twice = 多发一套公共牌
- Pot size 通常更大（心理障碍降低后玩家下注更激进）
- Rake 收入增加 15-25%（行业估算）

### 技术实现分析

```
已全下 → 检测 ALL_IN → 确认 RUN_IT_TWICE 开关
  → 复制当前 pot
  → 第一次开牌 → 结算 half-pot
  → 第二次开牌 → 结算 half-pot
  → 合并到各玩家 chip stack
```

关键实现点：

1. `Table.stateSnapshot` 需要复制两份
2. `HandAction` 记录需要区分两次开牌
3. `Settlement` 两条记录（playerId + handId + amount）
4. UI 需要展示"两次开牌"结果（双排牌面）
