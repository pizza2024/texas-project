## 本轮调研 — 手牌渲染方案分析

### 现状

当前 `ReplayPlayerCards.tsx` 使用 **ASCII 文本渲染**扑克牌：

```tsx
// 花色映射 — 使用字母而非符号
const suit = card.slice(-1);  // 'h','d','c','s'
// 颜色: 红桃h/方块d = 红色, 黑桃c/梅花s = 深色
```

**优点:** 零依赖，渲染快，跨平台一致
**缺点:** 与 GGPoker/WSOP 的精美卡牌图形相比视觉粗糙

### 竞品手牌渲染方案对比

| 平台 | 渲染方案 | 特点 |
|------|---------|------|
| **GGPoker** | Canvas + 精灵图 | 3D 卡牌翻转动画，触感反馈 |
| **WSOP (PlayStation)** | 自研渲染引擎 | 卡牌有厚度感，阴影层次丰富 |
| **CoinPoker** | SVG 组件 | 轻量，中等视觉质量 |
| **ACR** | PNG 精灵图 | 经典风格，无动画 |

### 本项目手牌渲染评估

| 维度 | 当前状态 | 建议 |
|------|---------|------|
| 牌面图形 | ASCII 字母 (h/d/c/s) | 升级为 suit 符号 (♥/♦/♣/♠) 或 SVG |
| 背面样式 | 渐变 + emoji 🎴 | 良好基础，可加深纹理 |
| 翻转动画 | CSS rotateY | 已实现 Phase 1，可接受 |
| 社区牌 | 同样 ASCII | 统一升级 |

### 建议优先级

1. **快速修复（不影响功能）:** 将 `h,d,c,s` 花色字母替换为 Unicode 符号 `♥/♦/♣/♠`，颜色对齐红黑
2. **中期提升:** 添加卡牌阴影和圆角，提升质感
3. **长期规划:** 考虑 Canvas 或 SVG 方案用于手牌和社区牌
