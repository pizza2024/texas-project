## 竞品调研：Club 系统深度分析

### Club 系统的重要性

Club（俱乐部）系统是扑克游戏 DAU 增长的核心驱动力。参考 Zynga Poker 和 ClubGG 的实践：

#### Club 核心功能矩阵

| 功能 | Zynga | ClubGG | 参考实现难度 |
|------|-------|--------|------------|
| 私人俱乐部创建 | ✅ | ✅ | 中 |
| 俱乐部排行榜 | ✅ | ✅ | 低 |
| 俱乐部专属房间 | ✅ | ✅ | 中 |
| 俱乐部成员管理 | ✅ | ✅ | 低 |
| 俱乐部聊天 | ✅ | ❌ | 低 |
| 俱乐部邀请码 | ✅ | ✅ | 低 |
| 俱乐部锦标赛 | ❌ | ✅ | 高 |
| 俱乐部等级/声望 | ✅ | ✅ | 低 |
| 会员专属福利 | 部分 | ✅ Gold | 中 |

#### Club 留住用户的机制

1. **社交绑定**: 俱乐部成员形成固定社交圈，迁移成本高
2. **归属感**: 俱乐部徽章、等级、排行榜满足荣誉感
3. **私人空间**: 与陌生人桌不同，俱乐部是"熟人圈"
4. **内容隔离**: 俱乐部房间、锦标赛形成独立内容体验

#### Club 盈利模式（ClubGG 参考）

| 等级 | 价格 | 权益 |
|------|------|------|
| Free | $0 | 创建1个俱乐部，5个成员 |
| Gold | ~$10/月 | 无限俱乐部，无限成员，录像回放，优先匹配 |

## Club 系统产品设计建议

### 核心功能优先级

| 优先级 | 功能 | 说明 | 实现复杂度 |
|--------|------|------|-----------|
| P0 | 俱乐部 CRUD | 创建、编辑、删除俱乐部 | 低 |
| P1 | 俱乐部成员管理 | 邀请码、踢出、角色（Owner/Admin/Member） | 低 |
| P1 | 俱乐部房间 | 俱乐部专属私人桌 | 中 |
| P2 | 俱乐部排行榜 | 按贡献/胜率排名 | 低 |
| P2 | 俱乐部聊天 | 俱乐部内部聊天 | 中 |
| P3 | 俱乐部锦标赛 | 定期举办的内部锦标赛 | 高 |
| P3 | 俱乐部等级 | Level 系统 + 专属标签 | 低 |

### 数据模型建议

```prisma
model Club {
  id          String   @id @default(uuid())
  name        String
  ownerId     String
  description String?
  avatarUrl   String?
  level       Int      @default(1)
  memberCount Int      @default(0)
  createdAt   DateTime @default(now())
  
  members     ClubMember[]
  rooms       Room[]
  chatMessages ClubChat[]
}

model ClubMember {
  id       String   @id @default(uuid())
  clubId   String
  userId   String
  role     String   @default("MEMBER") // OWNER, ADMIN, MEMBER
  joinedAt DateTime @default(now())
  
  club Club @relation(fields: [clubId], references: [id])
  user User @relation(fields: [userId], references: [id])
  
  @@unique([clubId, userId])
}

model ClubChat {
  id       String   @id @default(uuid())
  clubId   String
  userId   String
  content  String
  createdAt DateTime @default(now())
  
  club Club @relation(fields: [clubId], references: [id])
}
```
