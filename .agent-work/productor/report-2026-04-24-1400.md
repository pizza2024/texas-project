## 竞品调研 — Achievement / Quest 系统专项分析

### 主流平台 Achievement/Quest 特性

| 平台           | 系统名称                            | 核心机制                           | 奖励类型                |
| -------------- | ----------------------------------- | ---------------------------------- | ----------------------- |
| **WSOP**       | Challenges + Player Awards          | 赛事参与、牌局成就、社交分享       | Bracelet 虚拟展示、奖金 |
| **GGPoker**    | Achievement System + Daily Missions | 狩猎任务(Hunt)、日常任务、每周挑战 | 奖金、门票、rakeback    |
| **PokerStars** | StarsChest Rewards                  | Chest 开启，积分(StarsCoin)换奖励  | 赛事门票、现金、周边    |
| **888poker**   | Mission System + Club               | 每日/每周/每月任务，rakeback       | 奖金、赛事门票、实物    |
| **CoinPoker**  | Achievement Hangar                  | 成就徽章+里程碑，CHP 代币奖励      | CHP 代币、rakeback      |

### Achievement 系统分类

#### 1. Volume-Based（量驱动型）

```
✅ 最容易实现，适合所有玩家
- "完成 100 手牌"
- "累计赢得 10,000 筹码"
- "每周参与 5 个不同房间"
- "连续登录 7 天"
```

#### 2. Skill-Based（技术驱动型）

```
✅ 提升游戏深度，适合核心玩家
- "Royal Flush（皇家同花顺）"
- "用一对赢下底池"（Bad Beat）
- "河牌反转（River Quiz）"
- "连续 3 手牌加注"
```

#### 3. Social（社交驱动型）

```
✅ 提升留存，适合社交玩家
- "添加 5 个好友"
- "赠送筹码给好友"
- "分享牌局到社区"
- "参加俱乐部赛事"
```

#### 4. Milestone（里程碑型）

```
✅ 长期目标，锁住玩家
- "游戏时长达到 100 小时"
- "累计盈利达到 $1,000"
- "赢得第一个 Tournament"
- "ELO 达到 2000"
```

### Quest/Mission 结构设计

#### 每日任务（Daily Quest）

```typescript
interface DailyQuest {
  id: string;
  type: "volume" | "skill" | "social";
  title: string; // "完成 20 手牌"
  description: string; // "在任意房间完成至少 20 手牌"
  target: number; // 20
  progress: number; // 当前进度
  reward: {
    type: "chips" | "ticket" | "bonus";
    amount: number;
  };
  expiresAt: Date; // 每日 UTC 0 重置
  status: "available" | "completed" | "expired";
}
```

#### 成就徽章（Achievement Badge）

```typescript
interface Achievement {
  id: string;
  category: "volume" | "skill" | "social" | "milestone";
  name: string; // "Royal Flush"
  description: string; // "在任意牌局中完成皇家同花顺"
  icon: string; // emoji or asset path
  rarity: "common" | "rare" | "epic" | "legendary";
  condition: JsonLogic; // 触发条件
  reward: {
    type: "chips" | "title" | "rakeback";
    amount: number;
  };
  unlockedAt?: Date; // 解锁时间
}
```

### Quest UX 设计要点

#### 1. Quest 面板位置

```
推荐位置：
├── 游戏桌内：右下角 "Quest" 快捷入口（浮动按钮 + 红点）
├── 侧边栏：独立 "Quest" Tab（Rooms 页面侧边）
└── 个人中心：顶部 Banner + 独立 Tab
```

#### 2. Quest 进度展示

```
♠️ 每日任务
├── 📊 完成 20 手牌         [████████░░] 16/20  ⏳
├── 🏆 赢得 5 个底池        [██░░░░░░░░] 2/5    ⏳
└── 🎁 每日登录奖励        [✅] 可领取!         ← 点击领取

✨ 成就进度
├── Royal Flush            [  ] 未解锁
├── Bad Beat Hero          [  ] 未解锁
└── Social Butterfly       [██░░░░░░░░] 好友 2/5
```

#### 3. 奖励领取交互

- **可领取**：绿色脉冲动画 + "领取" 按钮
- **未完成**：灰色锁住 + 进度条
- **已领取**：灰色对勾 + 日期

#### 4. 通知触发时机

```
触发点：
├── 登录时：推送 "今日任务" Banner
├── 任务完成：Toast "任务完成! +500 筹码"
├── 成就解锁：全屏祝贺动画（稀有成就）
└── 每日重置：凌晨 UTC 0 重置 + 推送新任务
```

## 建议 Achievement/Quest 系统分级

### Phase 1 (MVP - 最低复杂度)

```typescript
// 数据库模型
model Achievement {
  id          String   @id @default(uuid())
  category    String   // volume | skill | social | milestone
  name        String
  description String
  icon        String
  rarity      String   // common | rare | epic | legendary
  condition   Json     // {type: "hands_played", target: 100}
  rewardType  String   // chips | ticket | rakeback
  rewardAmount Int
  createdAt   DateTime @default(now())
}

model UserAchievement {
  id             String    @id @default(uuid())
  userId         String
  achievementId  String
  progress       Int       @default(0)
  unlockedAt     DateTime?
  @@unique([userId, achievementId])
}

model DailyQuest {
  id          String   @id @default(uuid())
  userId      String
  type        String   // volume | skill
  title       String
  description String
  target      Int
  progress    Int      @default(0)
  rewardChips Int
  expiresAt   DateTime
  completed   Boolean  @default(false)
  @@index([userId, expiresAt])
}
```
