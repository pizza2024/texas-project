### 核心修复：轮盘概率加权（P2-BLAST-001）

**现状问题：**
`SpinWheel.tsx` 第16-26行硬编码了9个等概率段，每段 11.1%。注释写的是加权方案，但代码未实现。

**GGPoker 实际概率参考：**
- 2x: ~70-80%（压倒性多数）
- 5x: ~10-15%
- 10x: ~5-8%
- 25x: ~2-4%
- 100x: ~0.5-1%
- 1000x: ~0.1%
- 10000x: 极稀有

**修复方案（服务端实现）：**

```typescript
// Backend: 确定获奖倍率（randomWeightedMultiplier）
// 使用加权随机，服务端唯一真相
const WEIGHTED_RANGES = [
  { max: 2,  weight: 75 },  // 75% → 2x
  { max: 5,  weight: 13 },  // 13% → 3x/5x
  { max: 10, weight: 6 },   // 6%  → 10x
  { max: 25, weight: 3 },   // 3%  → 15x/25x
  { max: 50, weight: 1.5 }, // 1.5%→ 50x
  { max: 100,weight: 0.8 },  // 0.8%→ 100x
  { max: 1000,weight: 0.2 },// 0.2%→ 1000x (Jackpot)
];
```

**前端适配（SpinWheel.tsx）：**
- 概率分段在服务端生成 `PrizeResult`，前端只负责展示
- `SpinWheel.tsx` 保留 UI，接受 `segments[]` 和 `targetIndex` props
- 移除前端硬编码的概率逻辑
- 动画时间保持 4s，通过 ease 曲线模拟真实转盘停止感

### 次级修复：匹配倒计时（P2-BLAST-002）

**现状：** `MatchingOverlay.tsx` 只显示脉冲动画，无倒计时。

**修复方案：**
- 前端：`MatchingOverlay` 接收 `countdownSeconds` prop（默认30）
- 30秒无响应 → WebSocket `matchmaking_timeout` 事件 → 前端显示"匹配超时" → 自动返回大厅
- 后端：新增 `MatchmakingService` 30秒 TTL 逻辑（独立模块，可与 P2-BLAST-001 并行实施）
