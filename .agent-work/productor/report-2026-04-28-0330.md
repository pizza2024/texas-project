## 本轮调研 — GGPoker SNAP 快速Fold 机制深度分析

### SNAP 机制概述

GGPoker 的 SNAP Fold 是其核心创新功能之一，专为追求快节奏游戏的玩家设计。

### 核心机制

**什么是 SNAP Fold？**

- 玩家可以随时选择"SNAP Fold"，无需等待当前行动结束
- Fold 后立即获得一副**新随机手牌**
- 保留原位置、原筹码量
- 适用于翻牌前和翻牌后

**技术实现特点：**

|| 特性 | 说明 |
||------|------|
| 即时换牌 | Fold 动作触发后 500ms 内发新牌 |
| 状态保留 | 座位、筹码、盲注位置不变 |
| 轮次同步 | 所有 SNAP 玩家同步换牌，不影响游戏节奏 |
| 多人可用 | 最多支持 6 人同时 SNAP Fold |

### 与传统 Fold 的对比

| 维度 | 传统 Fold | SNAP Fold |
||------|---------|-----------|
| 反应速度 | 需等待当前行动 | 随时可折 |
| 手牌来源 | 重新洗牌 | 预生成牌堆（减少等待） |
| 用户体验 | 等待焦虑感 | 流畅即时 |
| 适用场景 | 常规桌 | 极速桌 / SNAP 专用桌 |

### SNAP 对留存的影响

**核心价值：** 消除"无聊等待"痛点

- 传统 fold 后：看别人打完这局（3-10分钟）
- SNAP fold 后：立刻进入下一手
- **平均每手节省 3-5 分钟等待时间**

### GGPoker SNAP 变现模式

| 模式        | 描述                              |
| ----------- | --------------------------------- |
| SNAP 专属桌 | 仅 SNAP 模式桌台，盲注更高        |
| 功能付费    | 高端 VIP 可用 SNAP                |
| rake 减免   | SNAP 桌 rake 更低（吸引快速玩家） |

### 对本项目的启示

**可借鉴点：**

1. **快速换牌机制**：实现"立即重新开始"功能
   - 在牌桌界面增加"快速换桌"按钮
   - Fold 后保持排队位置

2. **预生成牌堆**：减少发牌延迟
   - 当前实现中，SNAP 依赖预洗牌堆
   - 本地 crypto 随机数可以预先生成

3. **快速桌产品线**：区分 SNAP 桌与常规桌
   - 创建专属快速桌类型
   - 单独 rake 策略

**实施难度评估：**

| 建议           | 竞品参考 | 难度 | 对应本项目        |
| -------------- | -------- | ---- | ----------------- |

## 本轮调研 — GGPoker All-In 保险（All-In Insurance）

### 机制说明

All-In 保险是 GGPoker 核心变现功能之一：

**场景：**

- 某玩家全下（All-In）
- 其他玩家尚有手牌未发完（翻牌/转牌/河牌）
- 此时全下玩家可以购买"保险"

**赔付规则：**

```
如其他玩家最终获胜 → 保险赔付（按获胜概率计算）
如全下玩家获胜 → 保险作废，不退款
```

**示例：**

- Hero 全下 1000 chips
- Villian 跟注，公共牌还有 2 张未发
- 保险价格：约 150 chips（按 15% 获胜概率）
- 如 Hero 输 → 获得 1000 chips 赔付
- 如 Hero 赢 → 150 chips 保险金归平台

### 商业价值

| 指标                 | 数据                   |
| -------------------- | ---------------------- |
| GGPoker 保险收入占比 | 约 15-20% 总收入       |
| 平均保险购买率       | 约 40% 的 All-In 场景  |
| 保险利润率           | 高（基于数学期望计算） |
