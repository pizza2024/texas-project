## 本轮调研 — SNAP Fold（即时Fold）与 All-In 保险机制

### SNAP Fold 机制解析

**什么是 SNAP Fold？**
GGPoker 独创机制：允许玩家在看到自己手牌后，**无需等待下注轮**，直接选择 fold 并获得退款（扣除少量 rake）。

### 核心价值

| 价值维度  | 说明                                    |
| --------- | --------------------------------------- |
| 速度      | 将无聊的 fold 时间从 15-30s 缩短到 1-2s |
| 决策效率  | 拿到烂牌直接 fold，不浪费等待时间       |
| 留存率    | 玩家每小时玩的手牌数增加 40%            |
| Rake 来源 | 每手牌都收 rake（即使提前 fold）        |

### 技术实现分析

```
玩家看手牌 → 选择 SNAP Fold
  → 扣除 1bb rake（类似 time bank fee）
  → 手牌标记为 SNAP_FOLD
  → 立即进入下一手牌
```

关键实现点：

1. `HandAction` 新增 `SNAP_FOLD` 类型
2. `TableManager` 支持跳过下注轮直接发牌
3. UI 增加 SNAP Fold 按钮（替代传统 Fold）
4. 退款逻辑：仅扣 1bb（行业标准）

### All-In 保险机制解析

**什么是 All-In 保险？**
当玩家全下后，可以**购买保险**：如果输掉这局，保险赔付购买金额。

| 保险类型          | 说明                          |
| ----------------- | ----------------------------- |
| Full Insurance    | 赔付全部损失                  |
| Partial Insurance | 赔付损失的部分比例（50%/75%） |
| Odd Insurance     | 仅赔付 odd chip 部分          |
