### 四、P2-ROOM-UX-001~005 规格建议

#### P2-ROOM-UX-001：Ring Game / Tournament Tab 切换

**建议方案：**
- 在房间列表顶部增加 Tab 导航：All / Ring Games / Tournaments
- 保留当前 tier 筛选器
- Tab 切换时使用客户端路由（`?type=ring` / `?type=tournament`）
- 初次加载默认显示 "All"

**竞品参考：** GGPoker 三标签（All / Ring Games / Tournaments）+ 房间数量 badge

---

#### P2-ROOM-UX-002：房间状态 Badge

**建议方案：**
- 🟢 "Waiting"（绿色）- 房间状态为 WAITING 或玩家数 < maxPlayers
- 🔵 "Playing"（蓝色）- 房间状态为 DEALING / PREFLOP / FLOP / TURN / RIVER / SHOWDOWN
- 🟡 "Starting Soon"（黄色，可选）- 距离满员差 1-2 人

**位置：** 房间卡片左上角或右上角，与房间名称同行

**竞品参考：** GGPoker 三色 badge + 888poker 绿/红双色

---

#### P2-ROOM-UX-003：空状态引导 CTA

**建议方案：**
- 无匹配房间时显示插画 + 文案 + CTA 按钮
- 文案：「No rooms match your filters」
- 按钮：「Create a Room」或「Clear Filters」

**竞品参考：** GGPoker "Be the first to start one!"

---

#### P2-ROOM-UX-004：收藏常玩房间

**建议方案：**
- 房间卡片右上角 ⭐ 按钮
- 点击后切换收藏状态（本地 localStorage 存储 favorites 数组）
- 收藏的房间在列表顶部优先显示
- 可选：在 rooms 页面增加「Favorites」tab

**数据结构：**
```ts
// localStorage
{ "favoriteRooms": ["roomId1", "roomId2"] }
```

**竞品参考：** GGPoker ⭐ 收藏 + 优先排序；888poker hover 时显示收藏按钮

---

#### P2-ROOM-UX-005：分享邀请链接/二维码

**建议方案：**
- 私人房间（password-protected）卡片增加「Share」按钮
- 点击弹出 Modal：显示邀请码 + 复制链接按钮 + 二维码（使用 qrcode 库生成）
- 邀请 URL 格式：`https://app.example.com/rooms?join=<roomId>&code=<inviteCode>`
- 加入房间时检测 URL 参数，自动填充房间号和邀请码

**二维码展示：** 使用 `qrcode.react` 或 `qrcode` npm 包

**竞品参考：** WSOP 邀请码分享；GGPoker Share 按钮 + 链接 + 二维码

---

### 五、房间 Web URL 结构建议

当前问题：创建私人房间后无法分享，只能手动告诉朋友房间号。

**建议 URL 方案：**
```
/rooms?join=<roomId>&code=<inviteCode>
```

- `join` param：自动带入房间 ID
- `code` param：自动带入邀请码（免手动输入）
- 页面加载时检测 params：
  - 如果带 `join`+`code`：弹出密码输入框（如果房间有密码）
  - 如果带 `join` 无 `code`：直接尝试加入房间
