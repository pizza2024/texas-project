## 竞品调研 — Rakeback 体系专项分析

> 上一轮已分析 Daily Bonus、Emoji Reactions、Club Systems。本轮补充 Rakeback。

### Rakeback 定义

Rakeback = 平台从每局抽水中返还给玩家的比例。是德州扑克最重要的留存机制之一。

### 行业标准做法

| 竞品             | Rakeback 机制                                       |
| ---------------- | --------------------------------------------------- |
| CoinPoker        | CHP 代币持有量决定返水比例（5%~40%），周结算        |
| GGPoker/Natural8 | Ocean Points 返水（0.05~0.40 BB/100），VIP 等级加成 |
| PokerStars       | VIP 星级系统，StarsCoin 返水，跨游戏积分池          |
| 888poker         | 直落式返水（25%~40%），周结算                       |
| PartyPoker       | 分级返水（20%~40%），月度奖励                       |

### 核心参数解析

```
Rake = 平台从每局抽的佣金（通常 5%~10% of pot）
Rakeback = 返还给玩家的比例（平台出，激励玩家）
净 rake = 玩家实际贡献的抽水（按手牌数或 pot 比例分摊）

典型计算方式：
1. 周零负（Weekly Net Rake）：玩家一周贡献的净抽水
2. 返水比例：根据 VIP 等级 5%~40% 不等
3. 返水金额 = 净 rake × 返水比例
```

### 竞品返水比例对照

| 等级 | CoinPoker | GGPoker     | 888poker |
| ---- | --------- | ----------- | -------- |
| 新人 | 5%        | 0.05 BB/100 | 25%      |
| 常规 | 15%       | 0.15 BB/100 | 30%      |
| 高额 | 25%       | 0.30 BB/100 | 35%      |
| VIP  | 40%       | 0.40 BB/100 | 40%      |

### 建议实现方案（MVP）

```typescript
// 方案：周结算直落式返水
// 数据模型
model PlayerRakeback {
  id            String   @id @default(uuid())
  userId        String
  weekStartDate DateTime  // 周起始（UTC Monday）
  netRake       Decimal  // 净抽水（玩家贡献部分）
  rakebackRate  Decimal  // 返水比例（0.05~0.40）
  rakebackAmt   Decimal  // 返水金额
  status        String   // PENDING/PAID
  createdAt     DateTime @default(now())
}

// 返水计算时机
// 1. 每周一 UTC 0点结算上周
// 2. 通过 BullMQ 定时任务批量处理
// 3. 结算后：生成 Transaction 记录 + 发放 chips

// 返水比例速查表（可配置）
const RAKEBACK_TIERS = [
  { minRake: 0,        rate: 0.05 },  // 新人/非活跃
  { minRake: 100,     rate: 0.10 },  // 常规玩家
  { minRake: 500,     rate: 0.20 },  // 活跃玩家
  { minRake: 2000,    rate: 0.30 },  // 高额玩家
  { minRake: 10000,   rate: 0.40 },  // VIP
]
```
