### Stats Page UX 深度分析

#### 1. 本项目优势
- **数据准确**：盈利/亏损计算清晰，颜色区分合理
- **最近手牌列表**：快速回顾，实用性强
- **UI 风格统一**：深色扑克主题，与整体品牌一致

#### 2. 差距分析
- **无 VIP 等级感知**：玩家无法感知自己的成长阶段，缺乏成就激励
- **无图形化趋势**：纯数字展示，不如 GGPoker 的 equity curve 直观
- **无成就系统**：缺少徽章、里程碑等游戏化元素
- **无社交维度**：无法分享战绩、查看好友排名

#### 3. 优先实施建议

**🔴 高优先 — P2-PROFILE-001: VIP 等级 + 徽章墙**

```
数据基础已具备：
- User.role (PLAYER/ADMIN/SUPER_ADMIN)
- User.elo (ELO 评分已有)
- Transaction 历史数据

建议字段扩展：
- User.vipLevel: Bronze/Silver/Gold/Platinum (基于月累计充值 or 佣金)
- User.achievementBadges: string[]

实现路径：
Phase 1: 简单 VIP 等级展示（头像框颜色区分）
Phase 2: 成就徽章系统（基于手数/盈利里程碑）
Phase 3: 徽章墙 UI
```

**🟡 中优先 — P2-PROFILE-002: Equity 曲线图**

```
基于 Hand + Settlement 数据：
- 已有 Hand.createdAt 索引
- 已有 Settlement.amount

实现路径：
- 后端新增 /user/stats/equity-curve API（返回每日累计盈利）
- 前端用 Recharts（admin 已在用）渲染曲线
- 对标 GGPoker equity curve
```
