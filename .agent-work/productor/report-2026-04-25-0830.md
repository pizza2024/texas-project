## Club 私人邀请码 — 完整设计文档

### 竞品调研摘要

| 平台       | 邀请机制                            | Rakeback上限 | Club 特色                    |
| ---------- | ----------------------------------- | ------------ | ---------------------------- |
| GGPoker    | 私人邀请码（6位字母数字）+ 链接分享 | 60%+         | 专属高rakeback房间，私人Club |
| PokerStars | 俱乐部（Club）系统，无公开搜索      | 40-50%       | 5星VIP，私人游戏             |
| CoinPoker  | 推荐码 + 私人俱乐部                 | 40%          | 加密货币充值                 |
| 888poker   | 私人俱乐部邀请                      | 36%          | 亚洲市场较弱                 |

### 设计目标

1. Club Owner/Admin 可生成一次性或多次性邀请码
2. 被邀请玩家凭码加入 Club，自动获得 Member 身份
3. 邀请码可设置有效期和使用次数上限
4. 支持通过 URL 分享（`/club/join?code=XXXX`）
5. 未登录用户引导注册后自动加入

### DB Schema（Prisma Migration）

```prisma
model ClubInviteCode {
  id         String    @id @default(uuid())
  clubId     String
  code       String    @unique  // 6位大写字母数字，不含易混淆字符（0/O/1/I/L）
  creatorId  String
  maxUses    Int       @default(1)    // 0 = 无限
  usedCount  Int       @default(0)
  expiresAt  DateTime?
  createdAt  DateTime  @default(now())
  isActive   Boolean   @default(true)

  club       Club      @relation(fields: [clubId], references: [id], onDelete: Cascade)
  creator    User      @relation("ClubInviteCreator", fields: [creatorId], references: [id])

  @@index([clubId])
  @@index([code])
}

model Club {
  // ... existing fields
  inviteCodes ClubInviteCode[]
}
```

### API 设计

| 方法   | 路径                              | 权限        | 说明                                      |
| ------ | --------------------------------- | ----------- | ----------------------------------------- |
| POST   | `/clubs/:id/invite-codes`         | Owner/Admin | 生成邀请码                                |
| GET    | `/clubs/:id/invite-codes`         | Owner/Admin | 列出所有邀请码（含使用情况）              |
| DELETE | `/clubs/:id/invite-codes/:codeId` | Owner/Admin | 撤销邀请码                                |
| POST   | `/clubs/join-by-code`             | 登录用户    | 凭码加入 Club                             |
| GET    | `/clubs/validate-code?code=XXXX`  | 公开        | 验证邀请码有效性（返回 club name/avatar） |

**POST /clubs/:id/invite-codes**

```json
// Request
{ "maxUses": 5, "expiresInHours": 72 }

// Response
{
  "code": "XK7T9",
  "url": "https://app.texas.com/club/join?code=XK7T9",
  "maxUses": 5,
  "expiresAt": "2026-04-28T08:30:00Z",
  "club": { "id": "...", "name": "Pro Club", "avatar": "..." }
}
```

**POST /clubs/join-by-code**

```json
// Request
{ "code": "XK7T9" }

// Response 201
{ "clubId": "...", "clubName": "Pro Club", "role": "MEMBER" }

// Response 400 — 码无效/已过期/已用完
{ "message": "Invalid or expired invite code" }
```

### Code 生成规则

- 长度：6位
- 字符集：`ABCDEFGHJKMNPQRSTUVWXYZ23456789`（排除 0/O/1/I/L）
- 格式：`XXXXXX`（无分隔符）
- 唯一性：DB unique index + 生成时重试
- 有效期：默认72小时，可配置

### 安全设计

1. **速率限制**：同一 IP 5分钟内最多验证10次（防暴力猜解）
2. **错误提示统一化**：无效码和已过期都返回 "Invalid or expired invite code"（防枚举）
3. **码格式验证**：正则 `^[ABCDEFGHJKMNPQRSTUVWXYZ23456789]{6}$`
4. **已使用次数校验**：DB 层 `usedCount < maxUses`（当 maxUses > 0）
5. **邀请码撤销**：Owner 可随时撤销，撤销后码立即失效
