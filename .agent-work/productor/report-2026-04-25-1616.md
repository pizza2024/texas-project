### 关键发现

#### 1. GGPoker — 行业标杆

**胜率曲线（Equity Curve）**
- 时间轴叠加每玩家胜率折线
- 颜色区分各玩家
- 悬停显示精确百分比

**智能 Pot Odds 计算器**
- 浮窗形式，点击牌局任意节点显示
- 自动计算：需要胜率 vs 当前底池赔率
- 决策提示："Call" / "Fold" / "Raise"

**播放控制**
- 速度：0.25x / 0.5x / 1x / 2x / 4x
- 自动播放模式（到终点暂停/循环）
- 关键节点自动放大

#### 2. 888poker — SnapCam 特色

**SnapCam**
- 摊牌结果自动截取关键帧
- Pot Odds 计算器浮窗
- 视觉化展示赔率决策

#### 3. CoinPoker — Crypto 原生

- P2P 筹码市场（玩家间交易）
- USDT 充值即时到账
- Hand Replay 无胜率曲线，但有基础时间轴

## Hand Replay Phase 2 规格制定

基于竞品分析，为本项目 Phase 2 定义以下功能规格：

### P1 功能（下一冲刺）

| 功能 | 规格 | 竞品参考 |
|------|------|---------|
| 胜率曲线 | 时间轴叠加 equity % 折线，每玩家不同颜色，悬停显示精确值 | GGPoker |
| Pot Odds 计算器 | 浮窗，点击任意节点显示：当前 pot / 需要 call / 隐含赔率 | GGPoker + 888poker |
| 播放速度控制 | 0.5x / 1x / 2x / 4x 切换按钮 | GGPoker |

### P2 功能（后续迭代）

| 功能 | 规格 | 竞品参考 |
|------|------|---------|
| 自动播放模式 | 开启后自动播放到 SHOWDOWN，可选循环 | GGPoker |
| 关键决策提示 | 在关键节点（all-in/call）标注决策分析 | GGPoker |

### Phase 2 数据需求

胜率曲线需要服务端在 replay 数据中增加：
```
每节点各玩家的 equity %
```
这需要在 `hand-history.service.ts` 的 `buildReplayTimeline()` 中调用手牌评估器计算每个时间点的胜率。

## Tournament SNG 竞品参考

| 平台 | SNG 特色 | 奖金结构 |
|------|---------|---------|
| GGPoker | 多种 buy-in 级别，急速 SNG，淘汰加速 | 100% prize pool |
| 888poker | Snap Poker (快速增盲) | 100% prize pool |

| CoinPoker | Crypto buy-in，即时开始 | 100% prize pool |

### Tournament SNG Phase 1 关键规格（草案）

| 字段 | 值 |
|------|---|
| 参赛人数 | 固定 8 人 |
| Buy-in | 从钱包扣 chips |
| 座位 | 固定随机座位，淘汰后空位 |
| 满8人倒计时 | 60 秒自动开始 |
| 盲注结构 | 固定 (如 100/200，每5分钟涨盲) |
| 奖金分配 | 第1名 60%，第2名 30%，第3名 10% |
| 淘汰 | 0 chips 立即离开桌子 |
| Lobby | Tournament 专属入口页，显示进行中/已结束 |
