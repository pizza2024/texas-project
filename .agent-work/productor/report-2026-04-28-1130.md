### 问题描述

通过代码审查发现，Blast Phase 1+2 仅完成了**后端逻辑**（BlastService + Lobby API），但**前端UI完全缺失**：

| 缺失组件         | 说明                                | 优先级 |
| ---------------- | ----------------------------------- | ------ |
| BLAST Lobby Tab  | 赛事大厅Tab切换（全部/$1/$3/$7...） | P1     |
| 轮盘旋转动画     | 3D轮盘显示奖池倍数（3-5秒动画）     | P1     |
| 匹配等待UI       | 倒计时（30秒）+ 已报名人数（X/3）   | P1     |

| `BlastCard` 组件 | 赛事卡片（buyin/等待人数/倍数）     | P1     |
| BLAST 牌桌页面   | `/blast/[id]` 游戏桌                | P1     |

### 对比：Tournament SNG UI现状

已有Tournament UI可参考：

- `apps/web/app/schedule/` — 赛程列表页
- `ScheduleList.tsx` — 赛事卡片列表
- `schedule/[id]/page.tsx` — 赛事详情页

Blast UI应**复用 Tournament 的设计模式**，但使用不同的数据源和交互。
