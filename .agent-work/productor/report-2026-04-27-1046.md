### 匿名模式深度分析

**CoinPoker Anonymous Mode 机制：**

```
1. 玩家可选择"匿名桌" — 桌内玩家显示为 Anonymous #1, Anonymous #2
2. 不显示真实用户名、头像、历史数据
3. 特殊处理：战绩不公开，无法追踪
4. 适合：反 HUD 保护、休闲玩家隐私
```

**匿名模式的商业价值：**

| 维度     | 正面                          | 负面                           |
| -------- | ----------------------------- | ------------------------------ |
| 用户获取 | 休闲玩家安全感↑、隐私顾虑消除 | 硬核玩家迁移（数据追踪受阻）   |
| 留存     | 匿名桌降低被剥削感            | 无法建立玩家历史，降低社交粘性 |
| HUD 防御 | 阻止数据爬取/剥削             | 技术上难以完全阻止（屏幕识别） |
| 监管合规 | 部分司法管辖区要求            | 洗钱风险略微上升               |

**实施难度评估：**

```
Anonymous Mode 实施成本：Medium-High
- UI：前端将 username → "Anonymous #N"（仅房内可见）
- 后端：WebSocket 事件过滤 username 字段
- 数据库：战绩记录保留（审计用），但不暴露给对手
- 反 HUD：Bot/AI 玩家可通过图像识别绕过，需额外反脚本保护
```

### 本项目 Anonymous Mode 实施路径

