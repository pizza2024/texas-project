### Rebuy 机制详解

**核心定义：**

- **Rebuy：** 锦标赛开始后，在规定时间内，玩家可以重新购买与初始筹码相同的筹码包。价格通常等于报名费的一部分（常见 50-100%）。
- **Addon：** Rebuy 阶段结束后，提供最后一次购买额外筹码的机会，通常比 Rebuy 更大（1.5x-2x 筹码），价格略高或相同。
- **Re-Entry：** 淘汰后重新报名整个锦标赛，与 Rebuy 不同（不保留原有筹码和排名）。

**行业标准规格：**

| 参数           | 常见值                  | 说明                    |
| -------------- | ----------------------- | ----------------------- |
| Rebuy 时间窗口 | 起始后 30-60 分钟       | GGPoker 通常 30-45 分钟 |
| Rebuy 芯片量   | 初始筹码量 100%         | 通常等于起始筹码        |
| Rebuy 价格     | 报名费 50-100%          | GGPoker: 100% (即原价)  |
| Addon 芯片量   | 初始筹码 100-150%       | 略大于 Rebuy            |
| Addon 时间窗口 | Rebuy 结束后 1-3 分钟内 | 通常紧接 Rebuy 结束     |
| Addon 价格     | 报名费 50-100%          | 部分平台免费 Addon      |
| Rebuy 次数限制 | 无限制 或 最多 2-3 次   | GGPoker 无限制          |

### GGPoker Rebuy/Addon 详细机制

**UI 提示：**

- 界面显示 "Rebuy Available" 倒计时
- 选手筹码低于起始筹码时自动提示
- Addon 窗口有独立倒计时（通常比 Rebuy 短）
- 淘汰选手显示 "Re-Entry Available"（如有）

**数据结构建议（本项目）：**

```typescript
// Room 或 TournamentConfig 新增字段
interface RebuyConfig {
  enabled: boolean;
  rebuyPeriodMs: number; // e.g., 3600000 (60 min)
  rebuyChips: number; // e.g., 初始筹码量
  rebuyPrice: number; // e.g., 报名费的 50%
  maxRebuyCount: number; // -1 = 无限制
  addonEnabled: boolean;
  addonPeriodMs: number; // e.g., 180000 (3 min after rebuy)
  addonChips: number; // e.g., 初始筹码的 150%
  addonPrice: number;
}
```
