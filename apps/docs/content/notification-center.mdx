## 3. Data Model (Prisma)

```prisma
model Notification {
  id        String   @id @default(cuid())
  userId    String   @index()
  type      String   // "friend_online" | "club_tournament_starting" | ...
  title     String
  body      String
  metadata  Json?    // 类型相关的额外数据
  read      Boolean  @default(false) @index()
  createdAt DateTime @default(now()) @index()

  @@index([userId, createdAt(sort: Desc)])
  @@index([userId, read, createdAt(sort: Desc)])
}
```

**索引说明：**
- `userId + createdAt` 降序：用于拉取最近通知列表（分页）
- `userId + read + createdAt` 降序：用于未读数统计查询

## 4. WebSocket Events

### 4.1 Server → Client

**Event: `notification`**

触发所有需要实时推送的通知。Payload 为完整的 Notification 对象（见上文各类型 schema）。

### 4.2 Client → Server

**Event: `mark_read`**

客户端标记单条或批量通知为已读。

```ts
// Client → Server
{
  "notificationIds": ["notif_001", "notif_002"]  // string[]，支持单条或批量
}

// Server → Client (ack or pushed update)
{
  "marked": ["notif_001", "notif_002"]
}
```

**Event: `mark_all_read`**

```ts
// Client → Server
{}

// Server → Client
{
  "markedCount": 15
}
```

## 5. Frontend UI

### 5.1 Bell Icon (Header)

- 位于页面右上角，JWT 登录后显示
- 未读数 > 0 时显示红色 Badge（最多显示 "99+"）
- 点击打开下拉面板

### 5.2 Dropdown Panel

- 最多显示最近 20 条通知（分页加载）
- 每条显示：图标（按类型）、标题、摘要、相对时间
- 按类型区分颜色标签：
  - `friend_online` — 蓝色（朋友）
  - `club_tournament_starting` / `tournament_starting` — 紫色（赛事）
  - `rakeback_ready` — 金色（返水）
  - `deposit_received` — 绿色（财务）
  - `hand_result` — 橙色（战绩）

### 5.3 Mark All Read

- 面板底部「全部已读」按钮
- 调用 `mark_all_read` WS 事件

### 5.4 Empty State

- 无通知时显示：🎉 "暂无新通知"
- 有历史通知时提供「查看全部」跳转链接

## 6. Retention Policy

| 规则 | 值 |
|------|-----|
| 每用户最大保留条数 | **100 条**（按 createdAt 倒序） |
| 超过 100 条时 | 定时任务删除最旧的已读通知 |
| 超过 30 天 | 无条件删除（无论已读状态） |
| 清理触发 | 后台 Cron Job，每日 UTC 00:00 执行 |

```sql
-- 清理 SQL 示例（每日 Cron）
DELETE FROM "Notification"
WHERE "userId" = $1
  AND "createdAt" < NOW() - INTERVAL '30 days';

WITH ranked AS (
  SELECT id, ROW_NUMBER() OVER (PARTITION BY "userId" ORDER BY "createdAt" DESC) as rn
  FROM "Notification" WHERE "userId" = $1
)
DELETE FROM "Notification" WHERE id IN (SELECT id FROM ranked WHERE rn > 100);
```

```ts
      this.notificationService.create({
        userId: friend.friendId,
        type: 'friend_online',
        title: '好友上线',
        body: `你的好友 ${user.nickname} 已经上线`,
        metadata: {
          friendUserId: userId,
          friendNickname: user.nickname,

```

## 10. API Endpoints (REST)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/notifications` | 分页获取当前用户通知列表 |
| PATCH | `/notifications/read` | 批量标记已读（body: `{ ids: string[] }`） |
| PATCH | `/notifications/read-all` | 全部标记已读 |
| GET | `/notifications/unread-count` | 获取未读数（用于 Badge） |

```
GET /notifications?page=1&limit=20
Response: { data: Notification[], total, page, limit, unreadCount }

GET /notifications/unread-count
```
